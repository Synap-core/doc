# Introduction

Synap is an AI operating system for knowledge work.

It gives you:

- a sovereign data pod you control
- a structured data model (entities, views, channels)
- an event-governed mutation pipeline
- pluggable intelligence and external agent connectivity

## The core model

| Layer | Purpose |
|---|---|
| Entities | Typed data objects and relationships |
| Views | Lenses over the same data graph |
| Channels | Human + AI collaboration surface |

## Why teams adopt Synap

- keep control of data and infrastructure
- add AI without losing governance
- unify apps/workflows on one data foundation
