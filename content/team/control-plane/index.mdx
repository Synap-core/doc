# Control Plane

The SaaS orchestration layer for pod lifecycle, account management, and multi-service coordination.

## Purpose

- Define provisioning lifecycle for pods and related services.
- Document control-plane APIs, jobs, and infrastructure responsibilities.
- Centralize operational procedures and incident paths for CP-managed systems.

## Target users

- Control Plane engineers
- Infrastructure and operations teams
- Product/ops teammates needing provisioning and tenancy context

- [Pod management](/team/control-plane/pod-management) — lifecycle actions and orchestration
- [CP dashboard & pod updates](/team/control-plane/cp-dashboard-updates) — admin UI, update-status API, rolling-update semantics, failure modes
- [Pod install](/team/control-plane/pod-install) — setup path for deployed pods
