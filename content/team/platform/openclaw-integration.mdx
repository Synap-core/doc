### 1. Unified Skill (`synap`)
```bash
openclaw skill install synap
```
One skill covering knowledge graph, message relay, A2AI, and governance. Hub Protocol REST only (`/api/hub/*`). No MCP, no tRPC.

**Location:** `synap-backend/skills/synap/SKILL.md`

### 2. CLI (`@synap/cli`)
```bash
npx @synap/cli init
```
Three-path flow:
- **Path A:** Existing OpenClaw → detect, harden, pod, skill, seed
- **Path B:** Fresh server → resource check, bundle install, connect
- **Path C:** Laptop → managed/VPS/existing pod

Commands: `init`, `connect`, `security-audit`, `status`, `update`

**Location:** `synap-cli/` (separate repo: `synap-core/synap-cli`)

### 3. OpenAI-Compatible Endpoint
```
POST /v1/chat/completions
```
IS as AI provider for OpenClaw. Four-tier routing (90% cheap, 10% Claude). Model aliases: `synap/auto`, `synap/free`, `synap/balanced`, `synap/advanced`, `synap/complex`.

**Location:** `synap-backend/packages/api/src/routers/external/openai-compat.ts`

### 4. Pricing Update
- Pod: €15/mo (solo tier)
- Pod + AI: €30/mo (pro tier, save €5)
- Custom: contact sales
- OpenClaw addon: free (same server)
- Self-hosted: free forever

### 5. Telegram Deactivated
Feature flag: `TELEGRAM_BOT_ENABLED` (default OFF). OpenClaw handles external channels.

### 6. Dashboard Context
CP Dashboard shows per-user: tier, servers, OpenClaw status, spend, intelligence access.

### 7. Workspace templates

Two templates ship in `synap-backend/templates/`:

- `builder-workspace.json` — **canonical dev workspace** (Phase 8). Composes 13 DevPlane profiles with `project` / `task` / `skill` / `provider` from the older agent-os primitives. Seeded by Eve via `POST /api/hub/workspaces/from-definition` (allowlisted to `eve` and `coder` agents). See [Builder Workspace](./builder-workspace.mdx).
- `agent-os.json` — **still in use, NOT deprecated**. Read by `setup-openclaw.sh` and other downstream consumers that bootstrap an agent-OS shape directly. New builder use cases should pick `builder-workspace.json`; the agent-os template stays for OpenClaw setup and any external scripts already wired to it.

The `builder-workspace.json` template DROPS the `agent` profile entirely — agents are pod-level identities (`users.userType="agent"` + `agents` table + `agent_configs`), never workspace entities. `agent-os.json` retains its `agent` profile for backward compatibility with existing setup paths.

## Architecture

```
User's Machine                    Synap Infrastructure
┌─────────────────┐              ┌──────────────────────┐
│ npx @synap/cli  │              │ Synap Pod (€15/mo)   │
│   init          │──── HTTPS ──►│ /api/hub/* (REST)    │
│                 │              │ /v1/* (OpenAI compat) │
│ OpenClaw        │              │ PostgreSQL + Typesense│
│  + synap skill  │              │ + pgvector + MinIO    │
└─────────────────┘              └──────────────────────┘
```

Self-hosted: both on same server, free.
Managed: pod in cloud, OpenClaw anywhere.
