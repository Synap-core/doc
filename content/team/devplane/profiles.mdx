All DevPlane entities are workspace-scoped and defined in `lib/devplaneWorkspaceDefinition.ts`. The backend has no knowledge of these profiles — they are created by `createFromDefinition` on first login and verified on every subsequent login by the readiness gate.

## Required profiles constant

`hooks/useWorkspaceReadiness.ts` exports `REQUIRED_PROFILE_SLUGS` — an array of all 11 slugs above. The readiness gate checks that every slug in this array appears in `trpc.profiles.list` response. If any are missing, `createFromDefinition` is called to add them.

When adding a new profile to DevPlane:
1. Add it to `DEVPLANE_WORKSPACE_DEFINITION.profiles` in `lib/devplaneWorkspaceDefinition.ts`
2. Add its slug to `REQUIRED_PROFILE_SLUGS` in `hooks/useWorkspaceReadiness.ts`
3. Existing workspaces will automatically pick it up on next login via the readiness gate
