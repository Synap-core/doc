## Critical Path

```
A1 (person profile) ──→ B1 (relay router) ──→ C2-C6 (frontend cells)
                    ├──→ B5 (telegram import) ──→ E2 (upload endpoint)
                    └──→ D1-D8 (mobile app, parallel)

C5 (capture overlay) depends on existing capture pipeline (already built)
D2 (pod connect) depends on @synap-core/pod-connect (already built)
```

**Minimum viable demo:** A1 + B1 + B2 + C2 + C4 + C5 = ~5-7 days
(Person profiles extended, feed with manual captures, person detail)

## Open Decisions (Pre-Implementation)

1. **Repo structure:** New `relay-app/` at root level, or inside `other/` like synap-life-feed?
2. **Relay as Browser app:** Register Relay as a new app in `manifest.ts`, or integrate into existing Library/Dashboard?
3. **Telegram export format:** Support both Telegram Desktop JSON export and Telegram API JSON?
4. **Voice transcription:** Whisper API (via IS) or on-device (expo-speech)?
5. **Offline first:** How much data to cache locally on mobile? Just contacts list? Full timeline?
