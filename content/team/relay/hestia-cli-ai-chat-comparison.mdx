# Chat UI Options for the Eve Stack

The Eve stack supports two optional self-hosted chat UIs, available as Docker Compose profiles. **Open WebUI** is the primary option — it integrates directly with Synap IS and the Hub MCP server. **LobeChat** is a lightweight alternative focused on Ollama.

---

## Comparison

| Feature | Open WebUI | LobeChat |
|---------|-----------|----------|
| **Primary role** | Synap IS endpoint + MCP tools | Ollama-focused chat |
| **Synap IS integration** | Yes — `synap/auto`, `synap/balanced`, etc. | Not natively wired |
| **MCP / Hub tools** | Yes — via `POST /mcp` | No |
| **Database** | SQLite (built-in) | None (stateless) |
| **Ollama fallback** | Yes | Yes (primary) |
| **Document RAG** | Yes | No |
| **User management** | Yes | No (access code only) |
| **Plugin ecosystem** | Pipelines sidecar | Plugin marketplace |
| **Mobile support** | Good | Excellent |
| **Port** | 3011 | 3010 |
| **Eve component ID** | `openwebui` | `lobechat` |

---

## Open WebUI (primary)

Open WebUI connects to **Synap Intelligence Service** as its OpenAI-compat backend. Model aliases (`synap/auto`, `synap/balanced`, `synap/advanced`, `synap/complex`, `synap/free`) route through the IS, which applies provider policy, governance, and logs to Synap.

MCP tool calls (memory, entities, skills) go directly to **Synap Hub** via `POST /mcp`.

**Install via Eve:**

```bash
eve install --components traefik,synap,openwebui
# or pick "AI chat server" preset in `eve install`
```

**Start/stop:**

```bash
docker compose --profile openwebui up -d
docker compose --profile openwebui down
```

**Update:**

```bash
eve update --only openwebui
```

**Logs:** `docker logs hestia-openwebui`  
**Port:** `3011`

Full reference: [Open WebUI — Eve CLI docs](/team/eve-cli/open-webui)

---

## LobeChat (alternative)

LobeChat is a polished chat UI best suited for Ollama-only use. It does not natively integrate with Synap IS or Hub MCP. Use it when you want a plugin marketplace or strong mobile UX without Synap wiring.

**Start/stop:**

```bash
docker compose --profile lobechat up -d
docker compose --profile lobechat down
```

**Logs:** `docker logs hestia-lobechat`  
**Port:** `3010`

---

## Running both simultaneously

The compose profiles are independent:

```bash
docker compose --profile openwebui --profile lobechat up -d
```

---

## Decision guide

**Use Open WebUI when:**
- You want AI responses routed through Synap IS (model aliases, governance, logging)
- You need Hub tools (memory, entity search, skills) during conversations
- You need document RAG or multi-user accounts
- You want `eve update` lifecycle management

**Use LobeChat when:**
- You only need Ollama and prefer a lighter setup
- Mobile UX or plugin marketplace matters more than Synap integration
