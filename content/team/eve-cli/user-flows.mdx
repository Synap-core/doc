## The three paths (`eve setup`)

Eve organizes installation around **three product paths** (CLI profiles):

1. **Inference only** (`inference_only`) ‚Äî Local **Ollama** + **Traefik**-secured gateway (**no** Synap Data Pod).
2. **Synap only** (`data_pod`) ‚Äî **Data Pod** via official **`synap install`**; Eve can add **other organs** later without bundling Ollama in this profile.
3. **Both** (`full`) ‚Äî Data Pod **first**, then Ollama on **`eve-network`** + same gateway pattern (no host publish on **11434** for Ollama in the documented layout).

---

## Order of operations (recommended)

1. **Secrets / profile** ‚Äî Run `eve setup` (or prepare `.eve/secrets/secrets.json`) so Synap URL + Hub API key fields exist where needed.
2. **Builder** (if you use codegen deploy) ‚Äî `eve builder init` so OpenCode / OpenClaude share Hub `.env` + skills.
3. **Legs / tunnel / RSSHub / OpenClaw** ‚Äî After Brain surface is stable.

---

## Public exposure semantics (critical)

- **Synap domain** in the wizard configures the **Data Pod** public URL (Caddy, API, auth) ‚Äî ‚Äúwhere users sign in to the pod.‚Äù
- **Eve Legs** (Traefik, tunnel hostname) is a **separate** layer for **sidecar** and gateway routes. A public Synap domain does **not** automatically expose every Eve service.
- Documented in depth in repo **`docs/EVE_SETUP_PROFILES.md`**.

---

## `eve setup` vs `eve init`

- **`eve setup`** ‚Äî Full guided wizard (profiles, AI foundation, network, Synap options, optional tunnel).
- **`eve init`** ‚Äî **Alias** that forwards to **`eve setup`** unless you pass an explicit **`--synap-repo`** for legacy direct brain flows ‚Äî **one mental model**: prefer **`eve setup`**.

---

## `eve` vs `synap` scripts

- **`synap`** (in **backend** repo) ‚Äî Installs **Synap Data Pod** Docker stack (`deploy/docker-compose.yml`).
- **`eve`** ‚Äî Installs/orchestrates **Eve** sidecars and **calls** `synap install` when profile is `data_pod` or `full`. See [Synap integration](/team/eve-cli/synap-integration).

## Scripted / CI examples

Non-interactive patterns (`--yes`, `--profile`, `--domain`, `--synap-repo`, tunnel flags) are kept current in **hestia-cli** `README.md` and **`docs/EVE_SETUP_PROFILES.md`**. Use **`eve setup --dry-run`** or **`eve --json setup --dry-run`** for plans without side effects.
