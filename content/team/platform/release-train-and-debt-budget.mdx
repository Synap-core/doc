# Release train & technical debt budget

## Release train (recommended)

- **Pod (synap-backend):** Tag semantic versions; document breaking API changes in changelog.
- **Intelligence Service:** Version alongside pod when Hub Protocol or tool schemas change.
- **Clients (Browser, Relay):** Track minimum compatible pod version in release notes.

## Debt budget

- Each cycle, allocate **fixed capacity** (e.g. 15–20%) to reliability, docs, and cleanup — see **[System completion](/team/platform/system-completion)** tracks.

## Quarterly review

- Revisit SLOs from **[S-tier Definition of Done](/team/platform/s-tier-definition-of-done)** and onboarding funnel metrics from **[Onboarding analytics events](/team/platform/onboarding-analytics-events)**.
