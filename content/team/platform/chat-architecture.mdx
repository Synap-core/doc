## Implementation Steps

### Phase 1: Fix Immediate Issues
1. Fix `ChatInputNative` to use `lucide-react-native`
2. Update `chat-interface/package.json` to include `lucide-react-native` as peer dependency

### Phase 2: Unify Components
1. Create unified `ChatInput` that:
   - Uses Tamagui's `TextArea` (works on both platforms)
   - Conditionally imports icons from correct package
   - Shares all business logic via hooks

2. Create unified `MessageBubble` that:
   - Uses platform detection for animations
   - Shares styling between platforms

3. Create unified `MessageList` that:
   - Uses Tamagui's `ScrollView` (works on both)

### Phase 3: Re-exports for Backward Compatibility
1. Make `native/` files re-export from unified `components/`
2. Add deprecation warnings

### Phase 4: Update Consumers
1. Update `chat-app` to use unified components
2. Update `relay-app` to use `chat-app` directly (or unified chat-interface)

## Platform Detection Strategy

```tsx
// platform detection utilities
import { Platform } from 'react-native';

export const isWeb = Platform.OS === 'web';
export const isNative = Platform.OS !== 'web';

// Icon imports - use correct package per platform
export const Icons = isWeb 
  ? require('lucide-react')
  : require('lucide-react-native');

// TextInput - Tamagui handles this, but for custom:
export const TextInputComponent = isWeb ? 'textarea' : TextInput;
```

## Tamagui Advantage

Tamagui already abstracts most platform differences:
- `YStack`, `XStack` ‚Üí `View` (native) / `div` (web)
- `SizableText` ‚Üí `Text` (native) / `span` (web)
- `Button` ‚Üí works on both
- `ScrollView` ‚Üí works on both (has `os` prop for web/native behavior)

The ONLY platform-specific things are:
1. Icons (`lucide-react` vs `lucide-react-native`)
2. Text input element (`&lt;textarea>` vs `&lt;TextInput>`)
3. Animations (framer-motion on web, reanimated on native - optional)
