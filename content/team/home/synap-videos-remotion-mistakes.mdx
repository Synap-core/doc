# Common Mistakes When Using 3D in Remotion

This document explains what went wrong with your initial 3D component and how to fix it.

---

## ❌ The Problem: Your Original Code

Your code had several critical issues that make it incompatible with Remotion:

### Issue #1: Using Standard React Three Fiber Canvas

```tsx
// ❌ YOUR CODE - WRONG
import { Canvas, useFrame } from '@react-three/fiber';

return (
  <Canvas shadows camera={{ position: [0, 0, 6], fov: 45 }}>
    {/* ... */}
  </Canvas>
);
```

**Why this fails:**
- `Canvas` from @react-three/fiber is designed for **interactive 3D apps**
- It uses WebGL context that conflicts with Remotion's rendering pipeline
- Results in error: `TypeError: Cannot read properties of undefined (reading 'S')`

**✅ FIX:**
```tsx
import { ThreeCanvas } from '@remotion/three';
import { useVideoConfig } from 'remotion';

const { width, height } = useVideoConfig();

return (
  <ThreeCanvas width={width} height={height} camera={{ position: [0, 0, 6], fov: 45 }}>
    {/* ... */}
  </ThreeCanvas>
);
```

### Issue #2: Using useFrame() for Animation

```tsx
// ❌ YOUR CODE - WRONG
import { useFrame } from '@react-three/fiber';

useFrame((state, delta) => {
  const t = state.clock.getElapsedTime();

  if (groupRef.current) {
    groupRef.current.position.y = Math.sin(t * 0.5) * 0.2;
  }

  if (meshRef.current) {
    meshRef.current.rotation.y += delta * rotationSpeed;
  }
});
```

**Why this fails:**
- `useFrame()` runs **asynchronously** based on browser refresh rate
- Uses `delta` (time between frames) which varies
- Rendering frame 100 twice gives **different results each time**
- Videos will **flicker and be inconsistent**

**✅ FIX:**
```tsx
import { useCurrentFrame } from 'remotion';

const frame = useCurrentFrame();

// All animations based on frame number (deterministic)
const floatY = Math.sin(frame * 0.05) * 0.2;
const rotation = frame * rotationSpeed;

return (
  <group position={[0, floatY, 0]}>
    <mesh rotation={[0, rotation, 0]}>
      {/* ... */}
    </mesh>
  </group>
);
```

### Issue #3: Interactive Controls

```tsx
// ❌ YOUR CODE - WRONG
import { OrbitControls } from '@react-three/drei';

<OrbitControls
  enableZoom={false}
  enablePan={false}
  autoRotate={podState === 'active'}
  autoRotateSpeed={0.5}
/>
```

**Why this fails:**
- `OrbitControls` allows **user interaction** (mouse drag, scroll)
- User interaction doesn't make sense in a **rendered video**
- `autoRotate` uses internal timing, not frame-based

**✅ FIX:**
```tsx
// No OrbitControls in videos!
// Camera position is fixed or animated based on frame

const frame = useCurrentFrame();

// Camera can move based on frame
const cameraX = Math.sin(frame * 0.01) * 2;

<ThreeCanvas
  camera={{
    position: [cameraX, 0, 5],
    fov: 45
  }}
>
  {/* ... */}
</ThreeCanvas>
```

### Issue #4: React State for Animation

```tsx
// ❌ YOUR CODE - WRONG
const [mode, setMode] = useState('dark');
const [podState, setPodState] = useState('default');

<button onClick={() => setPodState('active')}>
  Activate
</button>
```

**Why this fails:**
- `useState` is for **interactive apps**, not videos
- Buttons don't make sense in rendered videos
- State doesn't advance with video playback

**✅ FIX:**
```tsx
import { useCurrentFrame } from 'remotion';

const frame = useCurrentFrame();

// State changes based on time in video
const state =
  frame < 90 ? 'idle' :
  frame < 180 ? 'active' :
  'processing';

// No buttons, state transitions automatically
```

### Issue #5: Environment Component

```tsx
// ❌ YOUR CODE - WRONG (but works)
import { Environment } from '@react-three/drei';

<Environment preset="city" />
```

**Why this is problematic:**
- `<Environment>` loads HDR images asynchronously
- Can cause rendering delays or inconsistencies
- Not necessary for most video use cases

**✅ FIX:**
```tsx
// Use simple lighting instead
<ambientLight intensity={0.4} />
<directionalLight position={[5, 5, 5]} intensity={0.8} />
<directionalLight position={[-5, -5, -5]} intensity={0.3} />
```

### Issue #6: UI Layout in 3D Component

```tsx
// ❌ YOUR CODE - WRONG
return (
  <div className="w-full h-screen flex flex-col md:flex-row">
    {/* Left panel with Canvas */}
    {/* Right panel with controls */}
  </div>
);
```

**Why this fails:**
- This is an **interactive application layout**
- Videos don't have "panels" or "controls"
- TailwindCSS classes might not be configured

**✅ FIX:**
```tsx
// Videos just render content, no UI chrome
return (
  <AbsoluteFill style={{ backgroundColor: "#0A0A0A" }}>
    <ThreeCanvas width={width} height={height}>
      {/* Your 3D content */}
    </ThreeCanvas>

    {/* Optional text overlays */}
    <div style={{ position: "absolute", top: 100, ... }}>
      <h1>Title</h1>
    </div>
  </AbsoluteFill>
);
```

## 🎯 The Mental Model Shift

### Interactive React App (Your Code)
- User controls time with mouse/keyboard
- Animations use `requestAnimationFrame` loop
- State changes from user interaction
- Canvas renders at screen refresh rate
- Each render can be different

### Remotion Video (Correct Approach)
- **Frame number controls everything**
- Animations calculate from frame number
- State changes from frame ranges
- Canvas renders **exactly** once per frame
- **Each render must be identical**

## ✅ Corrected Version: DataPod3D

Here's your component rewritten the Remotion way:

```tsx
import React from "react";
import { useCurrentFrame, useVideoConfig, spring, interpolate } from "remotion";
import { ThreeCanvas } from "@remotion/three";
import * as THREE from "three";

interface DataPod3DProps {
  scale?: number;
  rotationSpeed?: number;
  glowIntensity?: number;
  color?: string;
  animateIn?: boolean;
  startFrame?: number;
  state?: "idle" | "active" | "processing";
  cameraDistance?: number;
}

const DataPodMesh: React.FC<{
  frame: number;
  scale: number;
  rotationSpeed: number;
  color: string;
  state: "idle" | "active" | "processing";
  glowIntensity: number;
}> = ({ frame, scale, rotationSpeed, color, state, glowIntensity }) => {
  // Frame-based rotation (deterministic)
  const baseRotation = frame * rotationSpeed;

  const stateMultiplier =
    state === "processing" ? 2.5 :
    state === "active" ? 1.5 :
    1;

  const rotation = baseRotation * stateMultiplier;

  // Frame-based pulsing
  const pulseScale = state === "processing"
    ? 1 + Math.sin(frame * 0.1) * 0.1
    : 1;

  return (
    <group scale={scale * pulseScale}>
      {/* Core */}
      <mesh rotation={[rotation * 0.5, rotation, rotation * 0.3]}>
        <icosahedronGeometry args={[1, 1]} />
        <meshStandardMaterial
          color={color}
          emissive={color}
          emissiveIntensity={glowIntensity}
          metalness={0.8}
          roughness={0.2}
        />
      </mesh>

      {/* Glass shell */}
      <mesh rotation={[-rotation * 0.3, -rotation * 0.5, rotation * 0.2]}>
        <icosahedronGeometry args={[1.4, 0]} />
        <meshPhysicalMaterial
          color="#ffffff"
          metalness={0.1}
          roughness={0.05}
          transmission={0.95}
          thickness={0.5}
          ior={1.5}
          transparent
          opacity={0.6}
          side={THREE.DoubleSide}
        />
      </mesh>

      {/* Wireframe */}
      <mesh rotation={[rotation * 0.4, -rotation * 0.6, rotation]} scale={1.2}>
        <icosahedronGeometry args={[1, 1]} />
        <meshBasicMaterial
          color={color}
          wireframe
          transparent
          opacity={0.2}
        />
      </mesh>

      <pointLight
        position={[0, 0, 0]}
        intensity={glowIntensity * 2}
        color={color}
        distance={5}
        decay={2}
      />
    </group>
  );
};

export const DataPod3D: React.FC<DataPod3DProps> = ({
  scale = 1,
  rotationSpeed = 0.01,
  glowIntensity = 0.8,
  color = "#38BDF8",
  animateIn = true,
  startFrame = 0,
  state = "idle",
  cameraDistance = 5,
}) => {
  const frame = useCurrentFrame();
  const { fps, width, height } = useVideoConfig();

  // Spring-based entrance
  const entrance = animateIn
    ? spring({
        frame: frame - startFrame,
        fps,
        config: { damping: 100 },
      })
    : 1;

  const podScale = interpolate(entrance, [0, 1], [0, scale]);

  return (
    <div style={{ width: "100%", height: "100%", position: "relative", opacity: entrance }}>
      <ThreeCanvas
        width={width}
        height={height}
        camera={{ position: [0, 0, cameraDistance], fov: 45 }}
        style={{ backgroundColor: "transparent" }}
      >
        <ambientLight intensity={0.3} />
        <directionalLight position={[5, 5, 5]} intensity={0.6} />
        <directionalLight position={[-5, -5, -5]} intensity={0.3} />

        <DataPodMesh
          frame={frame - startFrame}
          scale={podScale}
          rotationSpeed={rotationSpeed}
          color={color}
          state={state}
          glowIntensity={glowIntensity}
        />
      </ThreeCanvas>
    </div>
  );
};
```

## 📋 Quick Reference: Do's and Don'ts

| ❌ Never Do | ✅ Always Do |
|-------------|--------------|
| `<Canvas>` from @react-three/fiber | `<ThreeCanvas>` from @remotion/three |
| `useFrame()` | `useCurrentFrame()` |
| `state.clock.getElapsedTime()` | `frame * constant` |
| `useState` for animation | Frame ranges for state |
| `OrbitControls` with user interaction | Fixed or frame-animated camera |
| `<Environment>` (usually) | Simple lights |
| Delta time (`delta`) | Frame number |
| `Math.random()` | `random("seed")` from Remotion |
| Interactive UI controls | Frame-based state transitions |
| Async animations | Synchronous frame calculations |

## 🎬 Testing Your 3D Components

To verify your component is deterministic:

1. Run `npm start`
2. Navigate to frame 100
3. Note the exact rotation/position
4. Navigate away and back to frame 100
5. **Should look identical** (not just similar)

If it changes, you're using non-deterministic animation.
