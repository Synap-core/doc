## Domain Architecture (production)

```
synap.live (or app.synap.live)  → synap-landing/ (Next.js, Vercel)
api.synap.live                  → synap-control-plane-api (Hono, Docker)
telegram.synap.live             → Telegram Mini App (Next.js, Vercel)
docs.synap.live                 → `synap-team-docs` (Fumadocs; repo `Synap-core/doc`)
{id}.synap.live                 → user data pods (Hetzner VMs)
```

## Launch Sequence (revised)

1. **Add 2 invite endpoints to CP API** — `preview` + `accept` (~2h)
2. **Wire invite email via Resend** — send link on workspace member add (~1h)
3. **Add `BROWSER_RELEASE_TOKEN` secret + publish v0.5.0** (~15min)
4. **Add `synap-landing/.env.example`** (~15min)
5. **Smoke test**: signup → provision pod → open Browser → invite teammate → accept via email link
6. **Confirm `FRONTEND_URL` env on CP API deploy** (~5min)
7. **Update landing copy + screenshots** (~1d)
8. Soft launch

**Estimated: ~1 day of engineering to unblock hard launch.**
