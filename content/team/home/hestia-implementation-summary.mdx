## Key Design Decisions Implemented

### 1. Agnostic Intelligence ✓

**Backend**: The `/intelligence/query` endpoint accepts any provider and returns OpenAI-compatible format:

```typescript
// Request
POST /api/hub/intelligence/query
{
  "hearth_node_id": "uuid",
  "messages": [{"role": "user", "content": "Hello"}],
  "model": "llama3.1:8b"
}

// Response (OpenAI-compatible)
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "choices": [{"message": {"role": "assistant", "content": "Hi!"}}]
}
```

**CLI**: Configuration supports any provider:

```yaml
intelligence:
  provider: ollama  # or openrouter, anthropic, openai, custom
  endpoint: http://localhost:11434
  model: llama3.1:8b
```

### 2. Pure Entity System ✓

**No new database tables.** All Hestia concepts are entities:

- `hearth_node` → Entity with profile `hearth_node`
- `intelligence_provider` → Entity with profile `intelligence_provider`
- `package_instance` → Entity with profile `package_instance`
- `hearth_deployment` → Entity with profile `hearth_deployment`

**Benefits**:
- Temporal queries work automatically
- Relations work via existing system
- Proposals/approvals work via existing governance
- Search/indexing work via existing infrastructure

### 3. Package-Based Architecture ✓

**Structure**:
```
hestia-cli/packages/
├── core/           # Required: CLI framework, types, config
├── install/        # Script installer (foundation)
├── usb/            # USB wrapper (calls install)
└── registry/       # Package discovery

plugins/
├── openclaw/       # Gateway commands
├── openclaude/     # Builder commands
└── control-plane/  # CP connector (optional)
```

**Package Interface**:
```typescript
interface Package {
  name: string;
  version: string;
  type: "core" | "gateway" | "builder" | "intelligence" | ...;
  source: {
    type: "docker_compose" | "binary" | "npm" | "git";
    url: string;
  };
  requires?: PackageDependency[];
  provides: string[];
}
```

### 4. Script-First, USB Wrapper ✓

**Script installer** (`packages/install/`) is the foundation:
- Phase 1: Foundation (Docker, UFW, SSH)
- Phase 2: Core + Gateway (Synap Backend + OpenClaw)
- Phase 3: Builder (OpenClaude)

**USB** (`packages/usb/`) just calls the installer:
```bash
# In USB late-command
./install.sh phase1 --mode=usb
```

## Usage Example

```bash
# 1. Install CLI
npm install -g @hestia/cli

# 2. Initialize hearth
hestia init
# → Interactive wizard
# → Creates ~/.hestia/config.yaml

# 3. Start hearth
hestia ignite
# → Starts core package (Synap Backend)
# → Starts intelligence package (Ollama)

# 4. Check status
hestia status
# → Shows all packages and health

# 5. Add gateway
hestia add gateway --source=openclaw
# → Installs OpenClaw
# → Configures Telegram/Discord

# 6. Add builder
hestia add builder --source=openclaude
# → Installs OpenClaude
# → Configures workspace

# 7. Deploy from channel
# User: "Build a website from my project"
# → OpenClaw → Synap Backend → OpenClaude
# → Deploys to /srv/www/
# → Returns URL
```
