## Flow 1: User signup and pod readiness

- Primary actor: end user
- Trigger: account creation
- Primary outcome: user lands in a ready workspace with required baseline entities/views
- Services involved: frontend/app surface, Control Plane, Data Pod, auth provider

### Step-by-step sequence

1. User submits signup from surface.
2. Auth provider validates identity and creates session.
3. Control Plane or provisioning layer determines tenancy path.
4. Data Pod initialization creates required workspace defaults.
5. Frontend confirms readiness and redirects user into workspace.

## Flow 2: AI request to validated mutation

- Primary actor: user + AI agent
- Trigger: AI action requiring data change
- Primary outcome: governed, reviewable, and traceable mutation
- Services involved: app surface, Intelligence Service, Data Pod event chain, proposal/governance layer

### Step-by-step sequence

1. User triggers AI action.
2. Intelligence Service prepares mutation request.
3. Data Pod permission check decides auto-approve vs proposal.
4. If approved, executor applies mutation and emits completion event.
5. UI updates state and shows action outcome/notification.

## Flow 3: External connector sync into knowledge graph

- Primary actor: external system + platform jobs
- Trigger: webhook, polling cycle, or scheduled import
- Primary outcome: external data normalized into entities/documents/relations
- Services involved: connector layer, ingestion pipeline, Data Pod repositories/jobs, search/indexing

### Step-by-step sequence

1. Connector receives or fetches source data.
2. Ingestion pipeline validates, maps, and deduplicates records.
3. Data Pod writes via governed path.
4. Indexing/search materialization updates read surfaces.
5. UI exposes imported content in relevant views.
