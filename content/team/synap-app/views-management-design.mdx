# Views Management Page - Design Brief

## 🎯 Overview

Create a beautiful, intuitive views management page where users can:
- Browse all their views (table, kanban, graph, etc.)
- Create new views
- Organize views (favorites, recent, folders/categories)
- Filter and search views
- Quick actions (duplicate, delete, share)

---

## 📋 User Flows

### Flow 1: Viewing All Views

```
User navigates to Views page
     ↓
Sees grid/list of all views
     ↓
Can filter by:
  - View type (table, kanban, graph, etc.)
  - Entity type (task, note, person, etc.)
  - Favorites only
  - Recent views
  - Search by name
```

### Flow 2: Creating a New View

```
User clicks "Create View" button
     ↓
Modal/Panel opens:
  - Select view type (table, kanban, etc.)
  - Select entity type(s) to show
  - Enter view name
  - (Optional) Set initial filters/query
     ↓
View created and opened
```

### Flow 3: Organizing Views

```
User can:
  - Star/Favorite views (toggle)
  - Create folders/categories (future)
  - Drag to reorder (future)
  - Group by entity type or view type
```

### Flow 4: Quick Actions

```
On each view card:
  - Click to open
  - Star/Favorite icon (toggle)
  - Menu with options:
    - Duplicate
    - Delete
    - Share (future)
    - Edit name/description
```

## 🎨 Design Requirements

### Layout Options

**Option A: Grid View (Card-based)**
- Cards with preview/thumbnail
- View name, type, entity type
- Last modified date
- Favorite indicator
- Quick actions menu

**Option B: List View (Table-like)**
- Compact rows
- Columns: Name, Type, Entity Type, Last Modified, Actions
- Sortable columns
- Checkbox selection for bulk actions

**Option C: Hybrid (Recommended)**
- Toggle between Grid and List
- Default: Grid (visual, modern)
- List for power users (dense information)

### Visual Elements

**View Cards Should Show:**
1. **Preview/Thumbnail**
   - Small screenshot or icon representation
   - Color-coded by view type
   - Shows entity count if available

2. **Metadata**
   - View name (editable inline?)
   - View type badge (Table, Kanban, Graph, etc.)
   - Entity type badge (Task, Note, etc.)
   - Last modified (relative time: "2 hours ago")

3. **Actions**
   - Favorite/Star button (always visible)
   - Overflow menu (3-dots) with:
     - Open
     - Duplicate
     - Delete
     - Share (future)
     - Edit

4. **Status Indicators**
   - Favorite (star icon, filled/hollow)
   - Pinned (pin icon, if implemented)
   - Shared (if shared with workspace)

### Filtering & Search

**Top Bar Should Include:**
1. **Search Bar**
   - Search by view name
   - Full-text search

2. **Filter Chips/Buttons**
   - View Type: All | Table | Kanban | Graph | List | Grid
   - Entity Type: All | Task | Note | Person | Event | File
   - Status: All | Favorites | Recent | Pinned
   - Sort: Recently Modified | Name A-Z | Name Z-A | Most Used

3. **View Toggle**
   - Grid view icon
   - List view icon

4. **Create Button**
   - Prominent "+ Create View" button
   - Dropdown with view type options:
     - Create Table View
     - Create Kanban View
     - Create Graph View
     - etc.

### Responsive Design

- **Desktop**: Grid with 3-4 columns, full sidebar filters
- **Tablet**: Grid with 2-3 columns, collapsible filters
- **Mobile**: Single column, bottom sheet filters

## 🎨 Design Style (Synap Branding)

### Colors & Theming
- Follow Synap's glassmorphism design language
- Use existing color tokens (`$background`, `$primary`, etc.)
- Support light/dark mode
- Subtle shadows and borders

### Typography
- View names: Medium weight, readable size
- Metadata: Smaller, muted color
- Badges: Compact, rounded

### Interactions
- Hover effects on cards (lift/shadow)
- Smooth transitions
- Loading states (skeleton cards)
- Empty states (illustrations + helpful text)

### Icons
- Use Lucide React icons (consistent with codebase)
- View type icons (Table, Columns, LayoutGrid, etc.)
- Entity type icons (from entity-views package)
