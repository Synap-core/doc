# Views

Views are named queries combined with rendering configurations. A view provides a specific lens over entity data — same data, different presentation.

## View types (16 defined, 12 implemented)

| Type | Status | Description |
|------|--------|-------------|
| Table | shipped | Traditional table view with columns, sorting, filtering. Default for entity lists. |
| List | shipped | Compact list of entities with key properties shown inline. |
| Grid | shipped | Grid of entity cards. Good for visual browsing. |
| Gallery | shipped | Visual gallery with larger thumbnails. Great for websites, articles. |

| Matrix | shipped | Two-dimensional grid view (rows × columns). |

## How views work

1. **Query** — Each view defines a filter query (entity type, tags, date range, status, etc.)
2. **Filter compilation** — `ViewFilterCompiler` translates high-level filters to SQL
3. **Data fetch** — tRPC or Hub Protocol executes query with workspace-scoped property resolution
4. **Cell rendering** — View renders as a cell in the cell-runtime registry
5. **Layout** — Bento grids compose multiple view cells with react-grid-layout

## Key files

- View renderer adapters: `synap-app/packages/core/view-renderer/`
- View types: `synap-app/packages/views/[type]/` (table, kanban, bento, calendar, etc.)
- Cell system: `@synap/capabilities` → `@synap/cell-runtime`
- Filter compiler: `synap-backend` ViewFilterCompiler
- Bento grid: `react-grid-layout` 12-col system
