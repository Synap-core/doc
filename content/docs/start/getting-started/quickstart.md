# Quickstart

**Goal:** from “I have (or will have) a pod” → **one clear next action**.

If you do not have a pod yet, start with **[Installation](./installation)** — there you choose **hosted (Synap Cloud)**, **Docker**, **CLI**, or **from source**.

---

## 1. Use Synap in the app (most people)

**[synap.live](https://synap.live)** — create an account, open your workspace, and use the **browser** or **desktop** experience. Download and updates are offered from the same landing flow when you sign up.

No API keys required to explore the product surface (entities, views, channels).

---

## 2. Connect AI agents & automations

**[AI agents & external tools](./ai-agents-and-tools)** — OpenClaw, `npx @synap/cli init`, Hub Protocol (`/api/hub/*`), and skills.

Use this path when you want **any AI agent or script** to read/write your workspace **through governed APIs**, not through the UI only.

---

## 3. Build or integrate (SDK / API)

- **[First queries with the SDK](/docs/integrate/development/sdk/first-query)** — React, TypeScript, or cURL against `/trpc`  
- **[Integrations overview](/docs/integrate/integrations/overview)** — webhooks, chat streams, API keys  
- **[Use case guides](/docs/start/guides)** — e.g. [Second brain](/docs/start/guides/by-use-case/second-brain), [Team collaboration](/docs/start/guides/by-use-case/team-collaboration)  

---

## 4. Go deeper on the system

- **[What is Synap?](/docs/start/concepts/what-is-synap)** — product framing  
- **[System concepts](/docs/architecture/concepts)** — architecture, events, graph  
- **[Next steps](./next-steps)** — structured checklist after install  

---

**Not sure?** Use **[synap.live](https://synap.live)** first, then come back to **AI agents & external tools** when you want automation.
