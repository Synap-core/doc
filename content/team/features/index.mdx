# Features & Flows

**Target Users**: Product managers, engineers, designers, QA — anyone validating or building cross-cutting features  
**Marketing**: Feature documentation is the product — every feature maps end-to-end across surfaces, architecture, and source code  
**Key User Stories**:
- As a PM, I want to trace each feature from trigger to outcome across all product surfaces so I can validate completeness
- As an engineer, I want code references linked to every feature so I can find the implementation quickly
- As a designer, I want surface coverage mapped so I know which products support each feature
**Key Features**: Data ingest, data organization, communication, intelligence, infrastructure — 20 features across 5 categories  
**Current Status**: All 20 feature docs active under 5 categories; last updated March 2026

## Overview

This section maps **every Synap feature end-to-end** across all product surfaces. For each feature, the documentation covers:

- **What it does** — One-sentence capability summary
- **Surface coverage** — Which products support this feature (Browser, Relay, Studio, Raycast, Platform)
- **How it works** — End-to-end flow from trigger to outcome
- **Architecture** — Key services and contracts involved
- **Code links** — Relevant repos and source files
- **Related features** — Cross-linked documentation

## Architecture

Features are organized into **five categories** that follow the Synap triptych (Entities → Views → Channels):

```
Data Ingest      →  Data Organization     →  Intelligence
   │                    │                        │
   ▼                    ▼                        ▼
Capture, Import,     Entities, Profiles,       Proactive AI,
Search               Views, Feeds              Workflows, Orchestrator

                              Communication
                              │
                              ▼
                              Chat, Channels, Notifications
                              
                              Infrastructure
                              │
                              ▼
                              Pod Sync, Proposals, Onboarding
```

## How to read this section

Each feature page links to deep-dive documentation in:

- **Product spaces** (`/team/browser/`, `/team/relay/`, etc.) — surface-specific documentation
- **Platform space** (`/team/platform/`) — backend architecture and contracts
- **Intelligence space** (`/team/intelligence/`) — AI agent system
- **Technologies** (`/team/technologies/`) — infrastructure and tooling
- **Tech Matrix** (`/team/tech-matrix/`) — repository code anchors and infrastructure

This section is designed for **product validation**: for each feature, you can jump to the right level of detail — overview → surface coverage → architecture → source code.
