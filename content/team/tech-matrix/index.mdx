# Tech Matrix

The **Tech Matrix** is the navigation layer between Synap's product documentation and our actual codebase. It exists because architecture docs explain *what* and *why*, but engineers need to know *where* the code lives.

## Why it exists

- **Feature docs ↔ source code gap** — `/docs` and `/team` pages describe systems, but finding the actual implementation requires memorising repo names and directory structures. The tech matrix closes this gap.
- **Onboarding** — new engineers can trace from high-level product intent → architecture docs → source files → running services in a single consistent path.
- **Multi-repo complexity** — 8 repos, 60+ packages, 200+ services. The matrix imposes a stable index on that complexity.

## How to use it

The tech matrix is structured as a set of pages that link in three directions:

1. **Product docs** — `/docs` (public) and `/team` (internal) describe features and architecture. From any page, follow links to the matching code-anchors section for the repo URL and key files.
2. **Code anchors** — `/team/tech-matrix/code-anchors` lists every major product area with its repo, key files, and architecture patterns. Each file reference can serve as a clickable anchor.
3. **Infrastructure** — Docker, CLI, and agent setup pages (`openclaw-docker-configuration`, `ollama-docker-setup`, `claude-code-setup`) cover the operational side of running and building Synap.
