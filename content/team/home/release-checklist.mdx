## 🔴 You Need to Do

### 1. GitHub repo — done 
`Synap-core/browser-exec` created. No further action needed on the repo itself.

### 2. Add GitHub Actions secrets (15 min)
- Create `https://github.com/synap-live/synap-browser` (private or public)
- This is where electron-builder uploads binaries and auto-updater checks for updates
- Set it to match `electron-builder.yml`: `owner: synap-live`, `repo: synap-browser`

Go to your **private source repo** → Settings → Secrets → Actions → New repository secret:

| Secret name | Value | Required? |
|-------------|-------|-----------|
| `BROWSER_RELEASE_TOKEN` | PAT with `repo` scope on `Synap-core/browser-exec` | **Yes** — without this nothing publishes |
| `APPLE_ID` | your@apple.com | For signed macOS |
| `APPLE_APP_SPECIFIC_PASSWORD` | from appleid.apple.com | For signed macOS |
| `APPLE_TEAM_ID` | 10-char team ID from developer.apple.com | For signed macOS |
| `CSC_LINK` | `base64 -i "Developer ID.p12" \| pbcopy` | For signed macOS |
| `CSC_KEY_PASSWORD` | .p12 password | For signed macOS |
| `WIN_CSC_LINK` | base64-encoded Windows .p12 | Optional |
| `WIN_CSC_KEY_PASSWORD` | Windows cert password | Optional |

The workflow degrades gracefully: if signing secrets are absent it builds unsigned (works for internal testing).

### 3. Apple Developer account — code signing (1–2 hours)
Required for macOS users to open the app without Gatekeeper warning.

```bash
# What you need from developer.apple.com:
APPLE_ID="your@apple.com"
APPLE_APP_SPECIFIC_PASSWORD="xxxx-xxxx-xxxx-xxxx"   # appleid.apple.com → App-Specific Passwords
APPLE_TEAM_ID="XXXXXXXXXX"                           # developer.apple.com → Membership
CSC_LINK="/path/to/Developer-ID-Application.p12"    # downloaded from Keychain Access
CSC_KEY_PASSWORD="your-cert-password"

# Then:
cd browser && APPLE_ID=... APPLE_APP_SPECIFIC_PASSWORD=... APPLE_TEAM_ID=... pnpm dist:mac
```

Without signing: users see "App can't be opened" Gatekeeper dialog. They can bypass with right-click → Open, but it's a bad first impression.

### 3. Windows code signing certificate (optional for first release)
- Purchase from DigiCert, Sectigo, or similar (~$300/year)
- Without it: Windows Defender SmartScreen shows "Unknown publisher" warning
- Can ship unsigned for beta; add later

### 4. Generate a GitHub Personal Access Token for publishing
```bash
# Scope needed: repo (full)
# Set as env var when running dist:
GH_TOKEN=ghp_xxxx cd browser && pnpm dist:mac
# electron-builder will auto-create the GitHub Release and upload assets
```

### 5. Deploy the CP API download routes
The new `/install` and `/download/latest` routes are in:
`synap-control-plane-api/src/routes/download.ts`

Deploy your CP API and they'll be live at `https://api.synap.live/install`.

### 6. Add a `GITHUB_TOKEN` env var to CP API (optional, prevents rate limiting)
The `/download/latest` endpoint calls GitHub API to find the right asset.
Public repos get 60 req/hour unauthenticated. For production:
```env
# In synap-control-plane-api/.env
GITHUB_TOKEN=ghp_xxxx   # read-only, public_repo scope is enough
```
Then update `download.ts` to pass it in the fetch headers.

```bash

```

npm version minor                  # 0.1.0 → 0.2.0
