## 1. Architecture (Confirmed Correct)

```
┌─────────────────────┐
│   Synap App         │
│   (Mobile / Web)    │
├──────────┬──────────┤
│  tRPC    │ Socket.IO│
└────┬─────┴─────┬────┘
     │           │
     ▼           ▼
┌─────────┐  ┌───────────┐
│ Backend │  │ Realtime  │
│ :4000   │  │ :4001     │
│         │  │           │
│ Auth    │  │ Socket.IO │
│ DB      │  │ Broadcast │
│ Events  │  │           │
└────┬────┘  └─────▲─────┘
     │             │
     │  /bridge/emit (fire-and-forget)
     │             │
     ├─────────────┘
     │
     │  POST /api/chat/stream (SSE)
     │  GET /api/hub/* (hub-protocol-rest)
     ▼
┌──────────────────────┐
│  Intelligence Hub    │
│  :3001               │
│                      │
│  AI Agents           │
│  Tools               │
│  LLM Calls           │
│  NO DATABASE         │
│                      │
│  Calls BACK to       │
│  backend via         │
│  hub-protocol-rest   │
└──────────────────────┘
```

**Key insight:** The app NEVER calls Intelligence Hub directly. Backend is the single gateway.

## 8. Bug Fixes Applied

### Fix 1: Streaming Not Wired (FIXED)

**File:** `packages/features/ai-chat/src/components/ChatView.tsx`

**Problem:** Component read from `useStreamingStore` but never called `useStreamingMessage` to subscribe to events.

**Solution Applied:**

```typescript
// Added import
import { useStreamingMessage } from "../hooks/useStreamingMessage";

// Added hook call before store read
useStreamingMessage(threadId || "");

// Existing store read now receives data
const streamingMessage = useStreamingStore((s) =>
  threadId ? s.getStream(threadId) : undefined
);
```
