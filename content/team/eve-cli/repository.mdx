```

├── packages/
│   ├── eve-cli/              # @eve/cli — eve entrypoint
│   ├── eve-dna/              # @eve/dna — schemas, secrets, profiles

│   ├── eve-eyes/             # @eve/eyes — RSSHub, database UI
│   ├── eve-legs/             # @eve/legs — Traefik, tunnels
│   ├── eve-cli-kit/          # @eve/cli-kit — shared flags, JSON output
│   ├── eve-usb/              # @eve/usb — USB / Ventoy helpers
│   └── eve-install/          # @eve/install — server install assets

├── commands.manifest.yaml    # CLI command inventory
├── docs/                     # Repo-level docs
└── .github/                  # CI workflows
```

## Developer workflow

```bash
git clone https://github.com/Synap-core/hestia-cli.git
cd hestia-cli
pnpm install
pnpm -r run build
pnpm --filter @eve/cli exec eve --help
```

Pre-push hooks (if enabled) and CI should run **`pnpm -r run build`** and manifest checks — see repo **`.githooks`** and **`.github/workflows`**.

### Post-pull sync (important)

After a `git pull`, always rehydrate workspace state before launching Eve:

```bash
cd /opt/eve
pnpm install
pnpm --filter @eve/cli... run build
pnpm --filter @eve/cli exec eve --help
```

This avoids stale workspace/bin state and guarantees the `@eve/cli` binary is resolvable.

## Operator install (server)

**`bootstrap.sh`** (raw on `main`) installs OS deps, Docker, Node, pnpm, clones **hestia-cli**, builds, and runs **`eve setup`** (unless `--no-setup`). See **`README.md`** for non-interactive examples and **canonical URLs**.

## Prebuilt release bundle (no local build)

For tagged releases, CI publishes `eve-cli-bundle-<version>.tar.gz` assets in GitHub Releases. This avoids compiling TypeScript on the target host.

```bash
VERSION=v0.1.0
curl -fsSL -o "eve-cli-bundle-${VERSION}.tar.gz" \
  "https://github.com/Synap-core/hestia-cli/releases/download/${VERSION}/eve-cli-bundle-${VERSION}.tar.gz"
mkdir -p /opt/eve && tar -xzf "eve-cli-bundle-${VERSION}.tar.gz" -C /opt/eve --strip-components=1
cd /opt/eve
node dist/index.js --help
node dist/index.js setup
```
