# Proactive Intelligence

AI doesn't just react — it posts proactively into the user's personal channel. Five cron workers generate insights, briefings, and health checks on schedule.

## Cron workers

| Worker | Frequency | Description |
|--------|-----------|-------------|
| **Morning Briefing** | ~15 min after midnight | "Good morning" summary of overnight events, upcoming tasks, recent activity |
| **Weekly Digest** | Hourly (accumulates) | Compiles weekly trends, project milestones, and AI-generated recommendations |
| **Health Check** | Daily | System health, sync status, pod connections, connector statuses |
| **Insight Generation** | On-demand | AI analyzes entity relationships, surfaces connections, suggests improvements |
| **Nudge Delivery** | Per schedule | User-configured reminders and nudges based on proactive preferences |

## User preferences

Proactive AI behavior is controlled via `workspace.settings.proactiveAi` (JSONB):

```jsonc
{
  "enabled": true,
  "nudgeDensity": "balanced", // "low" | "balanced" | "high"
  "schedules": {
    "morningBriefing": true,
    "weeklyDigest": true,
    "healthCheck": true,
    "insights": true
  }
}
```

## Rate limits

- Hub Protocol `POST /proactive/post`: max 3/hour, 10/day per agent
- Prevents AI spam while maintaining meaningful proactive interaction
