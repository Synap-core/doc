## Role in Synap / Eve

- **Synap Data Pod** — `docker compose` under **`backend/deploy/`** (services: api, realtime, postgres, caddy, …).
- **Eve** — additional compose stacks and **`eve-network`** for sidecars (Ollama, Traefik gateway, etc.).

---

## Nested Docker (LXC / Proxmox CT)

Running Docker **inside** an unprivileged CT often requires **nesting** and sometimes **privileged** mode. Typical symptom:

`error during container init: open sysctl net.ipv4.ip_unprivileged_port_start … permission denied`

That is **not** fixed by Synap `.env` — it is **CT/host configuration**. Enable **nesting** (e.g. Proxmox: CT options → Features → Nesting), consult current Proxmox + Docker docs.

## Compose project names

Synap deploy may set **`COMPOSE_PROJECT_NAME`** (e.g. `synap-backend`) — container names in logs depend on it. Always **`cd` to the correct `deploy/`** before `docker compose` commands.
