# Eve (Entity Creation System)

Eve is Synap's **sovereign installer CLI** — a **TypeScript command-line tool** (`eve`) and monorepo (**hestia-cli**) that turns a Bare server into a coherent stack: **Brain** (Synap Data Pod + optional local inference), **Legs** (exposure: Traefik, tunnels), **Arms** (OpenClaw), **Eyes** (RSSHub), and **Builder** (OpenCode / deploy hooks).

Eve is **not a replacement** for the Synap backend. It **orchestrates** installs and Docker sidecars alongside the official **`synap`** shell CLI.

---

## Target Users

- **Eve / hestia-cli contributors** — package authors, CLI UX designers, Docker compose maintainers
- **Operators & infrastructure engineers** — bootstrap flows, LXC/Proxmox deployment, path management, `synap` delegation
- **GTM & operations** — self-host positioning, "sovereign stack" messaging, launch narratives

---

## Marketing

- **Positioning:** The sovereign installer for Synap and sidecars — one system, one mental model.
- **North star metaphor:** A server becomes a **coherent digital entity** with specialized organs (Brain, Arms, Eyes, Legs, Builder), each owning a domain of capability.
- **Key differentiator:** Sovereignty — your machine, your data, optional fully local inference, no mandatory cloud for the core stack.
- **Messaging constraint:** Eve is complementary to Synap, not competitive; the Data Pod is the **Brain**.

---

## Key User Stories

- As an **operator**, I want to run `eve setup` once and have a complete stack deployed (inference only, Data Pod only, or both).
- As a **self-hosted user**, I want full control over my machine, data residency, and whether cloud models are used.
- As a **developer**, I want a pnpm monorepo with clear package boundaries and predictable port mental models.
- As a **product user**, I want `eve brain`, `eve legs`, `eve arms` — CLI commands that map directly to the organ metaphor.
- As a **CI operator**, I want scripted non-interactive patterns (`--yes`, `--profile`, `--domain`) for automated deployments.

## Overview

Eve turns a **bare server** into a **coherent digital entity** with five organs:

| Organ | Role | What it stands up |
|-------|------|-------------------|
| **Brain** | Memory, reasoning, API | Synap Data Pod (PostgreSQL, Redis, Caddy, Hub) |
| **Arms** | Actions, tools, MCP | OpenClaw |
| **Builder** | Create / deploy / codegen | OpenCode, OpenClaude, deploy hooks |
| **Eyes** | Intake, feeds | RSSHub |
| **Legs** | Exposure, routing, reachability | Traefik, Pangolin / Cloudflare tunnels |

CLI maps organs to commands: `eve brain`, `eve arms`, `eve builder`, `eve eyes`, `eve legs`.

**North star principles:**
1. **Sovereignty** — Operator controls the machine and data residency.
2. **Organic composition** — Shared network, secrets patterns, predictable ports.
3. **Synap-centric** — Hub, proposals, and pod contracts remain authoritative.
4. **Explicit exposure** — Synap public URL vs Eve Legs are separate user choices.
5. **Recoverable setup** — `.eve/` is inspectable and reproducible.

## Architecture

### Organ Topology

```
Internet
  │
  ├─ Caddy :80/443 ───→ Synap Data Pod (Caddy, API :4000)
  │
  └─ Traefik / Tunnel ─→ Eve sidecars (eve-network)
         ├─ Ollama (inference gateway 127.0.0.1:11435)
         ├─ RSSHub
         └─ OpenClaw (Arms)
```

### Synap vs Eve Boundary

| Layer | Owner | Notes |
|-------|-------|-------|
| HTTPS for Pod API/auth | **Synap** (Caddy) | Public pod domain in `eve setup` config |
| Inference gateway / Traefik for sidecars | **Eve** (Legs) | Does not replace Caddy on 80/443 |
| Hub Protocol, proposals, workspace | **Synap backend** | Eve may sync non-secret provider routing via `eve ai sync` |
| Local tool preferences (OpenRouter, etc.) | **Eve secrets** | `.eve/secrets/secrets.json` |

### Monorepo Packages (hestia-cli)

| Package | Responsibility |
|---------|----------------|
| `@eve/cli` | `eve` entrypoint, Commander + Clack |
| `@eve/dna` | Setup profile schema, secrets, USB manifest |
| `@eve/brain` | `runBrainInit`, Synap delegation, Ollama/Postgres/Redis helpers |
| `@eve/legs` | Traefik, gateway, tunnel helpers |
| `@eve/arms` | OpenClaw |
| `@eve/eyes` | RSSHub, database UI runners |
| `@eve/builder` | OpenCode / OpenClaude / deploy hooks |
| `@eve/cli-kit` | Shared flags, JSON output |
