Use this space as a routing page, not the full package catalog.

## Canonical package map

Use **[Synap Packages](/team/synap-packages)** for:

- package inventory
- ownership and boundaries
- repo/path references
- migration and integration notes

## What stays in `synap-app` docs

- surface-level status updates
- app-specific runtime notes
- frontend integration deep dives (for example intelligence streaming)

## What should not be duplicated here

- full per-package documentation already maintained in `synap-packages`
- platform architecture pages that belong to **[Platform](/team/platform)**
