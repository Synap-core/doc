# Debug: "Lost" updates (header, doc collaboration, workspace state)

## 1. Webpack cache warning – **not** the cause

The warning:

```text
[webpack.cache.PackFileCacheStrategy] Caching failed for pack: Error: Unable to snapshot resolve dependencies
```

means webpack **failed to write** the pack cache (e.g. due to pnpm symlinks or filesystem).  
When that happens, webpack **does not use** that cache; it rebuilds. So this warning does **not** make the app serve old code. It can only make builds a bit slower.

## 3. Likely causes of “everything looks different”

1. **Stale build (most likely)**  
   Next.js is serving an old build from `apps/web/.next`. The source on disk is new, but the running app is from an older build.

2. **Runtime state**  
   Panel layout and active app are persisted (e.g. localStorage). If that was cleared or the key changed, the UI would look “reset” (e.g. no panels, default app).

3. **Code never committed**  
   If “header”, “doc collaboration”, or other UI lived only in uncommitted edits or in a branch that wasn’t merged, git wouldn’t have them. Your last commit is `659e13e feat: doc-entity + fix issues at build`; you have 4 modified files and many stashes.
