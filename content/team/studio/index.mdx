# Studio

**Status:** Live on webpack, typecheck clean, real backend wired.
**Location:** `synap-app/apps/studio/`
**Port:** 3002
**Stack:** Next.js 15 (App Router) · React 19 · HeroUI v2 · Tailwind v3 · next-themes · tRPC · Socket.IO

---

## Purpose

Studio is the single focused web app. It replaces the earlier multi-app approach (data / views / documents / channels as separate surfaces) with one coherent shell built on HeroUI, organized around three core concepts:

- **Channels** — every agent + every object has a canonical conversation.
- **Documents** — rich markdown editing with AI proposal diffs.
- **Agents** — your AI team, summoned via @mention or called directly.

Plus the supporting surfaces: Tasks, Inbox, Graph, Entity detail, Home.

The aesthetic target is the "ChatGPT 2028" mockup — premium, calm, modern. No shadows, semantic tokens everywhere, Fraunces headings, generous spacing.

## Single-space UX

Unlike Browser, Studio **does not expose multi-workspace to the user**. Every Studio session is one implicit space — no workspace switcher, no "which workspace am I in", no per-workspace settings panel. The tier implication is:

- **Free tier** — shared pod, one workspace (multi-tenant pod).
- **Pro tier** — own dedicated pod, one workspace.

Upgrading doesn't mean "add a workspace." It means "get your own pod." Multi-workspace needs are eventually satisfied by users generating their own custom apps (future).

This shapes everything: onboarding skips workspace-naming, the sidebar has no workspace switcher, and all settings are pod-wide.
