# Eve OS Home — Design Spec

**Date:** 2026-05-05
**Status:** Design specification (v2 — UmbrelOS aesthetic). Implementation pending Phase 2 of [Eve OS Vision](./eve-os-vision.mdx). Replaces the v1 sidebar/tile design entirely.

This document is concrete enough that a developer can implement the Home app end-to-end after one read. It describes only what renders **inside the pane** provided by the shell. Wallpaper, dock, and pane chrome are documented in [Eve OS Shell](./eve-os-shell.mdx) — read that first.

## 1. Design intent

The Home is what the operator sees the moment they open Eve. It must communicate, in under one second:

> "This is **my** OS. These are **my** apps. Everything I need is here."

Three feelings to deliver, in order of priority:

1. **Premium calm.** The pane is already frosted glass over a slow gradient — the Home leans into that. Generous whitespace, light typography, no busywork.
2. **Sovereignty.** The operator's apps are the heroes. Eve branding is reduced to a single small mark beside the greeting. The colorful app grid is the surface that does the talking.
3. **Glanceable intelligence.** Three stat cards above the grid surface live system facts (agents running, events today, updates available). They hint at what the OS is doing without becoming a dashboard.

What it explicitly is NOT:

- Not a SaaS dashboard with KPIs and charts. Three glanceable cards is the ceiling.
- Not a workflow editor (that's the Agents app).
- Not a chat UI (that's OpenWebUI / a future native Synap chat).
- Not a marketplace browser (the `+ Add` icon launches that as a separate app).
- Not a settings panel (gear icon top-right of the pane header → `/settings`).

## 2. Pane composition

The Home renders inside the standard shell pane (see [Eve OS Shell §4](./eve-os-shell.mdx#4-layer-2--app-pane-the-rounded-popup)). The shell provides the rounded popup, the frosted background, and a 56px pane header. The Home owns only the body.

### 2.1 Pane header (provided by shell)

| Slot | Content |
|---|---|
| `back` | none — Home is the root |
| `title` | empty — the greeting carries identity, the title bar stays wordless |
| `actions` | a single `Settings` (Lucide gear) icon, navigates to `/settings` |

The wordless header is intentional. The greeting block immediately below makes "this is your OS" the first piece of information the eye lands on.

### 2.2 Pane body — four vertical zones

```
┌───────────────────────────────────────────────────────────────┐  ← shell pane, 56px header
│                                          [⚙]                  │     (gear → /settings)
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  [optional CpAuthBanner — only when token missing/expired]    │
│                                                               │
│                  ✦  Good evening, Antoine                     │  ← Zone A: Greeting (auto)
│                       Tuesday · May 5                         │
│                                                               │
│   ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │  ← Zone B: Stat cards
│   │ AGENTS       │ │ TODAY        │ │ UPDATES      │         │     (auto-height)
│   │ 3 running    │ │ 127 events   │ │ All up to date│        │
│   └──────────────┘ └──────────────┘ └──────────────┘         │
│                                                               │
│   ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐             │
│   │    │  │    │  │    │  │    │  │    │  │    │             │
│   │ ai │  │ S  │  │ A  │  │ C  │  │ D  │  │ DA │   …          │  ← Zone C: App grid
│   └────┘  └────┘  └────┘  └────┘  └────┘  └────┘             │     (flex-1, scrolls)
│   Chat   Synap  Arch    CRM   DevPlane  DevAgent              │
│                                                               │
```

The four zones, from top to bottom: an optional auth banner, a greeting block, a stat-cards strip, the app grid, and a search input. The grid takes whatever vertical space the others don't claim.

## 3. Zone A — Greeting block

Auto-height. Top padding 48px from the pane body's top edge, bottom padding 32px to the stat cards. Centered horizontally.

```
                ✦  Good evening, Antoine
                      Tuesday · May 5
```

| Element | Treatment |
|---|---|
| Brand glyph | Small mark to the left of (or above on mobile) the greeting line. Default: Lucide `Sparkles` at 24px, color `brand.agents` (emerald `#34D399`). When a logo SVG is dropped in, replaces the Lucide icon with the same bounding box. |
| Greeting line | "Good {morning|afternoon|evening}, {firstName}" — `~32px`, `font-weight: 300` (light), letter-spacing `-0.01em`. Filled with the AI-to-violet gradient as a `background-clip: text` effect: `linear-gradient(135deg, #34D399 0%, #A78BFA 100%)`. The gradient is subtle — the text reads as a soft luminous tone, not a rainbow. |
| Date subtitle | One line below, `13px`, `neutral-400`, `font-weight: 400`. Format "Weekday · Month D" (e.g. "Tuesday · May 5"). |

### Time-of-day logic

- 05:00–11:59 → "Good morning"
- 12:00–17:59 → "Good afternoon"
- 18:00–04:59 → "Good evening"

`firstName` resolves from the CP user profile. If unavailable (no `cp.userToken`), fall back to "Welcome back" with no name.

### Mobile

Below 640px width, the brand glyph stacks above the greeting line and the whole block centers. Greeting size shrinks to `26px`. Date subtitle unchanged.

## 4. Zone B — Stat cards strip

Auto-height. Three translucent cards in a horizontal row, equal width, `12px` gap. Total horizontal padding 24px from the pane body edges. Below the greeting, above the app grid, with `24px` margin to each.

### 4.1 Card visual

```
┌──────────────────────┐
│  AGENTS              │  ← 12px label, neutral-400, tracking +0.05em, uppercase
│                      │
│  3 running           │  ← 24px value, font-weight 300
│                      │
│  Hermes idle         │  ← 12px sublabel, neutral-500 (optional)
└──────────────────────┘
        110px tall
```

```css
height: 110px;
padding: 16px;
border-radius: 16px;
background: rgba(255, 255, 255, 0.08);
backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.08);
transition: background 200ms ease, box-shadow 200ms ease;
cursor: pointer;
```

On hover, the card gains a soft inner glow ring matched to its accent color:

```css
:hover {
  background: rgba(255, 255, 255, 0.10);
  box-shadow: inset 0 0 0 1px var(--card-accent),
              0 0 24px -8px var(--card-accent);
}
```

`--card-accent` is supplied per card (emerald for Agents, indigo for Today, amber for Updates) at 35% opacity. There is no `translateY` on hover — only background/glow.

Light mode flips the surface to `rgba(255, 255, 255, 0.55)` over a `rgba(0,0,0,0.04)` border, glow opacity halves.

### 4.2 The three default cards

| Card | Label | Value source | Sublabel | Click target |
|---|---|---|---|---|
| **Agents** | "AGENTS" | `useStats().agentsRunning` (count of agents emitting heartbeats in last 60s) | one of: "Hermes idle" / "OpenClaw routing" / "All quiet" | `/agents` |
| **Today** | "TODAY" | `useStats().eventsToday` (count of `events.since(startOfDay)` from Hub Protocol) | "across all channels" or "in your personal channel" | `/agents?view=timeline` |
| **Updates** | "UPDATES" | `useStats().updatesAvailable` (count of marketplace apps with newer version than installed) | "All up to date" when 0, otherwise "Tap to review" | marketplace (Phase 4: `/marketplace`; Phase 2: external CP URL) |

Cards are real `<a>` elements with `href` for keyboard nav. Sublabel is omitted (not just empty) when no useful value is available — preserves the visual rhythm.

### 4.3 Loading & empty

While stats resolve, render skeleton cards (same surface, value replaced by a 1.6s shimmer pulse on a 60% opacity bar). When all three values are zero/uneventful, cards still render with truthful values — Eve never hides the strip on quiet days, calm is the point.

### 4.4 Configurability (deferred)

Cards are hard-coded in v1. v2 may let the operator pick three from a wider set (storage, network, sync status…). Not in Phase 2 scope.

## 5. Zone C — App grid

The visual hero. Flex-1, takes whatever remaining vertical space the body has. Internally scrollable (the pane body itself does not scroll past Zone D).

### 5.1 Icons, not tiles

This is the largest deviation from the v1 doc. Apps render as **vivid colorful icons** in the UmbrelOS style, not gray tiles:

```
       ┌──────┐
       │      │  ← 80×80px colored rounded square
       │  ✦   │     (brand-color background, Lucide glyph centered in white/95)
       │      │
       └──────┘
        Synap     ← 14px, neutral-200, font-weight 500
        running   ← optional 12px, neutral-500
```

- Icon block: `80×80px` desktop, `64×64px` mobile, `border-radius: 18px`
- Background: brand color or gradient from `brand-colors.ts` registry (per-app)
- Glyph: Lucide icon at `40px` desktop / `32px` mobile, color `rgba(255,255,255,0.95)`, centered. For marketplace apps with `iconUrl`, the remote SVG/PNG renders in the same bounding box (still on the brand-color background — the registry supplies the color, even when the glyph is remote).
- Label below: app name, `14px`, `neutral-200`, `font-weight: 500`, single line, ellipsis if it overflows
- Optional second line: status or context — `12px`, `neutral-500`. Examples: `running` / `offline` / `degraded` / `12 unread`. Rendered only when the value is meaningful (omit, don't render an empty line).

### 5.2 Grid layout

```css
display: grid;
grid-template-columns: repeat(auto-fill, minmax(112px, 1fr));   /* desktop */
gap: 24px;
padding: 8px 24px 32px;       /* the 32px bottom keeps clear of Zone D */
align-items: start;
```

Mobile (<640px): `minmax(96px, 1fr)`, gap `16px`. The auto-fill keeps the grid responsive at any pane width. Typical desktop pane fits 6–8 columns; mobile fits 3–4.

The grid does not enforce a row cap — it scrolls vertically when the user has many apps. The scrollbar is the pane body's, hidden until the user scrolls (standard `overflow-y: auto` on the pane body).

### 5.3 Per-app brand colors

Each app's icon background is sourced from a brand-color registry — see §11 for the schema. Examples:

| App | Background | Glyph | Accent (dock indicator + hover ring) |
|---|---|---|---|
| OpenWebUI ("Chat") | `linear-gradient(135deg, #10B981, #34D399)` | `MessageSquare` | `#10B981` |
| Synap | `#34D399` (solid emerald) | `Brain` (or remote SVG) | `#34D399` |
| The Arch | `#5B21B6` (deep violet) | remote `iconUrl` | `#7C3AED` |
| CRM | `#E11D48` (rose) | `Users` | `#F43F5E` |
| DevPlane | `#F59E0B` (amber) | `LayoutGrid` | `#FBBF24` |
| Dev Agent | `linear-gradient(135deg, #4F46E5, #7C3AED)` | `Code2` | `#6366F1` |

The registry is opinionated — ten or so curated palettes mapped to known apps. Marketplace apps without a registry entry fall back to a deterministic palette derived from a hash of their slug (hue-rotated emerald), so new apps still feel intentional on first install.

### 5.4 Interaction states

| State | Visual | Timing |
|---|---|---|
| Default | Vivid icon, label `neutral-200` | — |
| Hover | `transform: scale(1.04)` + outer glow ring (`box-shadow: 0 0 24px -4px <accent-35%>`) + label brightens to `neutral-100` | 200ms ease-out |
| Pressed | `transform: scale(0.96)` | 80ms ease-in |
| Active (this app's route is current) | Identical to default — the dock owns the active indicator. The grid does not double-mark. | — |
| Loading (mid-install pulse) | Subtle `opacity` pulse on the icon, 1.6s | — |

**Explicitly forbidden:** `translateY` lift hover. The v1 doc had it; v2 does not. Hover is scale + glow only. The lift made tiles feel like cards — we want them to feel like icons, which press into the surface, not float above it.

### 5.5 Accessibility & semantics

Each icon is an `<a href={app.url}>` with `aria-label="Open {name}, status {status}"`. Tab cycles icons in DOM order (which matches pin order). Enter or Space activates. `prefers-reduced-motion` disables the scale + glow on hover; the icon brightens its label instead.

### 5.6 The `+ Add` icon

Always last in the grid. Same `80×80` bounding box. Dashed `1.5px` border in `neutral-500`, transparent fill, centered Lucide `Plus` glyph in `neutral-400`. Hover: border brightens to `neutral-300`, glyph to `neutral-100`. Click: opens marketplace (Phase 2: external CP URL in new tab; Phase 4: routes to in-OS `/marketplace`).

### 5.7 Click behavior per app source

Identical to v1 logic — only the visual changed:

- **External URL** (CP marketplace `app_type: 'url'`): `window.open(app.url, '_blank')`.
- **Eve component** (`app_type: 'eve_component'`): opens `webUrl` in new tab. Status dot reflects local component health.
- **Workspace pack** (`app_type: 'workspace_pack'`): does not appear on the OS Home — packs install into Synap.
- **Bundle** (`app_type: 'bundle'`): unwraps to constituent app icons (the bundle itself doesn't render).

### 5.8 Right-click / long-press

Contextual menu: `Pin`, `Unpin`, `Open in new window`, `Uninstall` (marketplace only), `Settings` (if app exposes one). Menu styled as a frosted floating popover matching the pane aesthetic — never a native browser menu.

## 6. Zone D — Search

A capsule-shaped input near the bottom of the pane body, centered, max-width `480px`, height `56px`. Bottom margin `32px` from the pane body's bottom edge (which is itself 24px above the dock, by the shell's contract — so the search has visible breathing room above the dock pill).

```css
height: 56px;
border-radius: 28px;       /* full pill */
padding: 0 24px 0 56px;    /* left padding leaves room for the search icon */
background: rgba(255, 255, 255, 0.06);
backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.08);
font-size: 15px;
color: var(--neutral-100);
transition: border-color 200ms ease, background 200ms ease,
            box-shadow 200ms ease;

::placeholder {
  color: var(--neutral-500);
}

:focus {
  outline: none;
  border-color: rgba(52, 211, 153, 0.35);
  background: rgba(255, 255, 255, 0.08);
  box-shadow: 0 0 0 4px rgba(52, 211, 153, 0.10);
}
```

A Lucide `Search` icon sits inside on the left at `20px`, color `neutral-400`. Placeholder: `Search apps, agents, or anywhere…`.

### 6.1 Behavior

- `Cmd+K` (or `Ctrl+K` on Linux) focuses the input from anywhere in the Home — global keyboard listener registered on mount, removed on unmount.
- Typing filters the app grid live, debounced 80ms. Filter logic: `name`, `description`, `category` substring match (case-insensitive).
- Empty input: full grid. Non-empty input: only matching icons; `+ Add` icon stays in place. If the result set is empty, the grid hides and a single line of helper text appears in its place: `No apps match. [Browse marketplace →]`.
- `Esc` clears the input and re-focuses it.
- Autocomplete OFF; `type="search"`; `aria-controls="home-app-grid"`; the grid carries the matching `id`.

### 6.2 Sticky-ish positioning

The search is the last element of the pane body's flex column. The grid above it is `flex-1` with `overflow-y: auto`, so the search effectively pins to the bottom of the visible body while the grid scrolls. We do **not** use `position: sticky` — the simpler flex layout produces the same effect without the z-index choreography.

## 8. Empty state

When the user has zero apps installed AND no auto-discovered local components — rare in practice (Eve seeds OpenWebUI + Synap by default), but we handle it cleanly.

Replace Zones B and C with a centered hero:

```
                       ┌──────────┐
                       │  ▣▣▣▣▣  │     ← Lucide LayoutGrid, 64px, neutral-400
                       │  ▣▣▣▣▣  │
                       └──────────┘

                Your OS is a blank canvas

         Install your first app from the marketplace.

                  [ Browse marketplace ]      ← pill button, emerald gradient
```

| Element | Spec |
|---|---|
| Icon | Lucide `LayoutGrid`, `64px`, `neutral-400`, no background |
| Headline | `24px`, `font-weight: 300`, `neutral-100`, top margin `24px` |
| Body | `15px`, `neutral-400`, max-width `360px`, centered, top margin `12px` |
| CTA | Pill button, height `44px`, padding `0 24px`, background `linear-gradient(135deg, #10B981, #34D399)`, white text, no shadow. Hover: brightness `1.05`. Press: `scale(0.98)`. |

Zone A (greeting) and Zone D (search) **remain visible** — the operator can still search the marketplace via Cmd+K (in Phase 4), and the greeting keeps the surface from feeling broken. The `+ Add` icon does not render here; the CTA replaces it.

## 9. Loading states

The Home is built to feel instant even on a cold cache. Different zones use different strategies:

| Zone | Strategy |
|---|---|
| **Greeting (A)** | Renders immediately, no skeleton. `firstName` defaults to "" (line collapses to "Good evening,") if profile hasn't resolved — this is fine for ~150ms and avoids a jarring name-pop. |
| **Stat cards (B)** | Three skeleton cards at full final dimensions. Value bar shimmers (1.6s `opacity` pulse, 60% to 100%). Sublabel hidden during skeleton. Real values fade in over 140ms as resolved. |
| **App grid (C)** | Render N skeleton icons matching the count cached from last visit (stored in `localStorage` as `home.iconCount`). Each skeleton is the icon's bounding box with the same shimmer pulse — no glyph, no label. As real apps resolve, fade them in (140ms opacity). |
| **Search (D)** | Renders immediately, fully interactive even before grid loads. Empty filter just shows skeletons. |

If the cached icon count is missing (genuine first load), render 6 skeletons — a sensible default for Eve's seeded apps.

## 10. Animation grammar

All animations match the [shell's animation grammar](./eve-os-shell.mdx#9-animation-grammar-locked). Home-specific entries:

| Surface | Property | Duration | Easing |
|---|---|---|---|
| Stat card hover | `background` + `box-shadow` (inner glow) | 200ms | `ease-out` |
| App icon hover | `transform: scale(1.04)` + `box-shadow` (outer glow) | 200ms | `ease-out` |
| App icon press | `transform: scale(0.96)` | 80ms | `ease-in` |
| Search input focus | `border-color` + `box-shadow` (focus ring) | 200ms | `ease-out` |
| Skeleton shimmer | `opacity` pulse 60%↔100% | 1.6s loop | `ease-in-out` |
| New icon resolved | `opacity` 0→1 | 140ms | `ease-out` |
| Newly installed icon (post-install return) | `transform: scale(0.85)→1` + `opacity` 0→1 | 240ms | `cubic-bezier(0.16, 1, 0.3, 1)` (overshoot pop) |

`prefers-reduced-motion: reduce` disables the scale animations on icons and cards (they swap to a 1px border-color brighten instead). Skeleton shimmer slows to 3s but doesn't disappear — the user still needs to see "this is loading."

No `translateY` anywhere. No drop shadows on icons or cards (depth comes from glow rings and borders). No layout-property animations.

## 11. Brand color registry

A new file: `app/(os)/lib/brand-colors.ts`. The shell's dock also reads this (so dock and grid stay synchronized when an app is added).

```typescript
// app/(os)/lib/brand-colors.ts

export type BrandColor = {
  /** CSS background — solid color or gradient string */
  bg: string;
  /** Lucide icon name. null means use remote iconUrl. */
  glyph: string | null;
  /** Solid color for dock active indicator + hover ring */
  accent: string;
};

export const BRAND_COLORS: Record<string, BrandColor> = {
  // Core / pinned
  home:      { bg: 'linear-gradient(135deg, #A78BFA, #818CF8)', glyph: 'Home',          accent: '#A78BFA' },
  agents:    { bg: '#34D399',                                    glyph: 'Sparkles',      accent: '#34D399' },
  settings:  { bg: '#475569',                                    glyph: 'Settings',      accent: '#94A3B8' },

  // Eve components
  openwebui: { bg: 'linear-gradient(135deg, #10B981, #34D399)', glyph: 'MessageSquare', accent: '#10B981' },
  synap:     { bg: '#34D399',                                    glyph: 'Brain',         accent: '#34D399' },
  'dev-agent':{ bg: 'linear-gradient(135deg, #4F46E5, #7C3AED)',glyph: 'Code2',         accent: '#6366F1' },

  // Marketplace classics
  'the-arch':{ bg: '#5B21B6',                                    glyph: null,            accent: '#7C3AED' },
  crm:       { bg: '#E11D48',                                    glyph: 'Users',         accent: '#F43F5E' },
  devplane:  { bg: '#F59E0B',                                    glyph: 'LayoutGrid',    accent: '#FBBF24' },
};

/** Deterministic fallback for unregistered apps. Hue-rotates emerald by slug hash. */
export function fallbackBrandColor(slug: string): BrandColor {
  const hue = hashToHue(slug);  // 0–359
  return {
    bg: `hsl(${hue}, 65%, 50%)`,
    glyph: null,                  // remote iconUrl will render
    accent: `hsl(${hue}, 65%, 60%)`,
  };
}
```

The shell's `app/(os)/components/dock-icon.tsx` and the Home's `app/(os)/components/app-icon.tsx` both read from this registry, so adding an entry instantly skins both surfaces.

## 12. Data model

The state shape is largely unchanged from v1 — only `ChannelStatus` is removed (the dock and the Agents app cover that surface area).

```typescript
type AppSource = 'cp_marketplace' | 'local_component' | 'pinned_default';

type App = {
  id: string;                    // stable across sources
  source: AppSource;
  slug: string;                  // brand-colors.ts registry key
  name: string;
  description?: string;
  iconUrl?: string;              // remote URL, used when registry glyph is null
  url: string;                   // where clicking opens
  category: 'ai' | 'productivity' | 'communication' | 'dev' | 'other';
  status: 'online' | 'degraded' | 'offline' | 'unknown';
  isEntitled: boolean;           // false → render as "Buy" CTA variant
  installCount?: number;         // for sort
};

type StatCardData = {
  agentsRunning: number;         // count active agents
  agentsSubLabel?: string;
  eventsToday: number;           // events.since(startOfDay)
  eventsSubLabel?: string;
  updatesAvailable: number;      // marketplace apps with newer version
  updatesSubLabel?: string;
};

type HomeState = {
  apps: App[];
  pinnedAppIds: string[];
  searchQuery: string;
  stats: StatCardData;
  isLoadingMarketplace: boolean;
  isLoadingStats: boolean;
  marketplaceError?: string;
  cpAuthRequired: boolean;       // drives CpAuthBanner visibility
};
```

### 12.1 Data sources merged

Identical to v1 — keep this logic intact during the rebuild:

- **`fetchMarketplaceApps(cpUserToken)`** — CP `GET /api/marketplace/apps?for_user=me`, returns entitled apps.
- **`fetchLocalComponents()`** — Eve dashboard `GET /api/components`, returns auto-discovered components with `webUrl`. Hydrates per-app live status onto matching `eve_component` entries by `metadata.componentId`.
- **`fetchStats()`** — three parallel calls: agent registry, `events.since(startOfDay)` via Hub Protocol, marketplace `?has_update=true` count.

```typescript
async function loadHomeApps(): Promise<App[]> {
  const [marketplaceApps, localComponents] = await Promise.allSettled([
    fetchMarketplaceApps(cpUserToken),
    fetchLocalComponents(),
  ]);
  return mergeAndHydrate(marketplaceApps, localComponents);
}
```

Marketplace failure (network or 401) → fall back to local components, set `cpAuthRequired: true`, banner appears. The OS stays usable.

### 12.2 Pinning persistence

Unchanged from v1. Pin order in `~/.eve/preferences.json` under `home.pinnedAppIds: string[]`. New apps go to the end; pinned defaults (Synap, OpenWebUI) appear first on first launch.

API: `GET /api/preferences/home`, `PUT /api/preferences/home`. New endpoints in the Eve dashboard's Next.js API routes.

## 13. Interaction flows

### 13.1 First load (returning user)

1. Pane mounts → reads `~/.eve/preferences.json` (instant).
2. Greeting renders immediately. Skeleton stat cards and N skeleton icons paint within the same tick.
3. Three parallel fetches in the background:
   - Local components (`~50ms`)
   - Marketplace (`~200ms` cached, `~600ms` cold)
   - Stats (`~100ms`)
4. Real cards and icons fade in (140ms opacity) as each resolves.
5. Search is interactive throughout.

### 13.2 First load (brand-new user)

1. Pane mounts. No prefs, no token.
2. CpAuthBanner appears above Zone A.
3. Greeting renders; stat cards render with zeroes; grid renders with the seeded local components only (Synap + OpenWebUI by default).
4. User clicks "Sign in" → OAuth handshake → token stored → marketplace re-fetches → entitled apps fade in.

### 13.3 Search

1. User focuses the input or hits `Cmd+K`.
2. Types → 80ms debounce → grid filters.
3. `Esc` clears the input. Grid restores.

### 13.4 Pin / unpin / reorder (drag-and-drop deferred)

v2 ships **without** drag-and-drop. Pinning toggles via the right-click contextual menu only. Reorder support returns in a follow-up — the icon-grid aesthetic makes drag handles harder to design well, so we defer until we have a clean solution. `pinnedAppIds` persistence still flushes via `PUT /api/preferences/home` on each pin/unpin.

### 13.5 Install from marketplace

1. User clicks the `+ Add` icon at the end of the grid.
2. Phase 2: opens external CP marketplace URL in new tab. Phase 4: routes to in-OS `/marketplace` (Channels-style sub-app).
3. After install, user returns to Home → `useHomeApps` re-fetches → new icon scale-pops in (240ms ease-out, see §10).

### 13.6 OAuth handshake

Identical to v1 §5.6. Code is owned by the OAuth client utility — referenced here only because the banner triggers it. See [Eve OS Vision §6](./eve-os-vision.mdx#6-auth-handshake-eve--cp).

## 15. Accessibility

- Each app icon is a real `<a>` with `href` (no div+onClick).
- Tab order matches DOM order (which matches pin order). Enter / Space activates.
- Search input is `<input type="search">` with `aria-controls="home-app-grid"`.
- Stat cards are `<a>` elements with `aria-label`s describing the metric and target.
- Status text under icons reads at 14px `neutral-200` against the pane glass — verified ≥4.5:1 contrast in both modes.
- The brand-color background of an icon is decorative; the glyph and label carry the meaning.
- `prefers-reduced-motion: reduce` disables scale + glow animations. Hover state instead applies a 1px border-color brighten. Skeleton shimmer slows to 3s.
- Focus rings: search input gets a 4px emerald-tinted ring; icons and cards get a 2px solid `brand.agents` outline (no glow when focused-via-keyboard, distinct from hover glow).

## 18. Reference: what this is NOT

To prevent scope creep during implementation:

- Not a workflow editor — that's the Agents app.
- Not a metrics dashboard — three stat cards is the ceiling, no charts.
- Not a chat UI — that's OpenWebUI / a future native Synap chat, accessed via the Chat icon.
- Not a marketplace browser — `+ Add` opens the marketplace as a separate surface.
- Not a settings panel — the gear icon in the shell pane header navigates to `/settings`.
- Not a notification center.
- Not a sidebar layout — sidebars exist only inside the Settings app, owned by that app.

The Home does ONE thing: launch apps and surface three glanceable facts about the OS. Everything else is one click away — through the dock, through the gear, or through search.
