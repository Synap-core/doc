### 3. Hub Protocol: AI Interoperability

**What it is:** Open standard for AI services to access your data

**Why It Matters:**

- ✅ **No Vendor Lock-In:** Switch AI providers without data migration
- ✅ **Mix AI Services:** Use multiple AIs together (best tool for each job)
- ✅ **Marketplace Ready:** Install community agents with one click
- ✅ **Custom AI:** Build and connect your own agents
- ✅ **Future-Proof:** New AIs plug in without rebuilding

**How It Works:**

**Step 1: Authentication (OAuth2)**

```
AI Service requests access to your data pod
You grant scoped permissions (read docs, create tasks, etc.)
Service receives time-limited access token
```

**Step 2: Standard API**

```
AI Service calls Hub Protocol endpoints:
- GET /hub/documents (search your documents)
- POST /hub/entities (propose creating task)
- GET /hub/conversations (access chat history)
```

**Step 3: Approval Workflow**

```
AI proposes action → Stored in your data pod → You review → Approve/reject
Only approved actions execute
```

**Example Scenarios:**

**Scenario 1: Use Synap's Hosted AI (Default)**

```
You: "Research our competitors"
→ Your Data Pod (open source)
→ Hub Protocol
→ Synap Intelligence Service (our 15 agents)
→ Research Agent does deep analysis
→ Results stored back in YOUR data pod

Benefits: Managed service, 15+ agents, automatic updates
```

**Scenario 2: Use Marketplace Agent**

```

```

**Hub Protocol Technical Details:**

**Authentication:**

```http
POST /oauth/token
client_id: your_ai_service_id
client_secret: your_ai_service_secret
scope: read:documents write:entities read:conversations

Returns: access_token (time-limited)
```

**Standard Endpoints:**

```http
# Search
POST /hub/search/documents
POST /hub/search/entities
POST /hub/search/vector

# Retrieve
GET /hub/documents/:id
GET /hub/entities/:id
GET /hub/conversations/:threadId

# Propose Actions (requires approval)
POST /hub/proposals/create_entity
POST /hub/proposals/update_document
POST /hub/proposals/create_branch
```

### 6. Approval Workflows

**What it is:** AI proposes actions, you approve before execution

**Why Open Source Matters Here:**

- All proposals stored in YOUR database
- Full audit trail you can query
- Export approval history anytime
- Custom approval rules (modify open-source code)

**How it works:**

1. AI suggests action (create task, update document, etc.)
2. Proposal stored in your data pod
3. You see in UI with AI's rationale
4. Approve or reject with one click
5. If approved: Action executes
6. Result stored in your database

**Example:**

```
User: "Create tasks from our meeting notes"

Task Agent:
  Reads meeting notes (from YOUR data pod)
  Identifies 5 potential tasks
  Proposes each task with rationale:

  Proposal 1: Create task "Review Q2 financials"
  Rationale: "Alice mentioned needed by Friday"

  Proposal 2: Create task "Call supplier about delay"
  Rationale: "Bob raised concern about shipment"

User: Reviews proposals
  - Approves Proposal 1 ✅
  - Edits Proposal 2 (changes deadline) ✅
  - Rejects Proposal 3 ❌

Result: 2 tasks created in YOUR database
```

**Granular Control:**

- Approve individual actions
- Edit proposals before approving
- Set default approvals for trusted actions
- Revoke past approvals
- Full audit trail in your database

**Privacy Benefit:**

- Proposals reviewed locally (your data pod)
- No data sent to AI unless approved
- You see what AI sees before execution

### For Developers

**Building Hub Protocol Agents:**

**Step 1: Register Your Agent**

```typescript
// Register with user's data pod
const registration = await dataPod.registerAgent({
  name: "My Custom Agent",
  description: "Specialized in X",
  scopes: ["read:documents", "write:entities"],
  webhookUrl: "https://my-agent.com/webhook",
});
```

**Step 2: Implement OAuth2**

```typescript
// User authorizes your agent
app.get("/oauth/authorize", (req, res) => {
  const { userId, dataPodUrl } = req.query;
  // Show authorization UI
  // User grants permissions
  // Redirect back with auth code
});

app.post("/oauth/token", async (req, res) => {
  const { code } = req.body;
  // Exchange code for access token
  const token = await dataPod.exchangeToken(code);
  // Store token for API calls
});
```

**Step 3: Call Hub Protocol APIs**

```typescript
import { HubProtocolClient } from "@synap/hub-protocol";

const client = new HubProtocolClient({
  dataPodUrl: user.dataPodUrl,
  accessToken: user.accessToken,
});

// Search user's documents
const docs = await client.searchDocuments({
  query: "pricing strategy",
  limit: 10,
});

// Propose action
const proposal = await client.proposeAction({
  type: "create_entity",
  data: {
    type: "task",
    title: "Review pricing",
    description: "Based on research findings",
  },
});
```

**Step 4: Handle Approvals**

```typescript
// Webhook when user approves/rejects
app.post("/webhook/approval", (req, res) => {
  const { proposalId, decision, userId } = req.body;

  if (decision === "approved") {
    // Execute the action
    await executeAction(proposalId);
  }
});
```
