## Vision statement

Eve turns a **bare server** into a **coherent self-hosted stack** centered on the **Synap Data Pod**. Synap is the centralized memory, task queue, and knowledge graph. Agents (OpenClaw, Hermes) operate around Synap, reading from and writing to it exclusively — no agent maintains its own long-term state.

---

## Design principles

1. **Synap as exclusive memory** — Every agent reads from and writes to Synap via REST API. No agent (OpenClaw, Hermes, opencode/OpenClaude) runs its own SQLite, Vector DB, or external state store. This is non-negotiable.

2. **Sovereignty** — Operator controls the machine, data residency, and whether cloud models are used at all for local tooling. No mandatory cloud dependency beyond what the operator chooses.

3. **Separation of concerns between agents** — OpenClaw handles **messaging ingress** (WhatsApp, iMessage, Telegram, voice memos, reasoning). Hermes Agent handles **development orchestration** (reading task queue, spawning codegen). They never speak directly to each other.

4. **Organic composition** — Services are **organs**: same network (`eve-network`), shared secrets patterns, predictable ports for sidecars vs Synap's Caddy.

5. **Synap-centric** — Hub, proposals, and pod contracts remain **authoritative**. Eve does not fork Synap semantics.

6. **Explicit exposure** — Public **Synap** URL vs **Eve Legs** (Traefik / tunnel) are **separate** user choices.

7. **Recoverable setup** — `.eve/setup-profile.json` and `.eve/secrets/` are inspectable; scripted paths stay reproducible.

## Agent responsibilities

| Agent | Handles | Does NOT handle |
|-------|---------|-----------------|
| **OpenClaw (Arms)** | Messaging (WhatsApp, iMessage, Telegram), voice memos, user-facing reasoning | Coding tasks, database access, direct communication with other agents |
| **Hermes Agent (Builder)** | Headless dev orchestration, task queue polling, spawning codegen tool (opencode or OpenClaude), writing results to Synap | Messaging apps, user-facing UI, direct communication with other agents |
| **opencode / OpenClaude** | Code generation (invoked via subprocess by Hermes) | Orchestration, queue management, persistence |

## What Eve is not

- **Not** a second Hub or auth system — identity and governance stay in Synap.
- **Not** a replacement for **`synap install`** — Eve **delegates** full Data Pod installs to the official **`synap`** script when using `data_pod` / `full` profiles.
- **Not** the same as **[synap-cli](https://github.com/Synap-core/synap-cli)** (OpenClaw bridge / agent init) — complementary tooling.
- **Not** an independent agent — Eve is infrastructure (Docker, networking, secrets). It does not "reason" or "decide."
