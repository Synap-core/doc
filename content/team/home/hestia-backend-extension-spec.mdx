# Backend Extension Specification

> Exact API contracts for eve integration — based on pure entity system and agnostic intelligence.

## 1. Philosophy

### Entity-First Approach
Following Decision #2 (Option A): **Everything is an entity.**

- `hearth_node` → Entity with `hearth_node` profile
- `package_instance` → Entity with `package_instance` profile  
- `intelligence_provider` → Entity with `intelligence_provider` profile
- `hearth_deployment` → Entity with `hearth_deployment` profile

**No new database tables.** We leverage the existing entity system.

### Agnostic Intelligence
Following Decision #3 & #6: **Any fire can warm the hearth.**

The Synap Backend provides a uniform interface to any intelligence provider:
- Local: Ollama, vLLM, llama.cpp
- Cloud: OpenRouter, Anthropic, OpenAI
- Custom: Any OpenAI-compatible endpoint

## 3. Hub Protocol Endpoints

### 3.1 Hearth Node Management

#### POST `/api/hub/hearth/register`

Register a new hearth node.

**Request:**
```json
{
  "hostname": "my-digital-hearth",
  "role": "primary",
  "install_mode": "script",
  "intelligence_provider": {
    "provider_type": "ollama",
    "endpoint_url": "http://localhost:11434",
    "model": "llama3.1:8b"
  }
}
```

**Response:**
```json
{
  "success": true,
  "hearth_node": {
    "id": "uuid",
    "entity_id": "uuid",
    "hostname": "my-digital-hearth",
    "role": "primary",
    "health_status": "healthy",
    "intelligence_provider_id": "uuid"
  },
  "api_key": {
    "id": "uuid",
    "key": "hub_xxxx",
    "scopes": ["hub-protocol.read", "hub-protocol.write", "hearth.*"]
  },
  "intelligence_provider": {
    "id": "uuid",
    "entity_id": "uuid"
  }
}
```

**Implementation:**
```typescript
// 1. Create intelligence_provider entity
const providerEntity = await entityService.create({
  workspaceId,
  profileSlug: 'intelligence_provider',
  title: 'Local Ollama',
  properties: request.intelligence_provider
});

// 2. Create hearth_node entity
const nodeEntity = await entityService.create({
  workspaceId,
  profileSlug: 'hearth_node',
  title: request.hostname,
  properties: {
    hostname: request.hostname,
    role: request.role,
    install_mode: request.install_mode,
    intelligence_provider_id: providerEntity.id,
    health_status: 'healthy',
    packages: []
  }
});

// 3. Generate API key with hearth.* scopes
const apiKey = await hubApiKeyService.create({
  workspaceId,
  userId,
  scopes: ['hub-protocol.read', 'hub-protocol.write', 'hearth.*'],
  metadata: { hearth_node_id: nodeEntity.id }
});

// 4. Update entity with api_key_id (secret)
await entityService.update(nodeEntity.id, {
  properties: {
    ...nodeEntity.properties,
    api_key_id: apiKey.id
  }
});
```

```json
    {
      "type": "function",
      "function": {
        "name": "deploy_artifact",
        "description": "Deploy an artifact"
      }
    }
  ],
  "stream": false
}
```

**Response (OpenAI-compatible):**
```json
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "created": 1712345678,
  "model": "llama3.1:8b",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you today?"
      },

    }
  ],
  "usage": {
    "prompt_tokens": 20,
    "completion_tokens": 10,
    "total_tokens": 30
  }
}
```

**Implementation:**
```typescript
// 1. Get hearth node and intelligence provider
const hearthNode = await entityService.get(hearthNodeId);
const providerId = hearthNode.properties.intelligence_provider_id;
const provider = await entityService.get(providerId);

// 2. Route to appropriate handler based on provider_type
const handler = getIntelligenceHandler(provider.properties.provider_type);

// 3. Call provider
const response = await handler.query({
  endpoint: provider.properties.endpoint_url,
  apiKey: getApiKey(provider.properties.api_key_env),
  model: request.model || provider.properties.model,
  messages: request.messages,
  tools: request.tools,
  temperature: request.temperature
});

// 4. Return standardized response
return c.json(response);
```

**Provider Handlers:**

```typescript
// Ollama handler
const ollamaHandler = {
  async query(config) {
    const response = await fetch(`${config.endpoint}/api/chat`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        model: config.model,
        messages: config.messages,
        tools: config.tools,
        options: { temperature: config.temperature }
      })
    });
    
    // Transform Ollama response to OpenAI format
    return transformOllamaToOpenAI(await response.json());
  }
};

// OpenRouter handler
const openrouterHandler = {
  async query(config) {
    const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${config.apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        model: config.model,
        messages: config.messages,
        tools: config.tools,
        temperature: config.temperature
      })
    });
    
    // Already OpenAI-compatible
    return await response.json();
  }
};
```

### 3.4 Deployment Management

#### POST `/api/hub/hearth/deploy`

Trigger deployment on a hearth node.

**Request:**
```json
{
  "hearth_node_id": "uuid",
  "source_type": "git",
  "source_url": "https://github.com/user/site",
  "artifact_type": "static",
  "build_config": {
    "command": "npm run build",
    "output_dir": "dist"
  },
  "requires_approval": true
}
```

**Response:**
```json
{
  "success": true,
  "deployment": {
    "id": "uuid",
    "entity_id": "uuid",
    "status": "pending",
    "requires_approval": true,
    "approval_request_id": "uuid"
  },
  "message": "Deployment created. Approval required to proceed."
}
```

**Implementation:**
```typescript
// 1. Create hearth_deployment entity
const deployment = await entityService.create({
  workspaceId,
  profileSlug: 'hearth_deployment',
  title: `Deploy ${request.source_url}`,
  properties: {
    hearth_node_id: request.hearth_node_id,
    source_type: request.source_type,
    source_url: request.source_url,
    artifact_type: request.artifact_type,
    status: request.requires_approval ? 'pending' : 'approved',
    started_at: new Date()
  }
});

// 2. If approval required, create proposal
if (request.requires_approval) {
  const approvalRequest = await proposalService.create({
    type: 'hearth_deployment',
    entityId: deployment.id,
    description: `Deploy ${request.source_url} to ${request.hearth_node_id}`,
    requester: userId,
    actions: [
      { type: 'execute_build', config: request.build_config }
    ]
  });
  
  await entityService.update(deployment.id, {
    properties: {
      ...deployment.properties,
      approval_request_id: approvalRequest.id
    }
  });
}

// 3. Queue build job (via jobs system)
await jobQueue.add('hearth.build', {
  deploymentId: deployment.id,
  hearthNodeId: request.hearth_node_id,
  buildConfig: request.build_config
});
```

## 7. Job Workers (pg-boss)

### hearth.build

**File:** `/packages/jobs/src/workers/hearth/build.ts`

```typescript
export const hearthBuildJob = job(
  'hearth.build',
  async (data: {
    deploymentId: string;
    hearthNodeId: string;
    buildConfig: any;
  }) => {
    // 1. Get hearth node endpoint from entity
    const node = await entityService.get(data.hearthNodeId);
    const apiKey = await getApiKey(node.properties.api_key_id);
    
    // 2. Call hearth node to execute build
    const result = await fetch(`${node.properties.endpoint}/build`, {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${apiKey}` },
      body: JSON.stringify(data.buildConfig)
    });
    
    // 3. Update deployment entity with result
    await entityService.update(data.deploymentId, {
      properties: {
        status: result.success ? 'deployed' : 'failed',
        url: result.url,
        build_log: result.logs,
        completed_at: new Date()
      }
    });
  }
);
```

### hearth.health-check

**Cron job:** Every 5 minutes

```typescript
export const hearthHealthCheck = cron(
  'hearth.health-check',
  '*/5 * * * *',
  async () => {
    // Find hearth nodes with stale heartbeats
    const staleNodes = await entityService.query({
      profileSlug: 'hearth_node',
      filter: {
        last_heartbeat: { $lt: new Date(Date.now() - 10 * 60 * 1000) },
        health_status: 'healthy'
      }
    });
    
    for (const node of staleNodes) {
      await entityService.update(node.id, {
        properties: { health_status: 'offline' }
      });
      
      await notificationService.create({
        type: 'hearth_offline',
        entityId: node.id,
        message: `Hearth ${node.title} appears offline`
      });
    }
  }
);
```
