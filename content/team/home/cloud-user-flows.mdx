## 2. Authentication & Account

### 2.1 Control Plane Auth (Better Auth)

| Aspect | Status | Details |
|--------|--------|---------|
| Login (email + password) | DONE | Via IPC `signInWithBetterAuth()` |
| Registration (email + password + name) | DONE | Via IPC `signUpWithBetterAuth()` |
| Credential storage (OS keychain) | DONE | `saveCredentialWithPassword()` for auto-reconnect |
| Biometric unlock on return | DONE | Prompts Touch ID / password to re-auth silently |
| Session validation on startup | DONE | `checkSession()` via IPC `getBetterAuthSession()` |
| Logout | DONE | Clears session + keychain credentials |
| Password reset | NOT DONE | No "forgot password" flow in browser |
| OAuth / social login | NOT DONE | Only email+password supported |

### 2.2 Pod Auth (Kratos — Self-Hosted)

| Aspect | Status | Details |
|--------|--------|---------|
| Login with Kratos flow | DONE | Creates flow, extracts CSRF, submits |
| Registration with Kratos | DONE | Dual mode toggle (login/register) |
| Existing session detection | DONE | Auto-skips login if valid session found |
| Error extraction from flow | DONE | Parses `ui.messages` and `node.messages` |

### 2.3 Handshake Token Exchange

| Aspect | Status | Details |
|--------|--------|---------|
| Token generation (Control Plane) | DONE | 5-min JWT with `aud=podUrl`, `iss=synap-control-plane` |
| Token exchange at pod (IPC) | DONE | `exchangeHandshakeToken(podUrl, token)` creates Kratos session |
| Pod ownership validation | DONE | Cross-references target hostname against user's pods |
