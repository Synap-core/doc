**Eve** is our **Entity Creation System**: it turns a bare machine into a coherent, self-hosted “digital entity” — **Brain** (Synap data pod + DB + cache + local AI), **Arms** (OpenClaw), **Builder** (creation/deploy tooling), **Eyes** (RSS/RSSHub intake), **Legs** (Traefik, TLS, exposure). The user-facing CLI command is **`eve`**.

**Monorepo in this workspace:** `hestia-cli/` (package `packages/eve-cli`, build `pnpm run build:eve`). Product narrative and deep README also live next to the older **Project Hestia** doc tree under `hestia/` — **Eve is the current product name**; treat Hestia-named files as legacy unless updated.

---

## Vision

- **Sovereignty:** Your server, your data, optional fully local AI — no mandatory cloud.
- **One coherent system:** Not a bag of unrelated installers; organs are wired to the same Docker network and mental model.
- **Synap as Brain:** The Synap backend is the knowledge graph and API core; Eve is how operators **stand up** that stack (and friends) reproducibly.
