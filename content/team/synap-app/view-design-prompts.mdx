## 🎨 Brand Guidelines (Apply to All Views)

### **Color System**
- **Light Mode Primary**: Amber/Orange tones (#F59E0B, #FBBF24, #FCD34D)
- **Dark Mode Primary**: Blue tones (#3B82F6, #60A5FA, #93C5FD)
- **Entity Type Colors**:
  - Tasks: Green (#10B981)
  - People: Blue (#3B82F6)
  - Notes/Documents: Amber (#F59E0B)
  - Events: Purple (#8B5CF6)
  - Files: Gray (#6B7280)

### **Visual Style**
- Modern, clean, minimal
- Glassmorphism for overlays (blur, translucency)
- Solid backgrounds for content cards
- Subtle shadows for depth
- Smooth animations and transitions
- Beautiful, professional, trustworthy

### **Typography**
- **Headings**: Fraunces (serif, elegant)
- **Body**: DM Sans (sans-serif, clean)
- Clear hierarchy (size, weight, color)

### **Spacing**
- Generous white space
- Consistent padding and gaps
- Never cramped or cluttered

# 1. LIST VIEW

## Prompt

```
Design a modern, beautiful list view interface for a knowledge management application called Synap. This is a list of entities (tasks, notes, people, events) displayed as cards with rich metadata.

VISUAL STYLE:
- Modern, clean, professional design
- Light mode: white background (#FFFFFF), amber accent (#F59E0B)
- Dark mode: dark gray background (#1E1E1E), blue accent (#3B82F6)
- Cards have subtle elevation with soft shadows
- Smooth hover effects (cards lift slightly, show actions)

LAYOUT:
- Vertical scrolling list
- Each item is a horizontal card (full width, ~80px height)
- Left side: checkbox (optional) + icon + content (title, subtitle, badges)
- Right side: metadata columns (category, timestamp, user avatar)
- Far right: action buttons appear on hover

CARD STRUCTURE:
1. Checkbox (16px, rounded, subtle border)
2. Entity Icon (48px square, rounded corners, colored background based on type)
   - Tasks: green background, checkmark icon
   - People: blue background, user icon
   - Notes: amber background, document icon
   - Events: purple background, calendar icon
3. Content Area (flex, grows):
   - Title (bold, ~16px, high contrast)
   - Subtitle (lighter, ~14px, secondary text)
   - Badges (small pills, colored by type: "Top Hit" in orange, status in green/red)
4. Metadata Columns (fixed width, centered):
   - Category (icon + text, ~128px)
   - Timestamp (icon + relative time, ~128px)
   - User (avatar + name, ~96px)
5. Hover Actions (appear on right):
   - 3-4 icon buttons in a rounded container
   - Subtle background, border
   - Actions: preview, share, edit

INTERACTIONS:
- Hover: card lifts (translateY(-2px)), shadow increases, actions appear
- Selected: highlighted border in primary color
- Top Hit: special styling (orange glow, badge)

VISUAL DETAILS:
- 12px gap between cards
- Smooth transitions (200ms ease)
- Icons are 16-24px, clean line style
- Consistent border radius (8-12px)
- Text truncates with ellipsis if too long

FEELING:
- Organized, scannable, efficient
- Information at a glance
- Delightful interactions
- Professional yet friendly
- "Everything has its place"

EXAMPLES TO SHOW:
- 5-7 list items with varying content
- Mix of entity types (tasks, people, notes)
- One item hovered (showing actions)
- One item selected (highlighted)
- One "Top Hit" item with special badge
- Both light and dark mode versions

AVOID:
- Clutter or cramped spacing
- Harsh shadows or borders
- Too many colors competing
- Text that's hard to read
- Generic/boring design
```

# 2. GRID VIEW

## Prompt

```
Design a modern, beautiful grid view interface for a knowledge management application called Synap. This displays entities as cards in a responsive grid layout, similar to Pinterest or Notion gallery view.

VISUAL STYLE:
- Modern, clean, card-based design
- Light mode: light gray background (#F9FAFB), white cards, amber accent (#F59E0B)
- Dark mode: dark background (#0F0F0F), dark gray cards (#1E1E1E), blue accent (#3B82F6)
- Cards have elevation with subtle shadows
- Smooth hover effects (scale, lift, reveal actions)

LAYOUT:
- Responsive grid: 1 column (mobile), 2 (tablet), 3 (desktop), 4 (large screens)
- Cards have consistent width, variable height based on content
- 24px gap between cards
- Cards contain: icon, type badge, title, description, footer

CARD STRUCTURE:
1. Header Section:
   - Large entity icon (64px, top-left, colored by type)
   - Type badge (top-right, subtle, shows "Task", "Note", etc.)
   - Hover: action button appears (top-right, opens entity)
2. Content Section:
   - Title (bold, ~18px, 2-3 lines max)
   - Description/Preview (lighter, ~14px, 3-4 lines max)
3. Footer Section (horizontal scroll if needed):
   - Status indicator (dot + text, colored)
   - Timestamp (icon + relative time)
   - User avatars (if multiple users)
   - Priority badge (if applicable)
   - Custom metadata (tags, category)

CARD STYLING:
- Background: white (light) / #1E1E1E (dark)
- Border: 1px solid, subtle
- Border radius: 16px
- Padding: 20px
- Min height: 240px
- Hover: translateY(-4px), shadow increases, border glows

ENTITY TYPE STYLING:
- Tasks: Green icon background, checkmark icon
- People: Blue icon background, user icon, show online status (green dot)
- Notes: Amber icon background, document icon
- Events: Purple icon background, calendar icon

INTERACTIONS:
- Hover: card lifts, action button appears (open icon, primary color)
- Click: opens entity in detail view or modal
- Footer scrolls horizontally if too much metadata
- Smooth animations (300ms ease-out)

FOOTER DETAILS:
- Horizontally scrollable (if content overflows)
- Items have min-width, don't shrink
- Hide scrollbar but maintain scroll functionality
- Each metadata item: icon + text, subtle styling
- Spacing between items: 12px

FEELING:
- Visual, scannable, explorable
- Content-first approach
- Inviting to browse and discover
- Beautiful cards you want to click
- "Gallery of ideas"

EXAMPLES TO SHOW:
- Grid with 8-12 cards
- Mix of entity types
- One card hovered (showing open action)
- Cards with varying content lengths
- Footer with multiple metadata items (showing scroll)
- Both light and dark mode versions

AVOID:
- Cards feeling cramped
- Text overflow without truncation
- Footer items overlapping
- Inconsistent card heights (too jarring)
- Boring, lifeless cards
```

# 3. TABLE VIEW

## Prompt

```
Design a modern, professional table view interface for a knowledge management application called Synap. This is a data-dense, spreadsheet-like view optimized for comparing and managing many entities.

VISUAL STYLE:
- Clean, professional, data-focused design
- Light mode: white background, subtle gray borders (#E5E7EB), amber highlights (#F59E0B)
- Dark mode: dark background (#0F0F0F), subtle borders (#374151), blue highlights (#3B82F6)
- Minimal decoration, focus on data clarity
- Subtle hover effects on rows

LAYOUT:
- Full-width table with fixed header
- Sticky header row (stays visible on scroll)
- Columns: checkbox, icon, title, status, priority, assignee, due date, tags, actions
- Rows: 48-56px height, alternating subtle background (zebra striping optional)
- Horizontal scroll if needed (many columns)
- Vertical scroll for rows

TABLE STRUCTURE:
1. Header Row:
   - Column titles (bold, ~14px, uppercase or sentence case)
   - Sort indicators (arrow icons, appear on hover)
   - Resize handles between columns
   - Filter icons (appear on hover)
   - Background: slightly darker than body
   - Sticky (stays at top)

2. Data Rows:
   - Checkbox column (40px, left-aligned)
   - Icon column (48px, entity type icon with colored background)
   - Title column (flexible width, primary data, bold)
   - Status column (120px, badge with color)
   - Priority column (100px, colored indicator)
   - Assignee column (120px, avatar + name)
   - Due Date column (140px, formatted date, colored if overdue)
   - Tags column (flexible, multiple tag pills)
   - Actions column (60px, three-dot menu, appears on hover)

CELL STYLING:
- Padding: 12px horizontal, 16px vertical
- Text: left-aligned for text, center for icons
- Truncate with ellipsis if too long
- Hover: show full text in tooltip

ROW INTERACTIONS:
- Hover: subtle background change, actions appear
- Selected: checkbox checked, row highlighted with primary color tint
- Multi-select: multiple rows can be selected
- Click: opens entity detail (not checkbox/actions)

COLUMN FEATURES:
- Sortable: click header to sort (arrow indicates direction)
- Resizable: drag border between headers
- Reorderable: drag header to reorder columns
- Filterable: click filter icon to show filter UI
- Hideable: right-click to hide columns

SPECIAL CELL TYPES:
- Status: pill badge (green/amber/red/gray)
- Priority: colored dot + text (red=high, amber=medium, gray=low)
- Assignee: small avatar (32px) + name
- Due Date: colored text (red if overdue, amber if soon, gray if not urgent)
- Tags: small pills, scrollable horizontally if many

FEELING:
- Professional, powerful, efficient
- Optimized for bulk actions and comparison
- "Spreadsheet meets beautiful design"
- Scannable and organized
- Command over data

EXAMPLES TO SHOW:
- Table with 10-15 rows
- Mix of entity types
- One row hovered (showing actions)
- Multiple rows selected
- Sortable headers with indicators
- Both light and dark mode versions

AVOID:
- Cramped cells
- Harsh grid lines (use subtle borders)
- Too many colors competing
- Information overload
- Generic spreadsheet look
```

# 4. GRAPH VIEW (NETWORK/KNOWLEDGE GRAPH)

## Prompt

```
Design a modern, beautiful graph/network view interface for a knowledge management application called Synap. This shows entities as nodes and relationships as edges in a force-directed network visualization.

VISUAL STYLE:
- Modern, futuristic, elegant design
- Light mode: light background (#F9FAFB), colorful nodes, subtle edges
- Dark mode: dark background (#0A0A0A), vibrant nodes, glowing edges
- Nodes have depth (shadows, gradients optional)
- Edges have color based on relationship type
- Beautiful, organic layout (force-directed)

LAYOUT:
- Canvas fills entire viewport
- Nodes are positioned using force-directed algorithm (related items closer)
- Central node (if focused) is larger and prominent
- Zoom controls (bottom-right)
- Filter panel (top-right, collapsible)
- Mini-map (bottom-left, shows overview)

NODE DESIGN:
- Size: 60-120px diameter (varies by importance/connections)
- Shape: Rounded rectangle or circle
- Structure:
  1. Background: colored by entity type, subtle gradient
  2. Icon: centered, white or contrasting color (32-48px)
  3. Label: below node, 1-2 lines, bold
  4. Badge: small indicator (connection count, status)
- Entity Type Colors:
  - Tasks: Green (#10B981)
  - People: Blue (#3B82F6)
  - Notes: Amber (#F59E0B)
  - Events: Purple (#8B5CF6)
  - Files: Gray (#6B7280)

NODE STATES:
- Default: solid color, subtle shadow
- Hover: scale up slightly (1.1x), glow effect, show details panel
- Selected: prominent glow/ring, stronger shadow
- Connected: when hovering a node, connected nodes highlight
- Dimmed: when filtering, non-matching nodes fade (opacity 0.3)

EDGE DESIGN:
- Line width: 2-4px (thicker for stronger relationships)
- Color: based on relationship type
  - "assigned_to": Orange
  - "depends_on": Red
  - "mentions": Purple
  - "links_to": Cyan
  - "relates_to": Gray
- Style: curved paths (bezier), not straight lines
- Arrow: subtle arrow at target end
- Hover: thicker, brighter, show relationship label

EDGE STATES:
- Default: subtle, doesn't overwhelm
- Hover: highlighted, shows tooltip with relationship type
- Connected: when hovering node, its edges highlight

UI OVERLAYS:
1. Zoom Controls (bottom-right):
   - Glassmorphic panel
   - Buttons: zoom in, zoom out, fit to screen
   - Current zoom level indicator
   
2. Filter Panel (top-right):
   - Glassmorphic panel, collapsible
   - Entity type checkboxes
   - Relationship type checkboxes
   - Search input
   
3. Mini-map (bottom-left):
   - Small overview of entire graph
   - Current viewport shown as rectangle
   - Simplified view (dots for nodes)

4. Node Detail Panel (appears on hover/select):
   - Floats near node
   - Shows: title, type, metadata, quick actions
   - Glassmorphic design

INTERACTIONS:
- Pan: drag canvas to move around
- Zoom: scroll wheel or pinch
- Node: click to select, drag to reposition
- Hover: shows details, highlights connections
- Double-click node: opens in detail view
- Physics: nodes repel each other, edges pull connected nodes together

VISUAL EFFECTS:
- Smooth animations (300ms ease-out)
- Subtle particle effects on edges (optional)
- Node shadows for depth
- Glow effects on hover/select
- Glassmorphism on UI panels (blur, transparency)

FEELING:
- Exploratory, organic, alive
- "Network of ideas coming to life"
- Discover connections naturally
- Beautiful, almost hypnotic
- Powerful yet playful
- "Your knowledge, visualized"

EXAMPLES TO SHOW:
- Graph with 15-25 nodes
- Mix of entity types
- Various edge types and colors
- One node hovered (showing detail panel and highlighted connections)
- One node selected (with glow)
- UI overlays (zoom, filter, mini-map)
- Both light and dark mode versions

AVOID:
- Cluttered, overlapping nodes
- Too many edges (looks like spaghetti)
- Harsh colors
- Confusing layout
- Generic network diagram look
- Nodes too small to read
```

# 5. WHITEBOARD VIEW (INFINITE CANVAS)

## Prompt

```
Design a modern, beautiful infinite canvas/whiteboard interface for a knowledge management application called Synap. This is a spatial workspace where users can arrange entities, notes, drawings, and embedded views freely.

VISUAL STYLE:
- Modern, clean, expansive design
- Light mode: subtle dot grid background (#F9FAFB), white elements
- Dark mode: subtle dot grid background (#0F0F0F), dark gray elements
- Elements have elevation (shadows, borders)
- Smooth zoom and pan
- Professional yet creative

CANVAS:
- Infinite in all directions
- Subtle dot grid (helps with spatial orientation)
- Grid size: 24px, dots are subtle (#E5E7EB light, #262626 dark)
- Zoom levels: 25% to 400%
- Smooth pan and zoom transitions

ELEMENT TYPES:
1. Entity Cards:
   - Similar to grid view cards
   - Draggable, resizable
   - Shows icon, title, preview
   - Can be different sizes (S, M, L)
   
2. Text Boxes:
   - Rich text editor (inline)
   - Markdown support
   - Resizable, colored backgrounds optional
   
3. Sticky Notes:
   - Small, colored squares
   - Quick notes, reminders
   - Colors: yellow, green, blue, pink, purple
   
4. Images:
   - Display uploaded images
   - Resizable, maintain aspect ratio
   
5. Embedded Views:
   - Any other view (list, graph, etc.) embedded
   - Has border and resize handles
   - Shows mini version of the view
   
6. Connectors:
   - Lines/arrows between elements
   - Can be labeled
   - Curved or straight

ELEMENT INTERACTIONS:
- Drag: move elements around
- Resize: corner handles appear on select
- Rotate: rotation handle at top (optional)
- Connect: draw lines between elements
- Group: select multiple, group into container
- Z-order: right-click to bring to front/send to back

UI CHROME:
1. Toolbar (top):
   - Glassmorphic panel, centered
   - Tools: select, text, sticky note, shape, connector, image, embed view
   - Each tool: icon + label
   
2. Zoom Controls (bottom-right):
   - Current zoom percentage
   - Buttons: zoom in, out, fit all, 100%
   
3. Mini-map (bottom-left):
   - Shows overview of canvas
   - Current viewport rectangle
   - Click to jump to area
   
4. Collaboration Cursors:
   - Other users' cursors shown in real-time
   - Colored by user
   - Shows user name next to cursor

VISUAL DESIGN FOR ELEMENTS:
- Cards: white/dark gray, subtle shadow, 12px border radius
- Sticky Notes: solid colors (yellow, green, pink), handwriting font optional
- Text Boxes: minimal, focus on content
- Embedded Views: border, shows "loading" state initially
- Connectors: 2-3px lines, arrows at ends, can be colored

SELECTION & EDITING:
- Selected element: blue outline, resize handles at corners/edges
- Multi-select: drag rectangle to select multiple
- Selected elements: can move together, align, distribute
- Alignment guides: snap to other elements (dotted lines appear)
- Smart guides: show center, edges when aligning

CANVAS INTERACTIONS:
- Pan: drag canvas (space + drag, or middle mouse)
- Zoom: pinch, scroll wheel, or zoom buttons
- Select: click element
- Multi-select: shift+click or drag rectangle
- Quick add: double-click canvas to add text
- Context menu: right-click for options

VISUAL EFFECTS:
- Smooth zoom with transform origin at cursor
- Element shadows increase when dragging
- Subtle fade-in when creating new elements
- Smooth transitions for all movements
- Cursor changes based on tool/action

SPECIAL FEATURES:
- Snap to grid (toggle-able)
- Auto-arrange selected elements
- Templates (pre-made layouts)
- Comments on elements (appear as pins)
- Presentations mode (hide UI, fullscreen)

FEELING:
- Expansive, limitless, free
- "Your ideas have room to breathe"
- Creative and organized
- Collaborative and alive
- "Thinking canvas"
- Freedom to organize spatially
- Both structure and creativity

EXAMPLES TO SHOW:
- Canvas with mix of elements:
  - 3-4 entity cards
  - 2-3 sticky notes
  - 1 text box with content
  - 1 embedded view (list or graph)
  - Connectors between some elements
- Toolbar and zoom controls visible
- Mini-map showing overview
- One element being dragged (with shadow)
- Collaboration cursor of another user
- Both light and dark mode versions

AVOID:
- Cluttered canvas
- Hard-to-see grid
- Clunky drag interactions
- Generic whiteboard look
- Missing visual feedback
- Elements looking flat/boring
```

# 6. TREEMAP VIEW (HIERARCHICAL VISUALIZATION)

## Prompt

```
Design a modern, beautiful treemap visualization for a knowledge management application called Synap. This displays hierarchical data as nested rectangles, where size represents importance/quantity.

VISUAL STYLE:
- Modern, clean, geometric design
- Light mode: white background, colored rectangles with subtle borders
- Dark mode: dark background (#0F0F0F), vibrant colored rectangles
- Rectangles have subtle gradients and borders
- Smooth zoom and animation

LAYOUT:
- Full viewport treemap
- Nested rectangles (parent contains children)
- Size = quantity or importance (number of children, relevance score, word count)
- Color = entity type or category
- Labeled rectangles (title visible)

RECTANGLE HIERARCHY:
1. Root Level (largest):
   - Fills viewport
   - Shows project/system name
   - Background: semi-transparent, subtle
   
2. Category Level:
   - Large rectangles (25-40% of parent)
   - Shows category name (Tasks, Notes, People, etc.)
   - Colored by type (green, amber, blue, purple)
   
3. Entity Level:
   - Smaller rectangles within categories
   - Shows entity title (truncated if needed)
   - Same color as parent, slightly lighter/darker

RECTANGLE STYLING:
- Border: 2px solid, slightly darker than fill
- Border radius: 8px (outer), 4px (inner)
- Padding: 8px between nested rectangles
- Fill: solid color + subtle gradient (lighter at top)
- Opacity: 0.9 for categories, 1.0 for entities

COLORS BY ENTITY TYPE:
- Tasks: Green spectrum (#10B981, #34D399, #6EE7B7)
- People: Blue spectrum (#3B82F6, #60A5FA, #93C5FD)
- Notes: Amber spectrum (#F59E0B, #FBBF24, #FCD34D)
- Events: Purple spectrum (#8B5CF6, #A78BFA, #C4B5FD)
- Files: Gray spectrum (#6B7280, #9CA3AF, #D1D5DB)

LABELS:
- Category labels: bold, ~18px, high contrast
- Entity labels: regular, ~14px, fits in rectangle
- Show only if rectangle is large enough (min 80x40px)
- If too small: show on hover
- Label position: top-left of rectangle, with padding

INTERACTIONS:
- Hover: rectangle brightens, border glows, shows tooltip with full info
- Click: zoom into that rectangle (it expands to fill viewport, showing children)
- Breadcrumb: top bar shows path (Root > Category > Entity)
- Back: click breadcrumb to zoom out
- Resize animation: smooth transition (400ms ease-out)

HOVER STATE:
- Rectangle: brightness +10%, border width +1px, shadow appears
- Tooltip: appears near cursor, glassmorphic panel
  - Shows: full title, type, count, metadata
  - Appears after 300ms hover

ZOOM INTERACTION:
- Click rectangle: animated zoom (selected rectangle expands to viewport)
- Children appear as the zoom completes
- Parent rectangles fade out
- Breadcrumb updates to show current level
- Back button/breadcrumb to zoom out

TOP UI:
1. Breadcrumb Navigation:
   - Top-left, glassmorphic panel
   - Shows: Root > Category > Entity
   - Click any level to zoom to it
   
2. Controls (top-right):
   - Size by: dropdown (count, importance, size)
   - Color by: dropdown (type, status, priority)
   - View mode: button to switch to other views
   
3. Legend (bottom-right):
   - Shows color mapping
   - Entity types and their colors

VISUAL EFFECTS:
- Smooth zoom animations
- Rectangles fade in when appearing
- Subtle gradient on fills
- Shadow on hover
- Glow effect on selected

FEELING:
- Organized, hierarchical, clear
- "See the big picture at a glance"
- Discover structure naturally
- Beautiful data visualization
- "Your data, spatially organized"
- Intuitive navigation
- Satisfying to explore

EXAMPLES TO SHOW:
- Root view with 3-4 categories visible
  - Each category with multiple entities
- Size variation (some large, some small)
- One rectangle hovered (glowing, tooltip)
- Breadcrumb showing current level
- Legend showing color mapping
- Both light and dark mode versions

EXAMPLES OF DATA:
- Project treemap:
  - Root: "Project Alpha"
  - Categories: "Tasks" (green), "Documents" (amber), "People" (blue)
  - Entities: individual tasks, docs, people
- Book treemap:
  - Root: "My Book"
  - Categories: "Chapter 1", "Chapter 2", "Chapter 3"
  - Entities: sections within chapters

AVOID:
- Rectangles too small to see
- Labels overlapping or unreadable
- Too many colors (confusing)
- Harsh borders or colors
- Janky zoom animations
- Feeling lost in hierarchy
```

# 7. TIMELINE VIEW (TEMPORAL VISUALIZATION)

## Prompt

```
Design a modern, beautiful timeline view for a knowledge management application called Synap. This displays entities chronologically, showing events, tasks, and activities over time.

VISUAL STYLE:
- Modern, clean, elegant design
- Light mode: white background, subtle gray line, colored events
- Dark mode: dark background (#0F0F0F), subtle white line, vibrant events
- Events displayed as cards along a timeline
- Smooth scrolling, zoom for time ranges

LAYOUT:
- Horizontal timeline (left to right = past to future)
- Central timeline line (2-3px, subtle)
- Events positioned along timeline
- Multiple "swim lanes" if many events at same time
- Time labels on timeline (dates, months, years)

TIMELINE LINE:
- Horizontal, runs through center
- Color: subtle gray (#D1D5DB light, #374151 dark)
- Width: 2-3px
- Milestones: dots/diamonds on line (project start, deadlines)
- Current time indicator: vertical line (primary color, glowing)

TIME LABELS:
- Positioned above timeline line
- Shows: dates, months, years (depends on zoom level)
- Font: ~12px, secondary color
- Spacing: every 100-200px
- Format changes with zoom:
  - Wide: "Jan 15, 2024"
  - Medium: "Jan 2024"
  - Narrow: "2024"

EVENT CARDS:
- Positioned on timeline at their date
- Connected to timeline with short line (20-30px)
- Card structure:
  1. Icon (entity type, colored circle, 32px)
  2. Title (bold, ~14px)
  3. Date/time (lighter, ~12px)
  4. Preview (1-2 lines, lighter)
  5. Status indicator (if task: done/pending)

CARD STYLING:
- Background: white (light) / #1E1E1E (dark)
- Border: 1px solid, subtle
- Border radius: 12px
- Padding: 16px
- Width: 240px
- Shadow: subtle
- Hover: lifts, shadow increases

CARD POSITIONING:
- If multiple events at similar time: offset vertically (swim lanes)
- Connector line from card to timeline
- Cards alternate above/below line (or all on one side)

ENTITY TYPE COLORS:
- Tasks: Green (#10B981) - checkmark icon
- Events: Purple (#8B5CF6) - calendar icon
- Notes: Amber (#F59E0B) - document icon
- Milestones: Red (#EF4444) - flag icon

INTERACTIONS:
- Scroll: horizontal scroll to move through time
- Zoom: buttons or pinch to zoom time scale
- Hover: card lifts, shows more details
- Click: opens entity detail
- Drag: move event to different date (if editable)

TIME NAVIGATION:
1. Zoom Controls (top-right):
   - Buttons: Day, Week, Month, Year, All
   - Current zoom level indicator
   
2. Date Navigator (top-center):
   - Shows current visible date range
   - Click to jump to date (date picker)
   - Prev/Next buttons
   
3. Mini Timeline (bottom):
   - Shows full timeline in miniature
   - Current viewport shown as rectangle
   - Click to jump to period

SPECIAL ELEMENTS:
- Today indicator: vertical line (primary color, glowing), "Today" label
- Milestones: diamond shapes on line (red for deadlines, green for completed)
- Periods: shaded regions for date ranges (projects, sprints)
- Connections: dotted lines between related events

GROUPING:
- Group by: day, week, month (depending on zoom)
- Group labels: date headers above groups
- Separator lines: subtle, between groups

VISUAL EFFECTS:
- Smooth scroll (momentum)
- Smooth zoom (cards resize/reposition)
- Cards fade in when scrolling
- Hover effects (lift, shadow)
- Today line pulses subtly

FILTERS (sidebar or top):
- Entity types: checkboxes
- Date range: slider or calendar
- Tags: multi-select
- Show completed: toggle

FEELING:
- Temporal, flowing, chronological
- "Your journey through time"
- Clear progression
- Easy to spot patterns
- "See what happened when"
```
