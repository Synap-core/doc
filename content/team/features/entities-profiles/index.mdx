# Entity Profiles

Every entity in Synap has a profile type (note, task, person, etc.). Profiles contain property definitions that determine which fields the entity has. Profiles can be extended with custom properties per workspace.

## Property schema layers

Property definitions follow a three-layer schema model:

| Layer | Scope | Description |
|-------|-------|-------------|
| **Layer 1: Global** (`profile_id=NULL, workspace_id=NULL`) | Visible in every workspace, for every profile. Set by system defaults. [More details](/team/platform/property-def-scoping) |
| **Layer 2: Profile base** (`profile_id=SET, workspace_id=NULL`) | Visible in every workspace that uses this profile. Workspace-owned profiles keep `workspace_id=NULL`. [More details](/team/platform/property-def-scoping) |
| **Layer 3: Workspace overlay** (`profile_id=SET, workspace_id=SET`) | Visible only in this workspace. Created when a workspace extends a pod-wide profile. [More details](/team/platform/property-def-scoping) |

## Custom profiles

Users can create custom entity types through the Profile Editor in Settings. Custom profiles can be set to pod-wide or workspace-scoped. The intelligence service auto-classifies incoming content into the appropriate profile type.
