## Topology

```
Internet
  │
  ├─ Caddy :80/443 ───→ Synap Data Pod (Caddy, API :4000, Auth)
  │
  └─ Traefik / Pangolin ─→ Eve sidecars (eve-network)
         ├─ Ollama (inference gateway 127.0.0.1:11435)
         ├─ OpenClaw (Arms)
         ├─ RSSHub (Eyes)
         ├─ Traefik dashboard
         └─ Open WebUI :3011 (hestia-network ∩ eve-network — optional add-on)
                ├─ → Synap IS /v1/chat/completions (OpenAI-compat)
                └─ → Synap Hub POST /mcp (tools, memory, entities)
```

**How to read it:** Traffic from the **internet** hits **Legs** (Traefik / tunnel) when you expose sidecar routes. **Synap** owns **Caddy :80/443** and the **API :4000** path for the Data Pod. **Eve sidecars** (Ollama, OpenClaw, RSSHub, etc.) sit on **`eve-network`**. Optional organs attach after Brain/Legs basics — exact compose depends on profile.

## Synap vs Eve (boundary)

| Layer | Owner | Notes |
|-------|-------|-------|
| HTTPS for Pod API/auth | **Synap** (Caddy) | Public "pod domain" in `eve setup` configures **Synap** install. |
| Inference gateway / Traefik for sidecars | **Eve** (Legs) | Does **not** replace Caddy on 80/443 for the Data Pod. |
| Hub Protocol, proposals, workspace, agent memory | **Synap backend** | All agent state (sessions, conversations, tasks) lives here. |
| Local tool preferences (OpenRouter, etc.) | **Eve secrets** | `.eve/secrets/secrets.json` — distinct from Synap internal IS routing. |

## Monorepo packages (hestia-cli) v2.0

| Package | Responsibility |
|---------|----------------|
| `@eve/cli` | `eve` entrypoint, Commander + Clack |
| `@eve/dna` | Setup profile schema, secrets, USB manifest |
| `@eve/brain` | `runBrainInit`, Synap delegation, Ollama/Postgres/Redis helpers |
| `@eve/legs` | Traefik, gateway, tunnel helpers |
| `@eve/arms` | OpenClaw install & wiring |
| `@eve/eyes` | RSSHub, database UI runners |
| `@eve/builder` | **Hermes Agent deployment**, `coder` engine binaries (claudecode / opencode / openclaude — one selected via `secrets.builder.codeEngine`), Dokploy optional, subprocess orchestrator. | `@eve/cli-kit` | Shared flags, JSON output |
| `packages/install/src/templates/ai-chat-docker-compose.yml` | Docker Compose template for chat UIs (Open WebUI, LobeChat) — no dedicated `@eve/*` package; managed as a compose profile |

### @eve/builder — v2.0 changes

The Builder organ has been refactored to replace the OpenClaude orchestration layer with **Hermes Agent** as the sole orchestrator (while adding OpenClaude as an optional codegen tool, not as an agent framework):

| Old (v1.0) | New (v2.1) |
|------------|-----------|
| Orchestrates OpenCode / OpenClaude / Claude Code directly | Deploys Hermes Agent container; one **`coder`** Hub identity drives one of three engine binaries selected by `secrets.builder.codeEngine` |
| Direct command execution | Hermes polls Synap (TaskPoller + IntentPoller) for delegated tasks AND background intents |
| No sub-agent model | Hermes spawns the active engine binary via local subprocess (TaskExecutor + engine adapter) |
| Optional Dokploy | Optional Dokploy (first-class citizen when enabled) |
| No workspace template | Eve seeds the **Builder Workspace** on install/update via `POST /api/hub/workspaces/from-definition` |

### OpenClaw (`@eve/arms`) — v2.0 changes

| Old (v1.0) | New (v2.0) |
|------------|-----------|
| OpenClaw maintains its own conversation history, user profiles, logs | OpenClaw fetches memory from and writes back to Synap via REST API |
| OpenClaw may handle coding tasks | OpenClaw does NOT do coding — creates Tasks in Synap for Hermes to handle |
| Local SQLite / vector storage | All persistent state in Synap PostgreSQL |

## State on disk (operator mental model)

| Path | Role |
|------|------|
| `.eve/setup-profile.json` | Last `eve setup` choices (profile, network hints, Synap install prefs). |
| `.eve/secrets/secrets.json` | Merged secrets: `synap.*`, `ai.*`, `inference.*`, `builder.*`, `arms.*` |
| `/opt/synap-backend` (typical) | Synap **backend** checkout — **`synap install`** runs with `cwd` = repo root and `SYNAP_DEPLOY_DIR` = `…/deploy`. |

Paths under **`.eve/`** are relative to **where you ran `eve`** (often `/opt/eve`). They are **not** the Synap repo root.
