# Eve OS Home вҖ” Design Spec

**Date:** 2026-05-05
**Status:** Design specification. Implementation pending Phase 2 of [Eve OS Vision](./eve-os-vision.mdx).

This document is concrete enough that a developer can implement the OS home page end-to-end after one read.

---

## 1. Design intent

The home is what the operator sees the moment they open Eve. It must communicate, in under one second:

> "This is **my** OS. These are **my** apps. Everything I need is here."

Three feelings to deliver:

1. **Calm** вҖ” generous whitespace, no busywork, low chroma
2. **Sovereignty** вҖ” minimal Eve branding, the operator's apps are the heroes
3. **Intelligence-aware** вҖ” AI surfaces (OpenWebUI primarily) have a subtle emerald accent, distinguishing them from regular apps without screaming

What it is NOT: a SaaS dashboard with KPIs and charts. There are no metrics on this page. Metrics live in `/settings`. The home is *only* the launcher.

## 3. Component breakdown

### 3.1 `TopBar` (`/app/(home)/components/top-bar.tsx`)

Single horizontal row. 64px tall. Sticky.

| Element | Position | Behavior |
|---|---|---|
| `SearchBar` | Center | Filters app grid in real time. Cmd+K opens it from anywhere. |
| `SettingsButton` (gear icon) | Right | Links to `/settings` (where current dashboard pages live) |
| `UserAvatar` | Far right | Click вҶ’ menu: profile, sign out, switch pod |

Search behavior:
- Empty state: placeholder text "Search apps, settings, or commandsвҖҰ"
- Typing: filters tiles in grid live (debounced 80ms)
- No results: shows inline "No apps match. [Browse marketplace вҶ’]"
- Cmd+K from anywhere on the page brings focus

### 3.2 `Greeting` (`/app/(home)/components/greeting.tsx`) вҖ” optional

Just below TopBar. 32вҖ“48px of vertical breathing room above and below.

- **Line 1:** "Good morning/afternoon/evening, {firstName}" вҖ” fontSize 24px, fontWeight 300 (light)
- **Line 2:** Date + time вҖ” fontSize 14px, neutral-500

Hide on screens narrower than 768px (mobile gets straight to apps).

### 3.3 `AppGrid` (`/app/(home)/components/app-grid.tsx`)

The core. Responsive grid of `AppTile` components plus one `AddAppTile` at the end.

Grid sizing:

| Breakpoint | Columns | Tile size | Gap |
|---|---|---|---|
| Mobile (`<640px`) | 3 | 96px | 12px |
| Tablet (`640pxвҖ“1024px`) | 4 | 112px | 16px |
| Desktop (`>1024px`) | 6 | 128px | 20px |
| Wide (`>1440px`) | 8 | 128px | 24px |

Tiles are square. Drag-and-drop reordering enabled (uses `dnd-kit`). Reorder updates `pinnedAppIds` order in user preferences via `POST /api/preferences`.

## 4. Data model

### 4.1 Page state

```typescript
type AppSource = 'cp_marketplace' | 'local_component' | 'pinned_default';

type App = {
  id: string;                    // stable across sources
  source: AppSource;
  name: string;
  description?: string;
  iconUrl?: string;              // remote URL (preferred)
  iconLucide?: string;           // fallback Lucide icon name
  url: string;                   // where clicking opens
  category: 'ai' | 'productivity' | 'communication' | 'dev' | 'other';
  status: 'online' | 'degraded' | 'offline' | 'unknown';
  isAI?: boolean;                // triggers emerald-accent variant
  isEntitled: boolean;           // false = locked, show "Upgrade" tile variant
  installCount?: number;         // for sort
};

type ChannelStatus = {
  id: 'telegram' | 'whatsapp' | 'signal' | 'matrix' | 'discord' | 'slack' | 'voice';
  name: string;
  status: 'online' | 'offline' | 'degraded';
  configured: boolean;           // false = greyed out, "Connect" hint
};

type HomeState = {
  apps: App[];
  pinnedAppIds: string[];        // user's chosen order; unpinned apps appear at end
  searchQuery: string;
  channels: ChannelStatus[];
  isLoadingMarketplace: boolean;
  marketplaceError?: string;
};
```

### 4.2 Data sources merged

The marketplace is the **unified catalog**. Four `app_type` shapes feed the home:

- `url` вҖ” external website link. Tile opens in a new tab.
- `eve_component` вҖ” surfaces an Eve-managed local component (OpenWebUI, Dev Agent). `metadata.componentId` binds it to a component in the Eve registry. Local component status (online/offline) hydrates the tile's status dot. **OpenWebUI** is pinned as "Chat" by default; **Dev Agent** carries `metadata.engineOptions: ["openclaude","opencode","claude-code"]` so install can prompt for engine choice.
- `workspace_pack` вҖ” installs into Synap pod. Typically does NOT appear on the OS home (lens inside Synap, not a launchable app); decide per-pack via metadata.
- `bundle` вҖ” composite; resolves to constituent apps' tiles.

```typescript
async function loadHomeApps(): Promise<App[]> {
  const [marketplaceApps, localComponents] = await Promise.allSettled([
    fetchMarketplaceApps(cpUserToken),     // Phase 1 endpoint вҖ” entitled apps
    fetchLocalComponents(),                // /api/components вҶ’ live status hydration
  ]);

  // For eve_component apps: cross-reference componentId with localComponents
  // to hydrate the live status dot. The marketplace is authoritative for
  // catalog presence; local components are authoritative for runtime status.
  return mergeAndHydrate(marketplaceApps, localComponents);
}
```

Marketplace failure (network, auth expired) вҶ’ fall back to local components only with a small banner: "Couldn't reach marketplace. Showing local apps. [Retry]". This keeps Eve usable when CP is unreachable.

### 4.3 Pinning persistence

User's pin order stored in `~/.eve/preferences.json` under `home.pinnedAppIds: string[]`. New apps go to end of list. Local components (Synap, OpenWebUI) are pinned by default at the front.

API: `GET /api/preferences/home`, `PUT /api/preferences/home`. New endpoints in Eve dashboard's API routes.

## 5. Interaction flows

### 5.1 First load (returning user)

1. Page mounts вҶ’ reads `~/.eve/preferences.json` for pin order (instant)
2. Renders skeleton tiles in pin order (instant)
3. Two parallel fetches:
   - Local components via `/api/components` (~50ms)
   - Marketplace via CP `/api/marketplace/apps` (~200ms)
4. As each resolves, real tiles fade in (200ms)
5. Channel statuses load via `/api/channels/status` (background, populates ChannelBar)

### 5.2 First load (brand-new user, no apps)

1. Page mounts вҶ’ no prefs, no apps known
2. Skeletons for ~1.5s while local components query
3. If query returns empty AND marketplace returns empty entitlements вҶ’ render `EmptyState`
4. If anything returns non-empty вҶ’ render normally with what we have

### 5.3 Search

1. User focuses search (or hits Cmd+K)
2. Types вҶ’ debounced 80ms
3. Filter `apps` by `name` includes / `description` includes / `category` matches
4. Grid reflows with filtered set
5. ESC clears, returns to full grid

### 5.4 Pin / unpin / reorder

1. User long-presses or right-clicks a tile вҶ’ contextual menu
2. Selects "Pin" / "Unpin" вҶ’ state updates, position changes
3. Or: drags tile вҶ’ drop into new position
4. `PUT /api/preferences/home` fires (debounced 500ms in case of multiple rapid drags)

### 5.5 Install from marketplace

1. User clicks `AddAppTile`
2. Routes to `/marketplace` (or opens external CP marketplace URL in new tab)
3. User picks an app, completes install (entitlement granted in CP)
4. User returns to home вҶ’ grid auto-refetches вҶ’ new app appears, animates in (scale 0.8 вҶ’ 1.0, 240ms ease-out)

### 5.6 OAuth handshake (first time only)

1. User opens Eve dashboard for the first time
2. No `cp.userToken` in `~/.eve/secrets.json`
3. Marketplace fetch fails with 401 вҶ’ trigger auth flow
4. Redirect to `https://cp.synap.sh/oauth/eve?return=...`
5. User authorizes in CP
6. Returns to Eve with code, exchanged for token, stored
7. Reload, marketplace works

While token is missing, the home still renders local components. The marketplace section just doesn't populate. Banner: "Sign in to see your apps from the Synap marketplace. [Sign in]"

## 8. Accessibility

- All tiles are `<a>` elements with `href` (real navigation, not div onClick)
- Keyboard: Tab cycles tiles, Enter activates, arrow keys navigate grid (custom handler with `dnd-kit` accessibility helpers)
- Screen readers: each tile has `aria-label="Open {name}, status {status}"`
- Search: `<input type="search">` with proper labels and `aria-controls` pointing at the grid
- Status dots: `aria-hidden` (purely visual), screen readers get the textual status via `aria-label`
- Color contrast: all text-on-background combinations verified вүҘ4.5:1 in both modes
- `prefers-reduced-motion`: disables tile lift hover, fades become instant

## 10. Open design questions (deferrals)

These can be answered later without blocking implementation:

1. **Greeting line wording.** "Good evening" vs. system message ("3 unread proposals waiting") vs. user-customizable. Default to neutral greeting; revisit when proactive AI surfaces here.
2. **Search depth.** v1: name + description matching only. v2: search across settings, marketplace catalog, even Synap entities ("find my notes about X")? Defer to Phase 4.
3. **Tile sizes.** All tiles are uniform 128px. Future: 2x1 widget tiles for at-a-glance info (next event, task count). Defer.
4. **Notifications.** A `notifications` indicator on the user avatar? Not in v1. Notifications surface in Synap, not Eve.
5. **Wallpapers / theming.** Umbrel has wallpapers. Out of scope for v1; revisit if Eve becomes user-skin-able.
6. **App categories / sections.** v1 is one flat grid. Future: collapsible sections by category? Probably not вҖ” pin order is the user's organization tool.

## 11. Reference: what this is NOT

To prevent scope creep during implementation:

- вқҢ Not a workflow editor (that's `/channels`)
- вқҢ Not a metrics dashboard (that's `/settings/components`)
- вқҢ Not a chat UI (that's OpenWebUI, accessed via the Chat tile)

- вқҢ Not a settings panel (that's `/settings`, gear icon top-right)
