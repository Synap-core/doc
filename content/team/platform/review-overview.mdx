- **Current state** — what exists, code references, quality assessment

- **Integration gaps** — how this connects to other features

## The 5 Missing Bridges

Cross-cutting integration gaps that block the full product vision:

| # | Bridge | Status | Impact |
|---|--------|--------|--------|
| 1 | **Native widget runtime** | **DONE** | Widgets are real React components |
| 2 | **Event-triggered skills** | **DONE** — skill_triggers table, IS executor, Hub Protocol, SkillsManager UI | The "magic moment" — AI reacts to data changes |
| 3 | **Profile packs** | **Phases A-C DONE** — Phase D (onboarding) pending | Composable workspace customization |
| 4 | **Remote execution tool** | Research only | AI runs commands on pod (DevOps use case) |
| 5 | **Secret request flow** | **DONE** — `request_secret` tool + proposal flow + VaultRequestCard | AI requests credentials via vault dialog |

*Skill editor in Browser was listed as bridge #5 but is more of a convenience feature than a blocking gap. Replaced with profile packs which has higher product impact.*

## Priority Order (Updated 2026-03-25)

Based on impact, dependency chains, and what's already built:

1. ~~**Widget System** (01)~~ — **DONE**. Native runtime implemented.
2. ~~**Profile Packs** (02 + PLAN-02)~~ — **Phases A-C DONE**. Phase D (onboarding integration) is separate.
3. ~~**Event-Triggered Skills** (06 + PLAN-03)~~ — **DONE**. `skill_triggers` table, IS executor, Hub Protocol endpoints, SkillsManager + SkillEditor browser UI, `POST /tasks/skill-trigger` IS route.
4. ~~**Vault AI Phase 1** (07)~~ — **DONE**. `request_secret` IS tool, `POST /hub/vault/request` endpoint, `VaultRequestCard` browser component wired into proposal feed.
5. ~~**Unified Relation Model** (PLAN-04)~~ — **DONE**. `syncPropertyToRelations` + `syncRelationToProperty`, `seed-property-relation-mappings`, `skill_triggers` schema. Auto-sync fully wired in entities + relations routers.
6. **Onboarding Flow** — Users need to get into the product smoothly (separate plan: `vast-floating-dusk.md`).
7. **Terminal Bridge** (04) — Unlocks DevOps workspace + AI debugging capabilities.
8. **Notifications full system** (05 + PLAN-05) — Remaining: DB notifications table, registry, per-channel adapters, Telegram inline actions.
