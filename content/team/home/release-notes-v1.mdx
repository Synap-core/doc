# Release Notes: Unified RSS Feed Infrastructure v1.0

**Release Date:** 2026-04-14  
**Version:** 1.0.0  
**Status:** Production Ready

---

## 🎯 Overview

This release introduces a **complete backend-agnostic RSS feed infrastructure** for Synap, enabling personalized content delivery from external sources (RSS feeds, social media, news) directly into user feed channels. The system is built on a unified protocol that consolidates all feed delivery through a single `DeliveryService`, eliminating duplication and ensuring consistency.

### Key Highlights
- **Backend-Agnostic**: Works with Control Plane RSSHub proxy, self-hosted RSSHub, direct RSS fetching, or custom providers
- **Unified Protocol**: Single `DeliveryService` for all feed channel posting (signals, proactive AI, notifications)
- **AI-Powered**: DeepSeek-based classification with user context (interests, persona, campaigns)
- **Production-Ready**: 12,000+ lines of code, comprehensive tests, full documentation

## ✨ New Features

### 1. Backend-Agnostic Feed Service (`@synap/feed-service`)

**New Package**: `synap-backend/packages/feed-service/`

A pluggable feed aggregation system supporting multiple provider backends:

- **DirectRSSProvider**: Direct HTTP fetch for RSS/Atom/JSON feeds
- **RSSHubProvider**: Self-hosted RSSHub instance with 1000+ routes
- **CPProxyProvider**: Control Plane managed RSSHub proxy
- **CustomProvider**: User-defined RSS backends

**Features:**
- Pluggable provider architecture via `IFeedProvider` interface
- Three classifier types: IS (AI-powered), Keyword (local), Noop (pass-through)
- Factory pattern for dependency injection
- Zod schema validation for all configurations
- Comprehensive error handling with retry logic

**Configuration:**
```bash
RSS_PROVIDER_TYPE=cpproxy|rsshub|direct|custom
RSSHUB_URL=http://rsshub:1200
CP_RSSHUB_PROXY_URL=https://cp.synap.io/api/rsshub-proxy
```

### 2. Unified DeliveryService

**Enhanced**: `synap-backend/packages/api/src/services/DeliveryService.ts`

Consolidates all feed channel posting into a single service:

- **Single Protocol**: One `deliver()` method for all surfaces (feed, chat, notification, email)
- **Proactive Integration**: Merged `postProactiveMessage()` logic into DeliveryService
- **Smart Features**:
  - Preference checking (proactiveAi.enabled, mutedUntil)
  - Deduplication (one per type per day)
  - Circuit breaker pattern for resilience
  - Retry logic with exponential backoff
  - Event emission (`proactive.post.completed`)
  - Socket.IO real-time broadcasts

**Usage:**
```typescript
const deliveryService = new DeliveryService();
await deliveryService.deliver({
  userId, workspaceId,
  content: { title, body, metadata },
  surfaces: [{
    type: "feed",
    proactiveOptions: {
      checkPreferences: true,
      deduplicate: true,
      proactiveType: "insight",
      emitEvents: true
    }
  }]
});
```

### 3. Hub Protocol Signals (7 Endpoints)

**New**: `synap-backend/packages/api/src/routers/hub-protocol/signals.ts`

Complete REST API for signal feed management:

| Endpoint | Description |
|----------|-------------|
| `POST /signals/fetch` | Fetch RSS feeds via CP proxy |
| `POST /signals/capture` | Capture signal → entity → feed channel |
| `POST /signals/classify` | AI classification with user context |
| `GET /signals/feed` | Get personalized signal feed |
| `GET /signals/context` | Get user personalization context |
| `POST /signals/subscriptions` | Manage RSS subscriptions |
| `POST /signals/batch` | Batch operations |

**Signal Capture Flow:**
```
RSS Item → Classify (IS) → Create Entity → Post to Feed Channel
```

### 5. Control Plane Provider Registry

**New**: `synap-control-plane-api/src/services/feed-provider-registry.ts`

Enterprise-grade RSS proxy with:

- **Multi-Provider Support**: RSSHub, FreshRSS, Miniflux, Custom
- **Health Monitoring**: Automatic health checks every 60 seconds
- **Load Balancing**: Weighted selection across healthy providers
- **Caching**: In-memory cache with 1-hour TTL
- **Rate Limiting**: 100 requests/hour per IP
- **Metrics Tracking**: Request counts, latency, error rates

**API Endpoints:**
```
GET  /api/feed-providers              # List providers
POST /api/feed-providers              # Register provider
GET  /api/feed-providers/:id/health   # Health check
GET  /api/rsshub-proxy/*              # Proxy to RSS (any route)
```

## 🔧 Improvements

### Architecture Improvements

| Before | After |
|--------|-------|
| 2 ways to post to feed (postProactiveMessage + DeliveryService) | **1 unified DeliveryService** |
| Signal capture didn't post to channels | **Signal capture → DeliveryService → Feed channel** |
| Duplicated preference logic | **Single preference check in DeliveryService** |
| Duplicated deduplication logic | **Single dedup in DeliveryService** |
| No retry logic in proactive post | **Circuit breaker + retry in DeliveryService** |
| CP-dependent only | **Backend-agnostic (4 provider types)** |

### Performance Improvements

- **Caching**: 1-hour TTL on CP RSSHub proxy
- **Batching**: Signal batch operations support
- **Lazy Loading**: Feed pagination with infinite scroll
- **Offline Support**: 15-minute cache in Relay app

### Developer Experience

- **Type Safety**: Full TypeScript with Zod validation
- **Documentation**: 600+ lines of documentation
- **Testing**: 2,000+ lines of integration tests
- **Backward Compatibility**: Deprecated APIs still work

## 🚀 Migration Guide

### For Existing Users

**1. Database Migration (Required):**
```bash
cd synap-backend/packages/database
pnpm drizzle-kit generate --name signal_feed_tables
pnpm drizzle-kit migrate
```

**2. Update Environment Variables (Optional):**
```bash
# Add to .env
RSS_PROVIDER_TYPE=cpproxy  # or rsshub, direct, custom
```

**3. Start RSSHub (Optional):**
```bash
cd synap-backend/deploy
docker compose --profile rsshub up -d
```

### For Self-Hosted Users

**New RSSHub Option:**
```bash
# In docker-compose.yml, add --profile rsshub
docker compose --profile rsshub up -d

# Update .env
RSS_PROVIDER_TYPE=rsshub
RSSHUB_URL=http://rsshub:1200
```

### For Developers

**Replace Old API:**
```typescript
// Old (deprecated)
import { postProactiveMessage } from "./utils/proactive-channel-post";
await postProactiveMessage({ userId, workspaceId, content, proactiveType });

// New (recommended)
import { DeliveryService } from "./services/DeliveryService";
const service = new DeliveryService();
await service.deliver({ userId, workspaceId, content, surfaces: [{ type: "feed" }] });
```
