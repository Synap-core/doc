# Stack & tools

Curated external references and short notes on the technologies Synap and Eve compose stacks use. **Behavioral truth** for Synap and Eve remains in the **backend** and **hestia-cli** repos.

When to use this space: **onboarding**, **debugging**, or **GTM** explaining "what runs under the hood." When not to: **API contracts** — use **[Platform](/team/platform/index)** and **[Architecture (public)](/docs/architecture/index)**.

---

## Target Users

- **Operators & infrastructure engineers** debugging deployment issues or onboarding new team members
- **GTM operations** explaining self-host stack components in sales or marketing materials
- **Eve contributors** needing quick reference to upstream docs for components they don't own

---

## Marketing

- **Purpose:** Authoritative short-form reference for "what runs under the hood" — not a blog, not a product page.
- **Scope:** Curated upstream links + 2–4 bullets on how Synap/Eve uses each technology + known caveats.
- **Out of scope:** Full upstream documentation, API contract specifications, product feature pages.

---

## Key User Stories

- As an **operator**, I need a quick reference for Docker-in-LXC pitfalls, sysctl errors, and which compose file to use.

- As a **GTM person**, I need to explain to stakeholders what third-party technologies power the self-host stack without reading 200-page upstream docs.

## Overview

This space is a **curated reference**, not a product spec. For each technology we:

1. Link to the **upstream documentation** (the behavioral truth).
2. Provide **2–4 bullets** on how Synap or Eve specifically uses it.
3. Call out **known pitfalls** and caveats.

**Three-layer integration model** (illustrated by RSSHub):
- **Control Plane** — proxy layer for rate-limited, provider-backed feeds
- **Data Pod** — RSS fetch worker, feed jobs, normalization
- **Intelligence Service** — classification and personalization of normalized items

---

## Architecture

Each technology page follows a **standard template**:

```
Technology overview (2–3 sentences)
```
