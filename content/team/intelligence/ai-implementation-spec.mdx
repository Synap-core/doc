# PART 1: VISION & ARCHITECTURE

## 1.1 Executive Vision

### What We're Building

**Synap Intelligence Service** is an **infinite conversation system** where users engage in one continuous dialogue that can branch into parallel explorations, merge insights, and maintain context across thousands of messages. The AI acts as a **persistent intelligent partner** integrated into the event-sourced backend, not a stateless chatbot.

### Key Differentiators

1. **Infinite Chat**: One conversation thread per context (workspace/personal) that never resets
2. **Git-like Branching**: Fork conversations to explore options, merge back with conflict resolution
3. **Meta-Agent**: AI that orchestrates other AIs, ensuring seamless transitions at context limits
4. **Event-Driven**: AI subscribes to backend events and proposes actions proactively
5. **Entity-Aware**: AI understands user's entities (tasks, contacts, documents) and their relationships

### User Mental Model

**Not this**: "I'm talking to a chatbot that forgets our conversation every time I close the tab"

**But this**: "I have an intelligent assistant that remembers everything we've discussed, can explore multiple ideas in parallel, and helps me manage my knowledge graph"

**Problems:**

```
└──────┬──────┘

│   - Updates conversation state                         │
│   - Returns response + saves checkpoint                │

┌────────────────────────────────────────────────────────┐

│   - Stores AI response                                 │

└────────────────────────────────────────────────────────┘

│   USER branches conversation                           │
```

- ✅ AI maintains context across sessions
- ✅ Branching creates parallel explorations
- ✅ Meta-agent enables infinite scaling
- ✅ Event-driven enables proactive AI
- ✅ Efficient context management (tiered storage)

## 1.3 Core Architectural Principles

### Principle 1: Conversation as First-Class Primitive

**Every feature builds on the conversation thread:**

- Thread = persistent state container
- Messages = events in conversation log
- Context = derived from thread state
- Branches = lightweight thread forks
- AI responses = thread mutations

**Implementation:** LangGraph StateGraph with PostgreSQL checkpointing

---

### Principle 2: Event-Driven Communication

**Everything flows through events:**

- User actions → Events
- AI analyses → Events
- Tool executions → Events
- Approvals → Events

**Pattern:** Event Sourcing + CQRS

- Write: Append-only event log
- Read: Materialized views from events

**Implementation:** PostgreSQL LISTEN/NOTIFY + Worker pool

---

### Principle 3: Hybrid Context Management

**Not "everything in context" OR "retrieve on demand"**
**But:** Tiered storage with intelligent retrieval

**Tier 1: Hot Context** (always loaded)

- Last 5-10 messages (verbatim)
- User preferences (name, communication style)
- Active entities (current task, open docs)
- Thread metadata (branch parent, merge conflicts)

**Tier 2: Warm Context** (summarized)

- Last 50-100 messages (summarized by topic)
- Key decisions made
- Important entities mentioned

**Tier 3: Cold Context** (retrieved on demand)

- Full conversation history (vector search)
- All user entities (semantic search)
- Long-term memory (Mem0)

**Implementation:** LangChain ConversationSummaryBufferMemory + pgvector

---

### Principle 4: Separation of Concerns

**Intelligence Service is SEPARATE from Backend:**

- Backend = data authority (entities, threads, events)
- Intelligence Service = reasoning authority (AI decisions)
- Communication = Hub Protocol (OAuth2 + REST) + Event Bus

**Why?**

- Licensing: Backend (open-source) vs Intelligence (proprietary)
- Scaling: AI service needs GPUs, backend doesn't
- Security: AI can't access database directly
- Flexibility: Swap AI providers without backend changes

**Implementation:** Hybrid monorepo (shared packages, separate deployments)

---

### Principle 5: Human-in-the-Loop

**AI proposes, user approves:**

- Read operations = auto-execute (search, analyze)
- Write operations = require approval (create, update, delete)
- Granular permissions = per entity type + field

**Pattern:** Proposal → Review → Execute

- AI generates proposal with rationale
- Backend stores in approvals table
- User sees in UI
- On approval, execute via event chain

**Implementation:** Event chain with worker subscribers (already exists in backend)

**Service Communication:**

- ✅ **Hono** - Intelligence Service HTTP server
- ✅ **Hub Protocol** - Service-to-service API
- ✅ **OAuth2 (Ory Hydra)** - Service authentication

### New Technologies (Add for Features)

**Event Bus:**

- 🆕 **PostgreSQL LISTEN/NOTIFY** - Simple event pub/sub
  - Why: Already have PostgreSQL
  - Alternative: Redis Streams (if scale requires)
  - NOT Kafka (overkill for early stage)

**Context Management:**

- 🆕 **LangChain ConversationSummaryBufferMemory** - Hybrid context
  - Keep recent messages verbatim
  - Summarize older messages
  - Better than pure summarization or pure retrieval

**Model Router:**

- 🆕 **OpenRouter** - Multi-model switching
  - Replace direct Anthropic calls
  - Enable model routing (cheap for simple, expensive for complex)
  - Fallback strategy (if Claude down, use GPT-4)

**State Management:**

- 🆕 **LangGraph Checkpointer (PostgreSQL)** - Conversation state
  - Persist thread state between requests
  - Enable time-travel (rewind conversation)
  - Support branching (fork from checkpoint)

### Tool Registry (Functional)

**File:** `apps/intelligence-hub/src/tools/registry.ts`

**5 Tools Implemented:**

1. **`vector_search`** (Context tool - read-only)
   - Searches user entities by semantic similarity
   - Calls Hub Protocol: `GET /api/hub/users/:userId/entities`
   - Returns top 5 results

2. **`memory_search`** (Context tool - read-only)
   - Searches long-term memory (Mem0)
   - Returns user preferences, facts
   - Auto-executes (no approval needed)

3. **`create_entity`** (Action tool - requires approval)
   - Creates task/contact/meeting/idea/project
   - Returns proposal in proposal mode
   - Executes via Hub Protocol in execution mode

4. **`create_document`** (Action tool - requires approval)
   - Creates document/note
   - Proposal → Approval → Execute pattern

5. **`create_branch`** (Action tool - requires approval)
   - **STUB**: Returns branchId but doesn't actually fork thread
   - Needs full implementation

**Tool Pattern:**

```typescript
const create_entity = tool({
  description: "Create a new entity",
  parameters: z.object({
    type: z.enum(["task", "contact", "meeting", "idea", "project"]),
    title: z.string(),
    description: z.string().optional(),
  }),
  execute: async (args, context) => {
    if (context.proposalMode) {
      return {
        proposed: true,
        action: "create_entity",
        description: `Create ${args.type}: "${args.title}"`,
        args,
      };
    }

    // Execution mode: Call Hub Protocol
    return await hubProtocol.createEntity({
      userId: context.userId,
      ...args,
    });
  },
});
```

**Strengths:**

## 2.3 Critical Gaps 🔴

### 1. No Conversation State Management

**Current:** Each request is independent, no thread continuity

**Need:**

- Thread state storage (PostgreSQL)
- LangGraph checkpointing (save conversation state)
- Context window tracking
- Message history management

**Impact:** Can't build infinite chat without this

---

### 2. No Meta-Agent

**Current:** Basic context injection (`assembleContext()`)

**Need:**

- Conversation analyzer (extract role/tone/decisions)
- Prompt synthesizer (generate handoff prompts)
- Seamlessness validator (test new AI instance)
- Rotation trigger (at 70% context window)

**Impact:** Can't scale beyond ~8K tokens without meta-agent

---

### 3. No Event Bus

**Current:** HTTP request/response only

**Need:**

- Event publishing (PostgreSQL LISTEN/NOTIFY)
- Event subscription (worker pool)
- Event schemas (TypeScript types)
- Event handlers (AI reacts to entity changes)

**Impact:** Can't build event-driven AI or proactive suggestions

---

### 4. No Approval Flow in Backend

**Current:** Intelligence Service generates proposals, but backend doesn't handle them

**Need:**

- Approvals table (store pending proposals)
- Approval endpoints (`/approve`, `/reject`)
- Execution logic (call Intelligence Service on approval)
- Frontend UI (show proposals)

**Impact:** Proposal system is broken (proposals generated but never executed)

---

### 5. No Branching Infrastructure

**Current:** `create_branch` tool returns stub branchId

**Need:**

- Thread forking (create new thread from parent)
- Branch metadata (parent, fork point, status)
- Context inheritance (copy relevant state)
- Merge operations (detect conflicts, resolve)
- Branch UI (visualize tree)

**Impact:** Can't implement parallel exploration

## 3.1 Priority 1: Infinite Chat with Thread Continuity

**Status:** 🔴 Missing (only request/response exists)  
**Phase:** MVP (April 2025)  
**Dependencies:** None  
**Complexity:** High (core architectural change)

---

### Functional Requirements

**FR-1.1: Persistent Conversation Thread**

- User can send unlimited messages in one continuous thread
- Thread persists across sessions (browser close/reopen)
- Thread ID remains stable (never changes)
- Each workspace/personal context has one thread

**FR-1.2: Message Ordering**

- Messages appear in chronological order
- No duplicate messages
- No lost messages (at-least-once delivery)

**FR-1.3: Context Continuity**

- AI remembers previous messages in thread
- AI can reference earlier parts of conversation
- Context is maintained even after days/weeks

**FR-1.4: Thread Metadata**

- Thread has: title, created_at, updated_at, message_count
- Title auto-generated from first message
- Title user-editable

**FR-1.5: Performance**

- Message send latency: <200ms (95th percentile)
- Thread load latency: <500ms (95th percentile)
- Supports threads with 10,000+ messages

### Technical Implementation

#### Data Models

**New Table: `conversation_threads`**

```typescript
// packages/database/src/schema/conversation-threads.ts

import {
  pgTable,
  uuid,
  text,
  timestamp,
  integer,
  jsonb,
} from "drizzle-orm/pg-core";

export const conversationThreads = pgTable("conversation_threads", {
  id: uuid("id").primaryKey().defaultRandom(),
  userId: uuid("user_id")
    .notNull()
    .references(() => users.id),
  workspaceId: uuid("workspace_id").references(() => workspaces.id),

  // Thread metadata
  title: text("title").notNull(),
  status: text("status").notNull().default("active"), // 'active' | 'archived'

  // Branching
  parentThreadId: uuid("parent_thread_id").references(
    () => conversationThreads.id
  ),
  forkPoint: integer("fork_point"), // Message index where branched

  // Statistics
  messageCount: integer("message_count").notNull().default(0),
  tokenCount: integer("token_count").notNull().default(0),

  // Timestamps
  createdAt: timestamp("created_at").notNull().defaultNow(),
  updatedAt: timestamp("updated_at").notNull().defaultNow(),
  lastMessageAt: timestamp("last_message_at"),

  // State snapshot (for quick context loading)
  stateSnapshot: jsonb("state_snapshot"), // { summary, recentMessages, entities }
});
```

**New Table: `thread_messages`**

```typescript
// packages/database/src/schema/thread-messages.ts

  id: uuid("id").primaryKey().defaultRandom(),

    .notNull()
    .references(() => conversationThreads.id),

  // Message content
  role: text("role").notNull(), // 'user' | 'assistant' | 'system'
  content: text("content").notNull(),

  // Ordering
```

```typescript
import { LangGraph, StateGraph, Checkpoint } from "@langchain/langgraph";
import { PostgresSaver } from "@langchain/langgraph-checkpoint-postgres";

export class ThreadManager {
  private checkpointer: PostgresSaver;
  private graphs: Map<string, StateGraph> = new Map();

  constructor(private db: Database) {
    this.checkpointer = new PostgresSaver(db);
  }

  /**
   * Get or create conversation graph for thread
   */
  async getGraph(threadId: string): Promise<StateGraph> {
    if (this.graphs.has(threadId)) {
      return this.graphs.get(threadId)!;
    }

    // Create new graph with checkpointing
    const graph = new StateGraph({
      channels: {
        messages: {
          value: (prev, next) => [...prev, ...next],
          default: () => [],
        },
        context: {
          value: (prev, next) => ({ ...prev, ...next }),
          default: () => ({}),
        },
      },
    });

    // Compile with persistent checkpointer
    const compiled = graph.compile({
      checkpointer: this.checkpointer,
    });

    this.graphs.set(threadId, compiled);
    return compiled;
  }

  /**
   * Add message to thread and get AI response
   */
  async processMessage(input: {
    threadId: string;
    message: string;
    userId: string;
  }): Promise<{ response: string; proposedActions: any[] }> {
    const graph = await this.getGraph(input.threadId);

    // Load current state
    const state = await graph.getState({
      configurable: { thread_id: input.threadId },
    });

    // Add user message
    const newMessages = [
      ...state.values.messages,
      { role: "user", content: input.message },
    ];

    // Update state
    await graph.updateState(
      { configurable: { thread_id: input.threadId } },
      { messages: newMessages }
    );

    // Run orchestrator
    const result = await this.orchestrator.executeWithContext({
      threadId: input.threadId,
      userId: input.userId,
      messages: newMessages,
      context: state.values.context,
    });

    // Save checkpoint
    await graph.updateState(
      { configurable: { thread_id: input.threadId } },
      {
        messages: [
          ...newMessages,
          { role: "assistant", content: result.content },
        ],
        context: result.updatedContext,
      }
    );

    return {
      response: result.content,
      proposedActions: result.proposedActions,
    };
  }

  /**
   * Get thread state snapshot for quick loading
   */
  async getStateSnapshot(threadId: string): Promise<{
    summary: string;
    recentMessages: any[];
    tokenCount: number;
  }> {
    const graph = await this.getGraph(threadId);
    const state = await graph.getState({
      configurable: { thread_id: threadId },
    });

    const messages = state.values.messages;
    const recentMessages = messages.slice(-10); // Last 10 messages

    // Summarize older messages
    const olderMessages = messages.slice(0, -10);
    const summary = await this.summarizeMessages(olderMessages);

    return {
      summary,
      recentMessages,
      tokenCount: this.countTokens(messages),
    };
  }
```

```typescript
// packages/ai-conversation/src/thread-manager.ts

export class ThreadManager {
  /**
   * Fork thread at specific checkpoint
   */
  async forkThread(input: {
    parentThreadId: string;
    branchThreadId: string;
    forkPoint: number;
  }): Promise<void> {
    const parentGraph = await this.getGraph(input.parentThreadId);

    // Get state at fork point
    const checkpoints = await parentGraph.getStateHistory({
      configurable: { thread_id: input.parentThreadId },
    });

    // Find checkpoint at fork point (by message count)
    const forkCheckpoint = checkpoints.find(
      (cp) => cp.values.messages.length === input.forkPoint
    );

    if (!forkCheckpoint) {
      throw new Error(`No checkpoint found at fork point ${input.forkPoint}`);
    }

    // Create branch graph with copied state
    const branchGraph = await this.getGraph(input.branchThreadId);
    await branchGraph.updateState(
      { configurable: { thread_id: input.branchThreadId } },
      forkCheckpoint.values
    );
  }

  /**
   * Merge branch back to parent
   */
  async mergeThreads(input: {
    parentThreadId: string;
    branchThreadId: string;
  }): Promise<{ messageCount: number }> {
    const parentGraph = await this.getGraph(input.parentThreadId);
    const branchGraph = await this.getGraph(input.branchThreadId);

    // Get current states
    const parentState = await parentGraph.getState({
      configurable: { thread_id: input.parentThreadId },
    });
    const branchState = await branchGraph.getState({
      configurable: { thread_id: input.branchThreadId },
    });

    // Find divergence point
    const divergenceIndex = this.findDivergence(
      parentState.values.messages,
      branchState.values.messages
    );

    // Get branch-only messages
    const branchOnlyMessages =
      branchState.values.messages.slice(divergenceIndex);

    // Append to parent
    await parentGraph.updateState(
      { configurable: { thread_id: input.parentThreadId } },
      {
        messages: [
          ...parentState.values.messages,
          { role: "system", content: `[Merged from branch]` },
          ...branchOnlyMessages,
        ],
      }
    );

    return { messageCount: branchOnlyMessages.length };
  }

  /**
   * Detect conflicts between parent and branch
   */
  async detectConflicts(input: {
    parentThreadId: string;
    branchThreadId: string;
    forkPoint: number;
  }): Promise<Conflict[]> {
    const conflicts: Conflict[] = [];

    // Get states
    const parentState = await this.getState(input.parentThreadId);
    const branchState = await this.getState(input.branchThreadId);

    // Compare entity modifications
    const parentEntities = this.extractEntities(
      parentState.values.messages,
      input.forkPoint
    );
    const branchEntities = this.extractEntities(branchState.values.messages, 0);

    for (const [entityId, branchValue] of Object.entries(branchEntities)) {
      if (parentEntities[entityId]) {
        // Same entity modified in both threads
        if (!deepEqual(parentEntities[entityId], branchValue)) {
          conflicts.push({
            type: "entity_modified",
            entityId,
            parentValue: parentEntities[entityId],
            branchValue,
          });
        }
      }
    }

    // Compare context divergence
    const semanticSimilarity = await this.compareContexts(
      parentState.values.context,
      branchState.values.context
    );

    if (semanticSimilarity < 0.7) {
      conflicts.push({
        type: "context_diverged",
        parentValue: parentState.values.context,
        branchValue: branchState.values.context,
      });
    }

    return conflicts;
  }
}
```

```typescript
import { encoding_for_model } from "tiktoken";

export class TokenCounter {
  private encoder = encoding_for_model("gpt-4"); // Claude uses similar tokenization

  /**
   * Count tokens in text
   */
  countText(text: string): number {
    return this.encoder.encode(text).length;
  }

  /**
   * Count tokens in message
   */
  countMessage(message: { role: string; content: string }): number {
    // Add overhead for message structure
    const overhead = 4; // <|start|>role<|message|>content<|end|>
    return this.countText(message.content) + overhead;
  }

  /**
   * Count tokens in entire thread

  countThread(messages: any[]): number {
    return messages.reduce((sum, msg) => sum + this.countMessage(msg), 0);
  }

  /**
   * Check if thread needs rotation
   */
  needsRotation(tokenCount: number, maxTokens: number = 100000): boolean {
    return tokenCount >= maxTokens * 0.7; // 70% threshold
```

```typescript
    // 2. Tier 1: Last 10 messages (verbatim)
    const hotMessages = allMessages.slice(-10);
    const hotTokens = this.tokenCounter.countThread(hotMessages);

    // 3. Tier 2: Messages 11-100 (summarized)
    const warmMessages = allMessages.slice(-100, -10);
    const warmSummary = await this.summarizeMessages(warmMessages);
    const warmTokens = this.tokenCounter.countText(warmSummary);

    // 4. Tier 3: Older messages (vector search if needed)
    const coldMessages: any[] = [];
    let coldTokens = 0;

    const remainingTokens = input.maxTokens - hotTokens - warmTokens;
    if (remainingTokens > 1000) {
      // Retrieve relevant older messages
      coldMessages = await this.retrieveRelevantMessages({
        threadId: input.threadId,
        query: hotMessages[hotMessages.length - 1].content, // Use last message as query
        excludeRecent: 100,
        maxResults: 5,
      });
      coldTokens = this.tokenCounter.countThread(coldMessages);
    }

    return {
      hotContext: hotMessages,
      warmContext: warmSummary,
      coldContext: coldMessages,
      totalTokens: hotTokens + warmTokens + coldTokens,
    };
  }

  /**
   * Summarize messages using LLM
   */
  private async summarizeMessages(messages: any[]): Promise<string> {
    if (messages.length === 0) return "";

    const prompt = `Summarize the following conversation, preserving key facts, entities, and decisions:

${messages.map((m) => `${m.role}: ${m.content}`).join("\n\n")}

Summary:`;

    const summary = await this.llm.generate(prompt);
    return summary;
  }

  /**
   * Retrieve relevant messages from vector store
   */
  private async retrieveRelevantMessages(input: {
    threadId: string;
    query: string;
    excludeRecent: number;
    maxResults: number;
  }): Promise<any[]> {
    const results = await this.vectorStore.similaritySearch({
      query: input.query,
      filter: {
        threadId: input.threadId,
        sequenceNumber: { $lt: -input.excludeRecent }, // Exclude recent N messages
      },
      k: input.maxResults,
    });

    return results.map((r) => r.metadata.message);
  }
}
```

```typescript
// packages/ai-conversation/src/permission-checker.ts

export class PermissionChecker {
  constructor(private db: Database) {}

  /**
   * Check if action should auto-approve
   */
  async checkPermission(input: {
    userId: string;
    actionType: string;
    entityType?: string;
    fieldName?: string;
    confidence: number;
  }): Promise<"auto_approve" | "require_approval" | "deny"> {
    // Get user's permission rules
    const rules = await this.db.query.aiPermissions.findMany({
      where: and(
        eq(aiPermissions.userId, input.userId),
        eq(aiPermissions.actionType, input.actionType),
        input.entityType
          ? eq(aiPermissions.entityType, input.entityType)
          : undefined,
        input.fieldName
          ? eq(aiPermissions.fieldName, input.fieldName)
          : undefined
      ),
    });

    // No rules = require approval (safe default)
    if (rules.length === 0) return "require_approval";

    // Check most specific rule first
    for (const rule of rules) {
      // Evaluate conditions
      if (rule.conditions) {
        const conditionsMet = this.evaluateConditions(rule.conditions, {
          confidence: input.confidence,
        });
        if (!conditionsMet) continue;
      }

      return rule.permissionLevel;
    }

    return "require_approval";
  }

  private evaluateConditions(conditions: any, context: any): boolean {
    // Simple JSON Logic evaluation
    // Example: { confidence: { $gte: 0.9 } }
    for (const [field, logic] of Object.entries(conditions)) {
      if (typeof logic === "object") {
        for (const [op, value] of Object.entries(logic)) {
          switch (op) {
            case "$gte":
              if (context[field] < value) return false;
              break;
            case "$lte":
              if (context[field] > value) return false;
              break;
            case "$eq":
              if (context[field] !== value) return false;
              break;
            // Add more operators as needed
          }
        }
      }
    }
    return true;
  }
}
```

```typescript
export class MetaAgent {
  constructor(
    private llm: LLM, // Use GPT-4 or Claude Opus (specialized model for meta-tasks)
    private conversationAnalyzer: ConversationAnalyzer,
    private promptSynthesizer: PromptSynthesizer,
    private seamlessnessValidator: SeamlessnessValidator
  ) {}

  /**
   * Orchestrate context rotation for thread
   */
  async rotateContext(input: {
    threadId: string;
    currentMessages: any[];
    reason: "token_limit" | "topic_shift" | "user_request";
  }): Promise<{
    success: boolean;
    newSystemPrompt: string;
    validationScore: number;
    rotationId: string;
  }> {
    console.log(
      `[Meta-Agent] Rotating context for thread ${input.threadId}, reason: ${input.reason}`
    );

    // Step 1: Analyze conversation
    const analysis = await this.conversationAnalyzer.analyze({
      messages: input.currentMessages.slice(-100), // Last 100 messages
      fullHistory: input.currentMessages, // For deeper analysis
    });

    console.log(`[Meta-Agent] Analysis complete:`, analysis);

    // Step 2: Synthesize new prompt
    const newPrompt = await this.promptSynthesizer.synthesize({
      analysis,
      targetTokens: 5000, // Compress to ~5K tokens
    });

    console.log(
      `[Meta-Agent] Synthesized prompt (${newPrompt.tokenCount} tokens)`
    );

    // Step 3: Validate seamlessness
    const validation = await this.seamlessnessValidator.validate({
      oldMessages: input.currentMessages,
      newSystemPrompt: newPrompt.systemPrompt,
      testQueries: [
        "What were we just discussing?",
        "What's my main goal here?",
        "Summarize our conversation so far",
      ],
    });

    console.log(`[Meta-Agent] Validation score: ${validation.overallScore}`);

    if (validation.overallScore >= 0.8) {
      // Success: Store rotation record
      const rotationId = await this.storeRotation({
        threadId: input.threadId,
        oldPrompt: "...", // Store old state for rollback
        newPrompt: newPrompt.systemPrompt,
        analysis,
        validationScore: validation.overallScore,
      });

      return {
        success: true,
        newSystemPrompt: newPrompt.systemPrompt,
        validationScore: validation.overallScore,
        rotationId,
      };
    } else {
      // Failure: Try again with adjusted approach
      console.warn(
        `[Meta-Agent] Validation failed (${validation.overallScore}), retrying...`
      );

      // Adjust strategy (more verbatim context, less summarization)
      const adjustedPrompt = await this.promptSynthesizer.synthesize({
        analysis,
        targetTokens: 8000, // Less aggressive compression
        verbatimMessages: 20, // Keep more recent messages verbatim
      });

      // Re-validate
      const revalidation = await this.seamlessnessValidator.validate({
        oldMessages: input.currentMessages,
        newSystemPrompt: adjustedPrompt.systemPrompt,
        testQueries: validation.failedQueries,
      });

      if (revalidation.overallScore >= 0.7) {
        // Acceptable: Use adjusted prompt
        const rotationId = await this.storeRotation({
          threadId: input.threadId,
          newPrompt: adjustedPrompt.systemPrompt,
          analysis,
          validationScore: revalidation.overallScore,
        });

        return {
          success: true,
          newSystemPrompt: adjustedPrompt.systemPrompt,
          validationScore: revalidation.overallScore,
          rotationId,
        };
      } else {
        // Still failing: Keep current instance
        return {
          success: false,
          newSystemPrompt: "",
          validationScore: revalidation.overallScore,
          rotationId: "",
        };
      }
    }
  }

  /**
   * Optimize context for branch creation
   */
  async optimizeForBranch(input: {
    parentThreadId: string;
    branchPurpose: string;
    parentMessages: any[];
  }): Promise<{
    relevantContext: any[];
    systemPrompt: string;
  }> {
    // Analyze parent conversation
    const analysis = await this.conversationAnalyzer.analyze({
      messages: input.parentMessages,
    });

    // Extract only context relevant to branch purpose
    const relevantContext = await this.extractRelevantContext({
      analysis,
      branchPurpose: input.branchPurpose,
    });

    // Generate specialized prompt for branch
    const branchPrompt = await this.promptSynthesizer.synthesize({
      analysis,
      purpose: input.branchPurpose,
      focusAreas: relevantContext.topics,
    });

    return {
      relevantContext: relevantContext.messages,
      systemPrompt: branchPrompt.systemPrompt,
    };
  }

  /**
   * Detect merge conflicts semantically
   */
  async detectMergeConflicts(input: {
    parentThreadId: string;
    branchThreadId: string;
    forkPoint: number;
  }): Promise<Conflict[]> {
    // Get post-fork messages from both threads
    const parentMessages = await getMessages(input.parentThreadId, {
      after: input.forkPoint,
    });
    const branchMessages = await getMessages(input.branchThreadId);

    // Analyze conclusions of each thread
    const parentAnalysis = await this.conversationAnalyzer.analyze({
      messages: parentMessages,
      focusOn: "conclusions",
    });
    const branchAnalysis = await this.conversationAnalyzer.analyze({
      messages: branchMessages,
      focusOn: "conclusions",
    });

    // Compare conclusions semantically
    const similarity = await this.compareConclusionsSemantically({
      parentConclusions: parentAnalysis.conclusions,
      branchConclusions: branchAnalysis.conclusions,
    });

    const conflicts: Conflict[] = [];

    if (similarity < 0.5) {
      conflicts.push({
        type: "conclusion_diverged",
        description: "Parent and branch reached different conclusions",
        parentValue: parentAnalysis.conclusions,
        branchValue: branchAnalysis.conclusions,
        similarity,
      });
    }

    // Check entity-level conflicts
    const entityConflicts = await this.detectEntityConflicts({
      parentAnalysis,
      branchAnalysis,
    });

    return [...conflicts, ...entityConflicts];
  }
}
```

```typescript
// packages/ai-conversation/src/prompt-synthesizer.ts

export class PromptSynthesizer {
  constructor(private tokenCounter: TokenCounter) {}

  async synthesize(input: {
    analysis: ConversationAnalysis;
    targetTokens: number;
    purpose?: string;
    verbatimMessages?: number;
  }): Promise<{
    systemPrompt: string;
    tokenCount: number;
    structure: any;
  }> {
    // Build structured prompt in YAML (easier to modify programmatically)
    const structure = {
      role: {
        identity: input.analysis.role.description,
        tone: input.analysis.role.tone,
        communication_style: input.analysis.communication_style,
      },

      user_context: {
        background: input.analysis.user_context.background,
        goals: input.analysis.user_context.goals,
        constraints: input.analysis.user_context.constraints,
      },

      conversation_state: {
        active_topics: input.analysis.active_topics.map((t) => ({
          topic: t.topic,
          depth: t.depth,
        })),
        key_decisions: input.analysis.key_decisions.map((d) => d.decision),
        open_questions: input.analysis.open_questions.map((q) => q.question),
      },

      entities: {
        relevant: input.analysis.entities_mentioned
          .filter((e) => e.relevance === "high")
          .map((e) => ({ id: e.entity_id, type: e.entity_type })),
      },

      special_instructions: this.generateSpecialInstructions(input.analysis),
    };

    // Convert to natural language prompt
    const systemPrompt = this.structureToPrompt(structure);

    const tokenCount = this.tokenCounter.countText(systemPrompt);

    // If over target, compress
    if (tokenCount > input.targetTokens) {
      const compressed = await this.compressPrompt({
        structure,
        targetTokens: input.targetTokens,
      });

      return {
        systemPrompt: compressed.prompt,
        tokenCount: compressed.tokenCount,
        structure: compressed.structure,
      };
    }

    return { systemPrompt, tokenCount, structure };
  }

  private structureToPrompt(structure: any): string {
    return `
# ROLE DEFINITION
You are ${structure.role.identity}.
Tone: ${structure.role.tone}
Communication: ${structure.role.communication_style.user_prefers} responses at ${structure.role.communication_style.technical_level} level

# USER CONTEXT
Background: ${structure.user_context.background}

Goals:
${structure.user_context.goals.map((g) => `- ${g}`).join("\n")}

Constraints:
${structure.user_context.constraints.map((c) => `- ${c}`).join("\n")}

# CONVERSATION STATE
Active Topics:
${structure.conversation_state.active_topics.map((t) => `- ${t.topic} (${t.depth})`).join("\n")}

Key Decisions Made:
${structure.conversation_state.key_decisions.map((d) => `- ${d}`).join("\n")}

Open Questions:
${structure.conversation_state.open_questions.map((q) => `- ${q}`).join("\n")}

# RELEVANT ENTITIES
${structure.entities.relevant.map((e) => `- ${e.type}: ${e.id}`).join("\n")}

# SPECIAL INSTRUCTIONS
${structure.special_instructions}

---

Continue the conversation naturally from here. The user should not notice any discontinuity.
`;
  }

  private generateSpecialInstructions(analysis: ConversationAnalysis): string {
    const instructions: string[] = [];

    // Based on communication style
    if (analysis.communication_style.user_prefers === "concise") {
      instructions.push("Keep responses brief and to-the-point");
    }

    // Based on active topics
    if (analysis.active_topics.some((t) => t.depth === "deep")) {
      instructions.push("User expects detailed technical discussions");
    }

    // Based on open questions
    if (analysis.open_questions.length > 0) {
      instructions.push(
        `Prioritize addressing these open questions: ${analysis.open_questions.map((q) => q.question).join(", ")}`
      );
    }

    return instructions.join("\n");
  }

  private async compressPrompt(input: {
    structure: any;
    targetTokens: number;
  }): Promise<{ prompt: string; tokenCount: number; structure: any }> {
    // Compression strategies (apply in order until under target):
    // 1. Shorten goal descriptions
    // 2. Reduce number of key decisions (keep most recent)
    // 3. Summarize active topics
    // 4. Remove low-relevance entities

    const compressed = { ...input.structure };

    // Strategy 1: Shorten goals
    compressed.user_context.goals = compressed.user_context.goals.slice(0, 3);

    let prompt = this.structureToPrompt(compressed);
    let tokens = this.tokenCounter.countText(prompt);

    if (tokens <= input.targetTokens) {
      return { prompt, tokenCount: tokens, structure: compressed };
    }

    // Strategy 2: Keep only last 5 decisions
    compressed.conversation_state.key_decisions =
      compressed.conversation_state.key_decisions.slice(-5);

    prompt = this.structureToPrompt(compressed);
    tokens = this.tokenCounter.countText(prompt);

    if (tokens <= input.targetTokens) {
      return { prompt, tokenCount: tokens, structure: compressed };
    }

    // Strategy 3: Summarize topics
    compressed.conversation_state.active_topics =
      compressed.conversation_state.active_topics.slice(0, 3);

    prompt = this.structureToPrompt(compressed);
    tokens = this.tokenCounter.countText(prompt);

    return { prompt, tokenCount: tokens, structure: compressed };
  }
}
```

```typescript
// packages/ai-conversation/src/seamlessness-validator.ts

export class SeamlessnessValidator {
  constructor(
    private llm: LLM,
    private embeddingModel: EmbeddingModel
  ) {}

  async validate(input: {
    oldMessages: any[];
    newSystemPrompt: string;
    testQueries: string[];
  }): Promise<{
    overallScore: number;
    factualContinuity: number;
    tonalConsistency: number;
    roleMaintenance: number;
    failedQueries: string[];
    details: any[];
  }> {
    const results = await Promise.all(
      input.testQueries.map((query) => this.validateQuery(query, input))
    );

    const scores = {
      factualContinuity: this.avgScore(results.map((r) => r.factual)),
      tonalConsistency: this.avgScore(results.map((r) => r.tonal)),
      roleMaintenance: this.avgScore(results.map((r) => r.role)),
    };

    const overallScore =
      scores.factualContinuity * 0.5 + // Factual is most important
      scores.tonalConsistency * 0.3 +
      scores.roleMaintenance * 0.2;

    const failedQueries = results
      .filter((r) => r.factual < 0.7 || r.tonal < 0.7)
      .map((r) => r.query);

    return {
      overallScore,
      ...scores,
      failedQueries,
      details: results,
    };
  }

  private async validateQuery(
    query: string,
    context: { oldMessages: any[]; newSystemPrompt: string }
  ): Promise<{
    query: string;
    factual: number;
    tonal: number;
    role: number;
  }> {
    // Get response from old instance (last few messages)
    const oldContext = context.oldMessages.slice(-10);
    const oldResponse = await this.llm.generate({
      system: this.extractSystemPromptFromMessages(context.oldMessages),
      messages: [...oldContext, { role: "user", content: query }],
    });

    // Get response from new instance (with new prompt)
    const newResponse = await this.llm.generate({
      system: context.newSystemPrompt,
      messages: [...oldContext, { role: "user", content: query }],
    });

    // Compare responses
    const factual = await this.compareFactualAccuracy(oldResponse, newResponse);
    const tonal = await this.compareTone(oldResponse, newResponse);
    const role = await this.compareRole(oldResponse, newResponse);

    return { query, factual, tonal, role };
  }

  private async compareFactualAccuracy(
    oldResponse: string,
    newResponse: string
  ): Promise<number> {
    // Use embeddings to measure semantic similarity
    const oldEmbedding = await this.embeddingModel.embed(oldResponse);
    const newEmbedding = await this.embeddingModel.embed(newResponse);

    const cosineSimilarity = this.cosine(oldEmbedding, newEmbedding);

    // Also check for factual contradictions using LLM
    const contradictionCheck = await this.llm.generate(`
Compare these two responses to the same question. Do they contain factual contradictions?

Response A: ${oldResponse}
Response B: ${newResponse}

Answer with a score from 0.0 (complete contradiction) to 1.0 (fully consistent):
`);

    const score = parseFloat(contradictionCheck.match(/\d\.\d+/)?.[0] || "0.5");

    // Average semantic similarity and contradiction check
    return (cosineSimilarity + score) / 2;
  }

  private async compareTone(
    oldResponse: string,
    newResponse: string
  ): Promise<number> {
    const prompt = `
Rate how similar the tone and style of these two responses are (0.0 = completely different, 1.0 = identical):

Response A: ${oldResponse}
Response B: ${newResponse}

Consider: formality, brevity, friendliness, technical level.
Score:
`;

    const score = await this.llm.generate(prompt);
    return parseFloat(score.match(/\d\.\d+/)?.[0] || "0.5");
  }

  private async compareRole(
    oldResponse: string,
    newResponse: string
  ): Promise<number> {
    const prompt = `
Do these responses maintain the same role/persona (e.g., technical advisor, creative partner)?

Response A: ${oldResponse}
Response B: ${newResponse}

Score (0.0 = different roles, 1.0 = same role):
`;

    const score = await this.llm.generate(prompt);
    return parseFloat(score.match(/\d\.\d+/)?.[0] || "0.5");
  }

  private avgScore(scores: number[]): number {
    return scores.reduce((sum, s) => sum + s, 0) / scores.length;
  }

  private cosine(a: number[], b: number[]): number {
    const dot = a.reduce((sum, _, i) => sum + a[i] * b[i], 0);
    const magA = Math.sqrt(a.reduce((sum, val) => sum + val * val, 0));
    const magB = Math.sqrt(b.reduce((sum, val) => sum + val * val, 0));
    return dot / (magA * magB);
  }
}
```

```typescript
export class ProposalApprovalWorker {
  constructor(
    private eventBus: EventBus,
    private intelligenceService,
    private db
  ) {
    eventBus.subscribe("ai.proposal.approved", this.handleApproval.bind(this));
  }

  private async handleApproval(event: Events.ProposalApproved): Promise<void> {
    try {
      const result = await this.intelligenceService.executeAction({
        actionType: event.payload.actionType,
        actionPayload: event.payload.actionPayload,
        proposalMode: false, // EXECUTE
      });

      await this.db
        .update(aiProposals)
        .set({ status: "executed", executionResult: result });
    } catch (error) {
      await this.db
        .update(aiProposals)
        .set({ status: "failed", executionError: error.message });
    }
  }
}
```

# PART 6: CRITICAL DECISIONS SUMMARY

## Decision 1: Architecture Pattern

**Chosen:** Hybrid Monorepo with Separate Deployments

**Rationale:**

- Shared TypeScript types prevent sync issues
- Independent scaling (AI needs GPUs, backend doesn't)
- Security (AI can't access database directly)
- Single PR for full-stack features

**Alternatives Rejected:**

- ❌ Full merge (tight coupling, scaling issues)
- ❌ Complete separation (duplicate code, type drift)

---

## Decision 2: Event Bus Technology

**Chosen:** PostgreSQL LISTEN/NOTIFY for MVP, Redis Streams for V2 if needed

**Rationale:**

- No external dependency (already have PostgreSQL)
- Sufficient throughput (<10K events/sec)
- ACID guarantees (events in transaction)
- Simple to implement

**Alternatives:**

- Kafka: Overkill, expensive, complex
- Redis Streams: Good but requires separate service

---

## Decision 3: Context Management Strategy

**Chosen:** Tiered Hybrid (Hot/Warm/Cold)

**Rationale:**

- Balances information retention with token efficiency
- Hot = recent verbatim (accuracy)
- Warm = summarized (context)
- Cold = vector search (relevance)

**Alternatives Rejected:**

- ❌ Pure summarization (loses nuance)
- ❌ Pure retrieval (expensive, slow)

---

## Decision 4: Meta-Agent Model

**Chosen:** Specialized Model (GPT-4 or Claude Opus for meta-tasks)

**Rationale:**

- Better conversation analysis
- More accurate prompt synthesis
- Worth the extra cost (~$0.01 per rotation)

**Alternatives:**

- Same model as chat (cheaper but lower quality)
- Fine-tuned model (complex, requires training data)

---

## Decision 5: Approval Flow Pattern

**Chosen:** Event-Driven with Existing Worker System

**Rationale:**

- Backend already has event chain + workers
- Proposals publish events, workers execute
- Non-blocking (doesn't slow down chat)
- Auditable (event log)

**Alternatives Rejected:**

- ❌ Synchronous execution (slow, blocking)
- ❌ New job queue system (unnecessary complexity)

---

## Decision 6: LLM Provider

**Chosen:** OpenRouter with DeepSeek Primary, Claude Fallback

**Rationale:**

- DeepSeek very cheap ($0.28/M tokens)
- OpenRouter enables easy model switching
- Fallback to Claude if DeepSeek fails
- €19/month pricing sustainable

**Alternatives:**

- Direct Anthropic: More expensive, single vendor
- Self-hosted: Too early (requires GPU infrastructure)

---

## Decision 7: Branching Merge Strategy

**Chosen:** Manual Conflict Resolution in MVP, Semantic Detection in V1

**Rationale:**

- MVP: Branches independent (no merge = simpler)
- V1: Add semantic conflict detection (embedding similarity)
- User resolves conflicts via UI

**Alternatives:**

- Automatic merge: Risky, could lose data
- No merge ever: Limits utility

**Why:** Enables asynchronous processing, foundation for V1 features

**Implementation Tips:**

**Why:** Novel pattern, no reference implementations

**Why:** User trust depends on it

**Why:** Slow AI feels broken

**Targets:**

**Why:** Distributed system with AI (unpredictable)

## Common Pitfalls to Avoid

### Pitfall 1: Forgetting to Update Thread State

**Problem:** Messages stored in DB but not in LangGraph state → AI forgets

**Solution:** Always update both:

```typescript
// Wrong
await storeMessage(message);

// Right
await storeMessage(message);
await threadManager.updateState(threadId, { messages: [..., message] });
```

---

### Pitfall 2: Not Handling Concurrent Requests

**Problem:** Two users send message to same thread simultaneously → race condition

**Solution:** Use database transactions + row-level locking:

```typescript
await db.transaction(async (tx) => {
  await tx.select().from(threads).where(eq(threads.id, threadId)).for("update");
  await tx.insert(messages).values(message);
});
```

---

### Pitfall 3: Token Counting Inaccuracy

**Problem:** Token count off by 20% → rotation triggers too early/late

**Solution:** Use tiktoken library, test against actual LLM API:

```typescript
const estimated = tokenCounter.count(text);
const actual = (await llm.generate(text)).usage.inputTokens;
assert(Math.abs(estimated - actual) < estimated * 0.05); // Within 5%
```

---

### Pitfall 4: Proposal Execution Without Approval

**Problem:** Proposal marked approved in DB but never actually executed

**Solution:** Event-driven execution (approval event → worker):

```typescript
// In approval endpoint
await db.update(proposals).set({ status: "approved" });
await eventBus.publish("ai.proposal.approved", { proposalId }); // Critical
```

---

### Pitfall 5: Meta-Agent Infinite Loop

**Problem:** Rotation triggers immediately after rotation (validation passes but context still >70%)

**Solution:** Cooldown period + token threshold check:

```typescript
const lastRotation = await getLastRotation(threadId);
if (Date.now() - lastRotation.timestamp < 60000) {
  return; // Don't rotate within 1 minute
}
```
