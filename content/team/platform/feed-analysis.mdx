### 1.2 Data Flow: RSS Feed Execution

```
1. CRON triggers every minute
   └─→ feed-scheduler queries channels WHERE type='feed' AND due

2. For each due RSS feed:
   └─→ Enqueue feed-rss-execute job (pg-boss)

3. feed-rss-executor:
   ├─→ Fetch RSS routes via CP proxy (GET /api/rsshub-proxy/{route})
   ├─→ Parse RSS/Atom/JSON feed
   ├─→ Filter seen URLs (within tracking window)
   ├─→ Call IS classify_feed_items tool (or stub if not ready)
   ├─→ Filter by relevance score
   ├─→ Limit items (maxItemsPerRun)
   ├─→ Post messages (individual OR batch based on format.mode)
   │   ├─→ Check cross-feed references ("also in X feed")
   │   ├─→ Insert into messages table
   │   └─→ Emit side effects (notifications, socket events)
   └─→ Update feed status (lastRunAt, nextRunAt, itemCount)

4. Frontend displays:
   └─→ useChannelMessages poll/new messages
   └─→ FeedCell renders with platform icons, relevance badges
   └─→ CrossFeedChips show "Also in: Other Feed"
```

### 1.3 Data Flow: Proactive Feed (Morning Briefing)

```
1. CRON triggers at scheduled time (e.g., 8am weekdays)
   └─→ feed-scheduler queries proactive feeds

2. feed-proactive-executor:
   ├─→ Query workspace data:
   │   ├─→ Tasks due today
   │   ├─→ Pending proposals
   │   ├─→ Recent entities (last 24h)
   │   └─→ User captures
   ├─→ Call IS generate_feed_digest tool (or stub)
   ├─→ Generate summary in configured style (bullet/narrative/structured)
   ├─→ Post batch message to feed channel
   └─→ Optionally: Post to notification, chat (per delivery.surfaces)

3. Frontend displays digest as single message OR expanded items
```

### 1.4 Data Flow: On-Demand Trigger (Relay App)

```
1. User opens Relay app
   └─→ useRelayFeed mounts
   └─→ Check if "Relay Feed" channel exists (create if not)

2. Auto-trigger refresh:
   ├─→ Check last trigger time (respect 5min cooldown)
   ├─→ POST /api/feeds/trigger (or trpc.feeds.trigger.mutate)
   ├─→ Backend updates triggerRequestedAt
   ├─→ feed-scheduler picks up immediately (no wait for cron)
   └─→ Execute feed-proactive-executor

3. Pull-to-refresh:
   └─→ Same trigger flow
   └─→ Shows loading spinner
   └─→ Refreshes messages
```

## 2. Type Architecture Analysis

### 2.1 Critical Finding: Type Mismatch

**VERDICT:** The frontend and backend have **semantically incompatible** type definitions for `FeedMessageMetadata`.

| Field | Backend (@synap-core/types) | Frontend (local) | Status |
|-------|----------------------------|------------------|--------|
| `relevanceScore` | `number` (0-100) | `number` (0-1) | 🔴 **RANGE MISMATCH** |
| `topics` | ❌ Missing | `string[]` | 🔴 Missing in backend |
| `categories` | `string[]` | ❌ Missing | 🟡 Missing in frontend |
| `sourceUrl` | `string` | ❌ Missing | 🔴 Missing in frontend |
| `originalUrl` | ❌ Missing | `string` | 🔴 Missing in backend |
| `crossFeeds` | ❌ Missing | `CrossFeedReference[]` | 🔴 **Missing in backend entirely** |

**Impact:** Using these types interchangeably will cause:
- Relevance scores displayed incorrectly (100x too large or small)
- Missing topic tags in UI
- Broken cross-feed reference chips
- Type assertion runtime errors

### 2.2 Export Issues

**CRITICAL:** The `@synap-core/types` package does **NOT export** the feeds module:

```json
// synap-backend/packages/types/package.json exports
"./feeds": {
  "types": "./dist/feeds/index.d.ts",
  "import": "./dist/feeds/index.js"
}
// ^^^ THIS IS MISSING!
```

**Result:** Feed types CANNOT be imported properly. Frontend duplicates types locally.

### 2.3 Type Safety Issues in Relay

The relay-app uses excessive `any` types:

| File | Line | Issue |
|------|------|-------|
| `useFeed.ts:71` | `msg: any` | Should be `ChannelMessage` |
| `useFeed.ts:178` | `channels: any[]` | Should be `Channel[]` |
| `feed-helpers.ts:77` | `entity: any` | Should be `Entity` |

### 2.4 Recommended Type Architecture

```
synap-backend/packages/types/
├── feeds/
│   ├── index.ts           # Main exports
│   ├── config.ts          # RSSFeedConfig, ProactiveFeedConfig
│   └── message.ts         # FeedMessageMetadata (canonical)
└── package.json           # ADD "./feeds" export

synap-app/
└── NO LOCAL TYPES!        # Import from @synap-core/types/feeds

relay-app/
└── NO LOCAL TYPES!        # Import from @synap-core/types/feeds
```

**Canonical FeedMessageMetadata:**
```typescript
interface FeedMessageMetadata {
  feedItem: true;
  feedType: 'rss' | 'proactive';
  
  // Source (unified)
  source: {
    platform: string;        // "hackernews", "reddit"
    url: string;            // Original URL
    route?: string;         // "/hackernews/frontpage"
  };
  
  // AI processing (normalized)
  topics: string[];
  categories: string[];     // Alias for topics
  relevanceScore: number;   // 0-1 (frontend divides by 100 if backend sends 0-100)
  aiClassified: boolean;
  aiSummary?: string;
  
  // Cross-feed (added to backend)
  crossFeeds: Array<{
    feedId: string;
    feedTitle: string;
    postedAt: string;
  }>;
  
  // Batch info
  batched?: boolean;
  batchId?: string;
  batchIndex?: number;
}
```

## 5. User Flow Gaps

### 5.1 Missing: Feed Creation UI

**Current State:** Backend supports `feeds.createFeedChannel`, but no UI exists.

**Impact:** Users cannot create RSS feeds. Only programmatically created feeds work (morning briefing, relay feed).

**Recommendation:** Add feed creation flow (deferred per your earlier decision, but should be in roadmap).

### 5.2 Missing: Feed Discovery

**Current State:** Feed channels exist in DB but aren't prominently shown in Browser sidebar.

**Impact:** Users don't know they have feeds.

**Recommendation:** Add "Feeds" section to sidebar (like "Channels", "AI Assistants").

### 5.3 Partially Implemented: Capture/Dismiss

**Current State:** 
- Types define `isCaptured`, `isDismissed` flags
- UI has buttons for these actions
- NO persistence implementation

**Impact:** Capture/dismiss don't actually do anything.

**Recommendation:** Implement persistence:
```typescript
// New table: feed_item_states
{
  userId: string;
  messageId: string;
  channelId: string;
  isCaptured: boolean;
  isDismissed: boolean;
  capturedAt?: string;
  dismissedAt?: string;
}
```

### 5.4 Missing: Cross-Feed References

**Current State:** Types define `crossFeeds: CrossFeedReference[]`, but:
- Backend doesn't populate this field
- UI shows chips but they're always empty

**Impact:** "Also in: X feed" chips never appear.

**Recommendation:** Implement URL tracking across feeds:
```typescript
// When posting feed item:
const existingInOtherFeeds = await db.query.messages.findMany({
  where: sql`${messages.metadata}->'source'->>'url' = ${item.url}`,
  // ...
});
```
