# Raycast

> **Status:** Draft. This area is intentionally incomplete until Raycast scope is finalized.

A **Raycast extension** for Synap — enabling quick capture and power-user workflows through fast command-entry on macOS. Designed as a lightweight surface for capture, retrieval, and handoff into Browser or other canonical surfaces for deep workflows.

---

## Target Users

- **Power users** on macOS who already use Raycast as their launcher / quick-action tool
- **Raycast integration engineers** building and maintaining the extension
- **Product & ops teammates** evaluating rollout timing and scope boundaries

---

## Marketing

- **Positioning:** Fast command-entry surface for Synap — capture and lookup without leaving your keyboard flow.
- **Key differentiator:** No replacement for Browser; intentionally complementary, handoff-first integration.
- **Messaging constraints:** Keep first-response low-friction; expose failure and fallback clearly; no hidden side effects.

---

## Key User Stories

- As a power user, I want to **quickly capture** a note, task, or bookmark from anywhere on my Mac with a keyboard shortcut.
- As a user, I want to **search and retrieve** entities without opening the full Browser.
- As an operator, I want capture actions that **escalate into governed proposals** when writes are needed, without bypassing platform contracts.
- As a new user, I want Raycast to serve as a **lightweight on-ramp** — no deep setup, just install the extension and start capturing.

## Overview

Raycast is a **supplementary capture surface** for Synap, not a replacement for the Browser. The model is:

1. **Invoke command** in Raycast → 2. **Resolve identity/pod context** → 3. **Run read or propose write action** → 4. **Return short confirmation** with link to full surface if needed.

Key architectural constraints:
- Read/write actions **must use existing platform contracts** (tRPC + Hub Protocol).
- Write operations remain **permission/proposal governed** — no Raycast-specific bypass path.
- **No custom secrets** stored in local logs or prompts; reuse existing auth/session patterns.

---

## Architecture

```
Raycast Extension (macOS)
  ├─ Command invoker → resolves identity/pod context
  ├─ Read path → tRPC queries via platform contracts
  ├─ Write path → checkPermissionOrPropose() → proposals
  └─ Response → short confirmation + link to Browser for deep workflows
```

Boundaries:
- **Raycast extension** handles UX only — all data operations go through Synap platform.
- **Synap backend** (tRPC/Hono) is the authoritative data layer.
- **Intelligence Service** invoked only when commands call AI capabilities.
