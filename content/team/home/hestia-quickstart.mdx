## Phase 2: Hub Protocol Orion Router (Day 2-4)

### Files to Create

1. **New Router**:
   ```
   synap-backend/packages/api/src/routers/hub-protocol/orion.ts
   ```

2. **Mount in Main Router**:
   ```
   synap-backend/packages/api/src/routers/hub-protocol-rest.ts
   ```
   - Add: `import orionRouter from './hub-protocol/orion'`
   - Add: `app.route('/orion', orionRouter)`

### Endpoints to Implement

Priority 1 (Core):
- [ ] `POST /api/hub/orion/register` - Node registration
- [ ] `POST /api/hub/orion/heartbeat` - Health updates
- [ ] `GET /api/hub/orion/status/:nodeId` - Node status

Priority 2 (Builder):
- [ ] `POST /api/hub/orion/deploy` - Trigger deployment
- [ ] `POST /api/hub/orion/approval/:requestId/respond` - Approval flow

### Testing
```bash
# Test registration
curl -X POST http://localhost:4000/api/hub/orion/register \
  -H "Authorization: Bearer $API_KEY" \
  -d '{"hostname": "test-node", "installMode": "script", "role": "control-plane", "brainMode": "local"}'

# Test heartbeat
curl -X POST http://localhost:4000/api/hub/orion/heartbeat \
  -H "Authorization: Bearer $API_KEY" \
  -d '{"nodeId": "...", "services": [...], "phases": {...}}'
```

### Definition of Done
- [ ] All Priority 1 endpoints return 200
- [ ] Entities created in database
- [ ] Heartbeats update timestamps
- [ ] API tests pass

## Risk Mitigation

### Technical Risks

1. **Backend changes break existing functionality**
   - Mitigation: Comprehensive tests, feature flags

2. **USB installer fails on diverse hardware**
   - Mitigation: Test matrix of common hardware, fallback to manual install

3. **CLI separation breaks existing users**
   - Mitigation: Gradual rollout, clear migration guide

### Schedule Risks

1. **Backend review/approval takes longer than expected**
   - Mitigation: Start with templates (no code review needed)

2. **USB hardware logistics delays testing**
   - Mitigation: Prioritize script install, USB as stretch goal
