# Incident Runbook

Use this runbook as the first-response map for production incidents.

Priority incident classes:

- Pod lifecycle failures (stuck provision/update/deprovision)
- Intelligence Service downtime or degraded response
- Billing/webhook failures (Stripe signal mismatch)
- DNS, routing, or certificate misconfiguration

For each class: confirm symptoms, isolate scope, execute rollback/safe mode, then file postmortem notes.
